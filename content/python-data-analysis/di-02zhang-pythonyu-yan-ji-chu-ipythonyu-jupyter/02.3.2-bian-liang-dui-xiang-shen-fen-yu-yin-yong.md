## 条目概览

Python 中的变量不是装值的盒子，而是绑定到对象的名字。教材在变量和参数传递部分强调：赋值会创建变量名到对象的引用关系；多个变量名可以引用同一个对象。

## 1. 概念概览

Python 中的变量不是装值的盒子，而是绑定到对象的名字。教材在变量和参数传递部分强调：赋值会创建变量名到对象的引用关系；多个变量名可以引用同一个对象。

这一本质决定了很多运行结果题：`b = a` 不复制列表内容，只是让 `b` 绑定到 `a` 当前绑定的同一个对象。

## 2. 学习重点

- 赋值是绑定，不是默认复制。
- `==` 比较值是否相等。
- `is` 比较是否为同一个对象。
- Python 引用和 C 指针有直觉相似处，但不能等同。
- 可变对象被多个变量引用时，原地修改会从所有引用处可见。

## 3. 规范定义

| 概念 | 解释 |
| --- | --- |
| 变量名 | 绑定到对象的名字 |
| 对象 | Python 中实际承载值、类型和身份的实体 |
| 对象身份 | 对象的唯一身份，可用 `is` 判断是否相同 |
| 值相等 | 两个对象内容相等，用 `==` 判断 |
| 引用 | 变量名与对象之间的绑定关系 |

## 4. 直觉解释

变量名像标签，对象像真实物品。`b = a` 是给同一个物品再贴一个标签，不是复印一份物品。

```text
a ──┐
    ├──→ [1, 2, 3, 4]
b ──┘
```

## 5. 原理与机制

示例：

```python
a = [1, 2, 3]
b = a
b.append(4)

print(a)
print(b)
```

输出：

```python
[1, 2, 3, 4]
[1, 2, 3, 4]
```

因为 `append` 修改的是列表对象本身，而 `a` 和 `b` 绑定到同一个列表对象。

再看值相等和身份相同：

```python
a = [10, 20]
b = a
c = [10, 20]

print(a == b)  # True
print(a is b)  # True
print(a == c)  # True
print(a is c)  # False
```

`a` 和 `c` 的值相同，但不是同一个列表对象。

## 6. 关键步骤、分类或公式

判断代码影响范围：

1. 先看是否有 `b = a` 这类共享引用。
2. 再看后续操作是原地修改对象，还是重新绑定变量。
3. 最后看题目问的是值相等 `==` 还是身份相同 `is`。

复杂度方面，身份和值比较不是同一件事：

$$
\text{identity}(a,b) \neq \text{equality}(a,b)
$$

其中 `identity` 判断两个名字是否指向同一个对象，`equality` 判断对象内容是否相等。

## 7. 例题与具体案例

### 例题：重绑定不影响原对象

```python
a = [1, 2, 3]
b = a
b = [4, 5]

print(a)
print(b)
```

输出：

```python
[1, 2, 3]
[4, 5]
```

`b = [4, 5]` 是让 `b` 绑定到新列表，不是修改原列表。

## 8. 易错点与边界情况

> [!warning] 对话中暴露的易错点
> “直接把 `a` 复制给 `b`，复制的是里面的元素”是不准确的。教材中的变量语义更准确地说是：赋值绑定对象引用。

Python 引用与 C 指针对比：

| Python 引用 | C 指针 |
| --- | --- |
| 变量名绑定对象 | 指针保存地址 |
| 普通代码不能做指针运算 | 可以做地址运算 |
| 通常不需要手动释放内存 | 动态分配常需手动释放 |
| `is` 比较对象身份 | 可类比比较指针是否指向同一位置 |
