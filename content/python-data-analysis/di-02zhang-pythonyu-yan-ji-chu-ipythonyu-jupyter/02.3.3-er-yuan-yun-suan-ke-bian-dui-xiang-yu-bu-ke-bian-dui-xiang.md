## 条目概览

教材介绍 Python 的二元运算符和可变、不可变对象。二元运算符包括加减乘除、比较、逻辑判断等；可变性则决定对象创建后是否能被原地修改。

## 1. 概念概览

教材介绍 Python 的二元运算符和可变、不可变对象。二元运算符包括加减乘除、比较、逻辑判断等；可变性则决定对象创建后是否能被原地修改。

本节的核心不是背运算符表，而是判断一行代码到底是在创建新对象、重新绑定变量，还是修改原对象。

## 2. 学习重点

- 列表、字典、集合通常是可变对象。
- 字符串、元组、数字通常是不可变对象。
- `+` 与 `+=` 在可变对象和不可变对象上可能表现不同。
- 方法是否原地修改，要看对象类型和方法定义。
- Python 自动管理内存，但对象是否释放取决于是否仍被引用。

## 3. 规范定义

| 类型 | 是否可变 | 示例 |
| --- | --- | --- |
| `list` | 可变 | `[1, 2, 3]` |
| `dict` | 可变 | `{"name": "小王"}` |
| `set` | 可变 | `{1, 2, 3}` |
| `tuple` | 不可变 | `(1, 2, 3)` |
| `str` | 不可变 | `"Python"` |
| `int` / `float` | 不可变 | `10`, `3.14` |

## 4. 直觉解释

可变对象像白板，可以在原白板上改内容；不可变对象像印刷出来的纸，内容不能改，只能重新印一张新的。

## 5. 原理与机制

列表上的 `+=` 通常原地修改：

```python
x = [1, 2]
y = x
y += [3]

print(x)  # [1, 2, 3]
print(y)  # [1, 2, 3]
print(x is y)  # True
```

元组不可变，`+=` 会创建新元组并重新绑定变量：

```python
x = (1, 2)
y = x
y += (3,)

print(x)  # (1, 2)
print(y)  # (1, 2, 3)
print(x is y)  # False
```

## 6. 关键步骤、分类或公式

判断 `+=` 时：

1. 左侧对象是否可变？
2. 该类型是否为 `+=` 实现了原地修改？
3. 其他变量是否也引用同一个对象？

常见复杂度记忆：

| 操作 | 典型复杂度 | 说明 |
| --- | --- | --- |
| 列表尾部 `append` | 平均 $O(1)$ | 偶尔扩容 |
| 列表中间 `insert` | $O(n)$ | 后续元素需要移动 |
| 列表拼接 `a + b` | $O(n+m)$ | 创建新列表并复制引用 |
| 列表原地扩展 `a += b` | $O(m)$ | 将右侧元素追加到原列表 |

其中 $n$ 是左侧列表长度，$m$ 是右侧可迭代对象长度。

## 7. 例题与具体案例

### 例题：字符串方法不是原地修改

```python
s = "  Python  "
t = s.strip()

print(s)
print(t)
```

输出：

```text
  Python
Python
```

字符串不可变，`strip()` 返回新字符串，不会修改原字符串。

## 8. 易错点与边界情况

> [!warning] 对话中暴露的易错点
> “`+=` 一定在源对象上操作”不总是准确。对列表通常原地修改；对元组这类不可变对象，只能创建新对象并重新绑定变量。

内存管理边界：

- 对象不再被任何变量或容器引用后，才可能被回收。
- CPython 常使用引用计数，并配合循环垃圾回收。
- Python 进程不一定立刻把释放的内存还给操作系统，可能保留给后续对象复用。
