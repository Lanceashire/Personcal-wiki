## 条目概览

Python 解释器是执行 Python 代码的程序。教材从最基础的命令行解释器开始：可以在终端输入 python 进入交互模式，也可以把代码保存为 .py 文件后运行。

## 1. 概念概览

Python 解释器是执行 Python 代码的程序。教材从最基础的命令行解释器开始：可以在终端输入 `python` 进入交互模式，也可以把代码保存为 `.py` 文件后运行。

本节的重点不是展开完整编译原理，而是建立一个能解释报错的执行模型：Python 代码需要被解释器读取、分析并执行；有些错误在执行前能发现，有些错误只有运行到对应语句才会出现。

## 2. 学习重点

- Python 解释器负责执行 Python 代码。
- 交互模式适合快速试验小片段；脚本文件适合保存完整程序。
- 运行时错误只有执行到对应语句时才暴露。
- CPython 通常会先把源代码编译为字节码，再由虚拟机执行；这和“直接变成机器码”不是一回事。

## 3. 规范定义

| 概念 | 含义 |
| --- | --- |
| 源代码 | 人写的 `.py` 代码 |
| 解释器 | 执行 Python 代码的程序 |
| 交互模式 | 输入一行、执行一行，适合试验 |
| 脚本模式 | 运行一个 `.py` 文件 |
| 字节码 | CPython 的中间表示，通常由虚拟机执行 |
| 运行时错误 | 程序执行到某一步才出现的错误 |

## 4. 直觉解释

可以把解释器理解成代码的执行管家：

```text
Python 源代码
  -> 语法检查与编译为字节码
  -> Python 虚拟机执行字节码
  -> 调用底层系统和机器指令完成工作
```

这不是期末 Python 基础考试最核心的计算题，但它能帮助判断错误发生阶段。

## 5. 原理与机制

示例：

```python
print("开始")
x = 10 / 0
print("结束")
```

运行结果是先输出：

```text
开始
```

然后出现 `ZeroDivisionError`，`"结束"` 不会输出。原因不是“Python 和 C 翻译方式不同”这么简单，而是除零错误属于运行时错误，只有执行到 `10 / 0` 时才出现。

## 6. 关键步骤、分类或公式

常见错误分类：

| 错误类型 | 出现阶段 | 示例 |
| --- | --- | --- |
| 语法错误 | 执行前解析代码时 | 少括号、缩进错误 |
| 运行时错误 | 执行到某条语句时 | 除以零、对 `None` 调用 `.strip()` |
| 逻辑错误 | 程序能运行但结果不对 | 折扣公式写错、错误过滤掉 0 分 |

## 7. 例题与具体案例

判断下面代码的输出：

```python
print("A")
value = None
print(value.strip())
print("B")
```

结果：

- `"A"` 会输出；
- `value.strip()` 报 `AttributeError`；
- `"B"` 不会输出。

## 8. 易错点与边界情况

> [!warning] 易错点
> 汇编语言不等于机器语言。汇编仍需要转换为机器指令。常见 CPython 也不是把 Python 源代码“一下子翻译成最终机器码”。

边界情况：

- 语法错误会阻止程序开始执行。
- 运行时错误前面的语句可能已经执行并产生影响。
- 逻辑错误最危险，因为它不一定报错。
