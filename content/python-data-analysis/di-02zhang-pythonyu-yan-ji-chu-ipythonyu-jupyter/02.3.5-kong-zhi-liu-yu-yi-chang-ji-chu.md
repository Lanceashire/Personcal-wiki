## 条目概览

控制流决定程序语句的执行顺序。教材介绍 if、for、while、break、continue、pass、range 和三元表达式等基础结构。对数据分析来说，控制流常用于清洗前的小规模逻辑、构造列表、处理异常输入和理解推导式。

## 1. 概念概览

控制流决定程序语句的执行顺序。教材介绍 `if`、`for`、`while`、`break`、`continue`、`pass`、`range` 和三元表达式等基础结构。对数据分析来说，控制流常用于清洗前的小规模逻辑、构造列表、处理异常输入和理解推导式。

## 2. 学习重点

- `if` 根据条件决定是否执行代码块。
- `for` 遍历可迭代对象。
- `while` 在条件为真时重复执行。
- `break` 终止循环，`continue` 跳过本轮，`pass` 表示占位不做事。
- 三元表达式用于在表达式层面做二选一转换。
- 异常不是“代码结构错”本身，而是运行中发生的错误事件。

## 3. 规范定义

| 结构 | 用途 | 示例 |
| --- | --- | --- |
| `if / elif / else` | 条件分支 | 判断成绩是否有效 |
| `for` | 遍历序列或迭代器 | 遍历列表、字典、行 |
| `while` | 条件循环 | 重复直到满足条件 |
| `break` | 退出整个循环 | 找到目标后停止 |
| `continue` | 跳过当前轮 | 遇到无效值就跳过 |
| `pass` | 空语句占位 | 暂时不写逻辑 |
| 三元表达式 | 表达式内条件选择 | `x if cond else y` |

## 4. 直觉解释

控制流就是程序的路线图。没有控制流，程序只能一行一行直走；有了控制流，程序可以根据数据状态选择不同路径。

## 5. 原理与机制

示例：过滤有效成绩。

```python
scores = [85, None, 0, 92]
valid = []

for score in scores:
    if score is None:
        continue
    if score <= 0:
        continue
    valid.append(score)

print(valid)
```

输出：

```python
[85, 92]
```

这里 `continue` 的作用是跳过无效值，不让后续代码处理它。

## 6. 关键步骤、分类或公式

三元表达式：

```python
value_if_true if condition else value_if_false
```

示例：

```python
label = "pass" if score >= 60 else "fail"
```

与推导式结合时要注意两种 `if`：

```python
result = [
    x ** 2 if x % 2 == 0 else -x
    for x in data
    if x > 0
]
```

- 前面的 `if ... else ...` 是转换逻辑；
- 末尾的 `if x > 0` 是筛选逻辑。

## 7. 例题与具体案例

### 例题：等价改写

普通循环：

```python
result = []
for x in data:
    if x > 0:
        if x % 2 == 0:
            result.append(x ** 2)
        else:
            result.append(-x)
```

列表推导式：

```python
result = [x ** 2 if x % 2 == 0 else -x for x in data if x > 0]
```

## 8. 易错点与边界情况

> [!warning] 对话中暴露的易错点
> 推导式中前面的 `if ... else ...` 不是筛选，而是转换；末尾的 `if` 才是筛选。

边界情况：

- `while` 循环要确保条件最终能变为假，否则会死循环。
- `range(stop)` 不包含 `stop`。
- `break` 只退出当前最近一层循环。
- `continue` 只跳过当前这一轮。
