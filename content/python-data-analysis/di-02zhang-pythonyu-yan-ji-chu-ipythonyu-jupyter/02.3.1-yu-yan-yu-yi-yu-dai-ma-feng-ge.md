## 条目概览

Python 的语言语义包括代码如何分块、如何调用函数、如何访问属性和方法、如何写注释。教材强调 Python 使用缩进表示代码块，这和许多使用花括号的语言不同。

## 1. 概念概览

Python 的语言语义包括代码如何分块、如何调用函数、如何访问属性和方法、如何写注释。教材强调 Python 使用缩进表示代码块，这和许多使用花括号的语言不同。

本节目标是建立“代码结构感”：看到一段 Python 代码时，能判断哪些语句属于同一个代码块，哪些函数会被调用，哪些对象方法会被执行。

## 2. 学习重点

- 缩进决定代码块，不是单纯为了美观。
- `#` 后面的内容是注释，不参与执行。
- 函数调用使用括号：`func(x, y)`。
- 属性和方法通过点号访问：`obj.attr`、`obj.method()`。
- Python 变量和对象的关系会在下一节详细展开。

## 3. 规范定义

| 语法元素 | 示例 | 含义 |
| --- | --- | --- |
| 缩进代码块 | `if x > 0:` 下一行缩进 | 表示从属于 `if` 的代码 |
| 注释 | `# this is a comment` | 给人看的说明，解释器忽略 |
| 函数调用 | `print(value)` | 调用函数执行某个动作 |
| 方法调用 | `name.strip()` | 调用对象所属类型提供的函数 |
| 属性访问 | `obj.shape` | 访问对象上的属性 |

## 4. 直觉解释

Python 的缩进就像文章的层级。缩进错了，不只是排版难看，而是代码含义变了。

```python
score = 85

if score >= 60:
    print("及格")
    print("进入统计")
print("程序结束")
```

前两个 `print` 属于 `if` 代码块；最后一个 `print` 不属于。

## 5. 原理与机制

缩进错误属于语法层面的问题，通常在程序真正运行前就会被发现。方法调用错误则可能是运行时错误。例如：

```python
x = None
x.strip()
```

这段代码语法没问题，但运行时会报错，因为 `None` 没有字符串方法 `.strip()`。

## 6. 关键步骤、分类或公式

读代码时按顺序识别：

1. 看缩进，确定代码块边界。
2. 看函数调用，确定哪些动作会执行。
3. 看点号，确定是在访问属性还是调用方法。
4. 看对象类型，判断方法是否存在。

## 7. 例题与具体案例

判断输出：

```python
score = 59

if score >= 60:
    print("pass")
print("checked")
```

输出：

```text
checked
```

`print("checked")` 没有缩进，不属于 `if`。

## 8. 易错点与边界情况

> [!warning] 易错点
> Python 的缩进不是装饰。多一个空格、少一个空格，都可能让代码块结构改变，甚至直接触发 `IndentationError`。

边界情况：

- 只要条件语句后有冒号，下一层通常需要缩进。
- Notebook 中复制代码时容易混入不一致缩进。
- 方法调用必须确认对象类型，例如 `None.strip()` 一定错误。
