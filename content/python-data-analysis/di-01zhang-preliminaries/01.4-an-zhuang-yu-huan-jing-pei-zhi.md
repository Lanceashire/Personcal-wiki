## 条目概览

数据分析学习不只需要会写 Python，还需要一个稳定、可复现的运行环境。教材第一章推荐使用 conda 生态来管理 Python 版本和第三方包，并介绍 Miniforge、必要包安装、IDE 与编辑器选择。

## 1. 概念概览

数据分析学习不只需要会写 Python，还需要一个稳定、可复现的运行环境。教材第一章推荐使用 conda 生态来管理 Python 版本和第三方包，并介绍 Miniforge、必要包安装、IDE 与编辑器选择。

环境配置的目标不是“装得越多越好”，而是保证后续代码能稳定运行、包版本可控、Notebook 与脚本都能使用同一个解释器环境。

## 2. 学习重点

- 环境管理解决的是“不同项目依赖不同版本”的问题。
- 包管理解决的是“安装、升级、卸载第三方库”的问题。
- IDE 或编辑器解决的是“在哪里写、运行、调试代码”的问题。
- 初学数据分析时，优先追求稳定和可复现，而不是追求复杂配置。

## 3. 规范定义

| 概念 | 含义 | 典型工具 |
| --- | --- | --- |
| Python 解释器 | 实际执行 Python 代码的程序 | CPython |
| 虚拟环境 | 隔离某个项目依赖的一套环境 | conda env、venv |
| 包管理器 | 安装和管理第三方包 | conda、pip |
| 发行版 / 环境方案 | 打包 Python 与常用科学计算库的方案 | Miniforge、Anaconda |
| IDE / 编辑器 | 写代码、运行、调试的界面 | VS Code、PyCharm、JupyterLab |

## 4. 直觉解释

可以把环境理解成“厨房”。Python 是灶台，NumPy、pandas、Matplotlib 是工具和食材。不同项目需要不同食材版本，如果所有东西都扔进同一个厨房，迟早会互相影响。虚拟环境就是给每个项目准备独立厨房。

## 5. 原理与机制

### 5.1 为什么需要环境隔离

假设项目 A 需要旧版本 pandas，项目 B 需要新版本 pandas。如果都装在同一个全局 Python 中，升级 B 可能破坏 A。独立环境可以让每个项目拥有自己的包版本。

### 5.2 conda 与 pip 的关系

conda 不只安装 Python 包，也能管理一些非 Python 依赖；pip 主要安装 Python 包。实际学习中可以优先使用 conda-forge 环境安装科学计算栈，必要时再用 pip 补充。

### 5.3 Notebook 与脚本环境一致

Jupyter Notebook 背后也要连接某个 Python kernel。如果 Notebook 使用的 kernel 与命令行环境不同，就会出现“明明安装了包但 Notebook 找不到”的问题。

## 6. 关键步骤、分类或公式

推荐环境检查流程：

```text
确认 Python 版本
  ↓
创建独立环境
  ↓
安装 numpy pandas matplotlib jupyter scipy scikit-learn statsmodels
  ↓
在终端和 Notebook 中分别验证 import
  ↓
开始学习和练习
```

示例命令：

```powershell
conda create -n pydata python=3.11
conda activate pydata
conda install -c conda-forge numpy pandas matplotlib jupyterlab scipy scikit-learn statsmodels
```

验证：

```python
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt

print(np.__version__)
print(pd.__version__)
```

## 7. 例题与具体案例

### 例题：判断环境问题

现象：

```text
ModuleNotFoundError: No module named 'pandas'
```

排查顺序：

1. 当前 Python 环境是否就是安装 pandas 的环境？
2. 在终端中运行的 Python 与 Notebook kernel 是否一致？
3. 是否只在另一个环境里安装了 pandas？
4. 是否拼写错误，例如 `import panda`？

## 8. 易错点与边界情况

> [!warning] 易错点
> 不要看到 `ModuleNotFoundError` 就盲目重复安装。先确认当前解释器和包安装位置是不是同一个环境。

常见问题：

- 终端能 import，Notebook 不能 import：多半是 kernel 不一致。
- VS Code 运行失败：可能选错解释器。
- pip 和 conda 混用过多：可能造成版本冲突。
- 系统里有多个 Python：命令行调用的可能不是你以为的那个。
