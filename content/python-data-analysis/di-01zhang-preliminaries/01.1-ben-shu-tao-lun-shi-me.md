## 条目概览

教材第一章首先限定全书主题：用 Python 完成数据分析工作。这里的“数据分析”不是只会调用某个库函数，而是一套从数据到结论的工作流，包括读取数据、理解字段、清洗错误、转换结构、探索规律、可视化结果，并在必要时建立统计或机器学习模型。

## 1. 概念概览

教材第一章首先限定全书主题：用 Python 完成数据分析工作。这里的“数据分析”不是只会调用某个库函数，而是一套从数据到结论的工作流，包括读取数据、理解字段、清洗错误、转换结构、探索规律、可视化结果，并在必要时建立统计或机器学习模型。

本节要建立一个重要判断：学习 Python 数据分析时，Python 语法本身只是地基；真正目标是用语法组织数据处理过程，用库处理规模更大、结构更复杂的真实数据。

## 2. 学习重点

- 明确本书不是通用 Python 百科，而是面向数据分析任务。
- 数据分析关注的是“数据结构 + 计算方法 + 解释结果”的组合。
- 第一章只建立地图，不急着深入所有工具细节。
- 对话中已经暴露的容器、引用、切片和推导式问题，会直接影响后续 pandas 与 NumPy 的理解。

## 3. 规范定义

在本课程笔记中，数据分析可以理解为：

> 对原始数据进行获取、清洗、转换、探索、建模、可视化和解释，从而支持判断或决策的过程。

常见数据形态包括：

| 数据形态 | 示例 | 后续常用工具 |
| --- | --- | --- |
| 表格数据 | CSV、Excel、数据库表 | pandas |
| 数值数组 | 向量、矩阵、时间序列数值 | NumPy |
| 文本字段 | 姓名、类别、评论、日志 | Python 字符串方法、pandas 字符串方法 |
| 时间数据 | 日期、时间戳、周期 | pandas 时间序列功能 |
| 分类变量 | 性别、城市、等级、标签 | pandas、统计建模、可视化 |

## 4. 直觉解释

可以把数据分析看成一道连续工序：

```text
问题 -> 数据 -> 清洗 -> 转换 -> 分析 -> 可视化 -> 结论
```

如果前面的数据理解和清洗出错，后面的图表和模型会显得很“高级”，但结论可能完全不可靠。

## 5. 原理与机制

数据分析中的每一步都有输入和输出：

| 步骤 | 输入 | 输出 | 常见错误 |
| --- | --- | --- | --- |
| 提出问题 | 业务背景 | 可回答的问题 | 问题太泛，无法用数据验证 |
| 理解字段 | 原始数据说明 | 字段含义表 | 把编码值误认为真实数值 |
| 检查质量 | 数据表 | 缺失、重复、异常报告 | 只看前几行就开始分析 |
| 清洗转换 | 原始字段 | 可分析字段 | 把 `None`、空字符串、空格混为一谈 |
| 分析建模 | 干净数据 | 指标、模型、比较 | 混淆相关与因果 |
| 可视化解释 | 分析结果 | 图表与结论 | 图表美观但不能回答问题 |

## 6. 关键步骤、分类或公式

第一章不涉及复杂公式，但可以用一个简单的流程约束来检查完整性：

$$
\text{可靠结论} = f(\text{明确问题}, \text{可靠数据}, \text{合适方法}, \text{谨慎解释})
$$

其中：

- 明确问题：知道要回答什么；
- 可靠数据：字段含义和质量经过检查；
- 合适方法：分析方法与问题匹配；
- 谨慎解释：不把相关关系随意说成因果关系。

## 7. 例题与具体案例

给定一个学生成绩表：

| name | score | class |
| --- | ---: | --- |
| 张三 | 85 | A |
| 李四 |  | A |
| 王五 | 0 | B |
| 赵六 | 92 | B |

数据分析前不能只问“平均分是多少”，还要先问：

- 空白成绩是缺考、未录入，还是无效数据？
- `0` 是真实成绩，还是缺失值编码？
- 是否要按班级分组比较？
- 是否有重复学生？

这些问题对应对话中已经出现过的 `None`、空字符串、空白字符串、`score > 0` 等判断。

## 8. 易错点与边界情况

> [!warning] 易错点
> 数据分析不是“先把代码写出来再说”。如果字段含义不清，代码越熟练，错误传播越快。

常见边界情况：

- 缺失值不一定都写成 `None`，也可能是空字符串、空格、`NA`、`-999`。
- 数字 `0` 有时是真实值，不能自动当作缺失。
- 表格里的文本字段常包含首尾空格，需要清洗。

## 9. 与相关知识点的联系

- 后续 Python 基础：变量、容器、字符串方法决定能否写出正确清洗逻辑。
- 后续 pandas：`DataFrame` 是表格数据分析的核心结构。
- 后续 Matplotlib：可视化用于把分析结果变成可解释图形。
