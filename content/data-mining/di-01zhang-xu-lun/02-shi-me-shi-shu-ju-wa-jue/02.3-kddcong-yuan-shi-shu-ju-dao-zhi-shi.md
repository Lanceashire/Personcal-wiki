## 条目概览

Knowledge Discovery in Databases 数据库中的知识发现

## 1. KDD 是什么

KDD：

```text
Knowledge Discovery in Databases
数据库中的知识发现
```

KDD 指：

> 把未经加工的原始数据转换为有用信息的完整过程。

数据挖掘是 KDD 中不可缺少的环节，但不是全部。

---

## 2. 完整链条

```text
输入数据
→ 数据预处理
→ 数据挖掘
→ 后处理
→ 信息与知识
→ 决策支持
```

---

## 3. 输入数据可能来自哪里

- 平面文件
- CSV
- Excel
- 关系表
- 集中式数据库
- 分布式站点
- 日志
- API
- 传感器

---

## 4. 为什么“原始数据”不能直接使用

原始数据常常包含：

- 缺失值
- 重复记录
- 噪声
- 错误字段
- 不同编码
- 不同单位
- 不同粒度
- 无关属性
- 隐私字段

例如用户表和订单表可能分别使用：

```text
U001
```

和：

```text
0001
```

表示同一个用户。
如果不统一，连接会失败。

---

## 5. KDD 中各环节作用

| 环节 | 作用 | 示例 |
|---|---|---|
| 数据融合 | 合并来源 | 用户表 + 订单表 |
| 数据清洗 | 修正问题 | 去除重复、处理缺失 |
| 记录选择 | 选择对象 | 仅分析近一年客户 |
| 特征选择 | 选择属性 | 保留购买频率、金额 |
| 数据挖掘 | 发现模式 | 聚类、分类、关联 |
| 后处理 | 评估与解释 | 可视化、统计检验 |
| 决策支持 | 接入行动 | 推送优惠券 |

---

## 6. 直觉比喻

```text
原始矿石
→ 清洗
→ 筛选
→ 提炼
→ 检验
→ 加工为产品
```

算法只是“提炼”的一环。

---

## 7. 易错点

```text
训练模型
≠
完整 KDD
```

```text
数据清洗
≠
附属工作
```

```text
发现规则
≠
规则可以直接上线
```
