## 3. 为什么需要数据挖掘

### 3.1 数据越来越多，但“数据多”不等于“知识多”

一张只有几十行的表格，人可以手动浏览并发现问题。但当数据变成数百万条交易记录、持续生成的网站日志、大量医学影像、覆盖不同时间与地点的气候观测，或者包含成千上万个基因特征的数据集时，仅靠人工浏览已经不现实。

这时需要自动化方法帮助我们完成两件事：

1. 从已有数据中发现规律；
2. 根据已有规律预测未来或未知结果。

### 3.2 典型应用场景

| 场景 | 可用数据 | 希望解决的问题 |
|---|---|---|
| 零售与电子商务 | POS 记录、浏览日志、客服记录 | 顾客分群、个性化推荐、交叉销售、欺诈检测 |
| Web 与社交网络 | 页面浏览、文本、图像、视频、社交关系 | 搜索排序、垃圾信息过滤、推荐、内容理解 |
| 物联网 | 智能手机、可穿戴设备、低成本传感器 | 智能家居、城市规划、安全监控、节能 |
| 地球科学 | 卫星、海洋、大气、温度和降水观测 | 气候关系分析、生态扰动识别、趋势预测 |
| 生物信息学 | 基因表达、序列和蛋白质数据 | 基因功能理解、疾病相关因素分析 |
| 医疗健康 | 电子健康记录、ECG、MRI、fMRI | 疾病辅助判断、患者分层、精准医疗 |

> [!warning] 易错点
> 计算总销售额、按学号排序、检索某条记录，通常不属于数据挖掘。这些任务虽然处理数据，但没有尝试从数据中发现新的、可推广的规律。

## 4. 什么是数据挖掘

### 4.1 核心定义

数据挖掘关注的是：

> 在大型数据集合中，自动发现有价值的信息、模式或关系，并在某些情况下预测未来观测结果。

三个关键词：

| 关键词 | 含义 |
|---|---|
| 大型数据集合 | 数据规模或复杂度超出人工逐条分析能力 |
| 自动发现 | 借助算法，而不是完全依赖人工观察 |
| 有价值的信息 | 结果必须对理解问题、决策或预测有帮助 |

### 4.2 数据挖掘不等于普通查询

| 问题 | 是否通常属于数据挖掘 | 原因 |
|---|---:|---|
| 查询顾客 A 的订单记录 | 否 | 已知目标，只需检索 |
| 找出含有某关键词的网页 | 通常不是 | 主要依赖索引和检索 |
| 识别哪些顾客可能流失 | 是 | 需要从历史数据中学习模式 |
| 发现哪些商品经常一起购买 | 是 | 需要发现隐含关系 |
| 判断一笔交易是否异常 | 是 | 需要建立“正常模式”并比较 |

#### 边缘情况

搜索系统本身主要完成检索，但可以使用数据挖掘方法提升结果质量，例如学习排序、个性化推荐、垃圾内容识别。因此，现实系统中“查询”和“挖掘”往往协同工作，而不是绝对分离。

## 5. 数据挖掘与 KDD

### 5.1 KDD 是完整流程，数据挖掘是其中关键环节

KDD（Knowledge Discovery in Databases，数据库中的知识发现）指从原始数据出发，经过准备、挖掘、验证和解释，最终得到有用知识的完整过程。

```text
原始数据
→ 数据预处理
→ 数据挖掘
→ 后处理
→ 可用于理解或决策的知识
```

### 5.2 为什么数据预处理重要

现实数据往往存在多源融合、噪声、重复记录、无关属性、格式不统一等问题。数据预处理的目标是把原始输入转换为适合后续分析的形式。

常见工作包括：

1. 数据集成；
2. 数据清洗；
3. 删除重复观测；
4. 选择与任务相关的记录；
5. 选择有用特征；
6. 转换数据格式。

> [!note] 后续展开
> 第 2 章会进一步讨论数据类型、数据质量、数据预处理以及相似性与相异性度量。本章只建立整体概念。

### 5.3 为什么后处理重要

算法输出的模式不一定都值得采用。后处理用于判断结果是否有效、删除虚假模式、通过可视化辅助解释，并将结果集成进决策系统。

> [!warning] 易错点
> 数据挖掘不是“把算法跑出来就结束”。数据准备、结果验证和落地解释同样重要。

## 6. 数据挖掘要解决的挑战

### 6.1 可伸缩性

数据规模增大后，原本可以运行的算法可能变得过慢，甚至无法在内存中完成。常见思路包括抽样、核外算法、并行计算、分布式计算和更合理的数据结构。

### 6.2 高维性

一个对象可能包含大量属性，例如基因表达数据、长时间序列、图像或文本表示。维数增加后，传统方法可能性能下降，计算成本也可能迅速增长。

后续关联：维归约、特征选择、维灾难、数据。

### 6.3 异构数据与复杂数据

现实数据并不总是一张规则表格，还可能包含文本、图像、视频、序列、图结构、时空数据和半结构化日志。处理这类数据时，需要保留时间、空间、连接关系和文档结构等信息。

### 6.4 数据所有权、分布与隐私

数据可能分散在多个机构中，无法简单汇总。分布式数据挖掘需要考虑：

1. 如何降低通信成本；
2. 如何合并不同节点的结果；
3. 如何保护隐私和数据安全；
4. 模型是否可能对特定群体产生偏差；
5. 结果是否可以解释。

## 7. 数据挖掘从哪里来

数据挖掘是跨学科领域：

| 来源领域 | 提供的思想 |
|---|---|
| 统计学 | 抽样、估计、假设检验 |
| 机器学习 | 建模、学习理论、分类与预测 |
| 人工智能 | 搜索与推理 |
| 模式识别 | 从观测中识别结构 |
| 数据库 | 高效存储、访问和查询 |
| 信息论 | 信息量与不确定性度量 |
| 最优化 | 寻找更优参数或解 |
| 可视化 | 辅助解释模式 |
| 分布式计算 | 处理大规模数据 |

### 7.1 数据挖掘与数据科学

数据科学比单一算法更强调综合使用多种工具，从数据中产生可用洞见。

```text
数据科学
├── 数据采集与工程
├── 数据清洗
├── 统计分析
├── 数据挖掘
├── 机器学习
├── 可视化
└── 业务或科学解释
```

> [!warning] 边缘情况
> 纯粹依赖数据驱动方法可能受到抽样偏差、因果关系不清、黑箱模型难解释和领域知识缺失的影响。高预测精度不自动等于形成了可靠的科学解释。

## 8. 两大类数据挖掘任务

### 8.1 预测任务

目标：根据其他属性预测某一个特定属性。

| 术语 | 含义 |
|---|---|
| 目标变量 / 因变量 | 希望预测的属性 |
| 解释变量 / 自变量 | 用来进行预测的属性 |

例如：根据花瓣长度、花瓣宽度预测鸢尾花类别；根据历史交易判断信用卡交易是否可能欺诈。

### 8.2 描述任务

目标：找出数据中潜在的联系、结构、趋势、轨迹、簇或异常。描述任务通常具有探索性，发现模式后还需要验证和解释。

## 9. 四类核心任务

### 9.1 预测建模

预测建模建立目标变量与解释变量之间的映射关系。

- 分类：目标变量是离散类别，例如预测鸢尾花类别。
- 回归：目标变量是连续数值，例如预测未来销售额或价格。

| 分类 | 回归 |
|---|---|
| 预测离散类别 | 预测连续数值 |
| 是否欺诈 | 预计损失金额 |
| 是否患病 | 预计住院时长 |

### 9.2 关联分析

关联分析寻找数据中经常共同出现的特征或项目。

```text
{商品 A} → {商品 B}
```

典型应用是 购物篮分析：从交易记录中发现可用于交叉销售或促销设计的商品关系。

> [!warning] 易错点
> “经常同时出现”不等于“存在因果关系”。

### 9.3 聚类分析

聚类分析将相似对象放入同一组，并尽量让不同组之间差异更明显。

典型应用：顾客分群、新闻文章按主题分组、气候区域划分、数据压缩。

| 分类 | 聚类 |
|---|---|
| 已有标签 | 通常没有标签 |
| 学习如何预测类别 | 探索数据自然分成哪些组 |
| 属于预测任务 | 通常属于描述任务 |

### 9.4 异常检测

异常检测寻找特征明显不同于大多数对象的观测值。这些对象也称 异常点 或 离群点。

典型场景：信用卡欺诈、网络攻击、疾病异常模式、生态扰动。

评价重点：

- 检测率：真正异常中有多少被发现；
- 误报率：正常对象中有多少被错标为异常。

> [!warning] 边缘情况
> 异常不一定等于错误。它可能是录入错误，也可能是罕见但真实的重要事件。

## 11. 公式与变量说明

本章主要建立概念框架，没有要求掌握复杂公式。但需要提前理解预测任务的通用形式：

$$
\hat{y} = f(x_1, x_2, \ldots, x_d)
$$

| 符号 | 含义 |
|---|---|
| $x_1, x_2, \ldots, x_d$ | 用于预测的解释变量或特征 |
| $d$ | 特征数量 |
| $f$ | 从数据中学习得到的模型 |
| $\hat{y}$ | 模型给出的预测结果 |
| $y$ | 真实结果 |

## 15. 易混淆概念总表

| 容易混淆的概念 | 核心区别 |
|---|---|
| 数据挖掘 vs 普通查询 | 查询检索已知目标；挖掘发现未知但有用的模式 |
| 数据挖掘 vs KDD | 数据挖掘是关键步骤；KDD 是完整过程 |
| 数据挖掘 vs 数据科学 | 数据科学范围更广，包含工程、统计、挖掘、解释和落地 |
| 分类 vs 聚类 | 分类通常有标签；聚类通常无标签 |
| 分类 vs 回归 | 分类预测离散值；回归预测连续值 |
| 关联 vs 因果 | 共同出现不代表因果关系 |
| 异常 vs 错误 | 异常可能是错误，也可能是罕见但真实的重要事件 |
