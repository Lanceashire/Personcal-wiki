## 条目概览

散列冲突是指不同关键字被映射到同一位置。 常见处理方法包括链地址法与开放定址法。

> [!summary]
> 散列冲突是指不同关键字被映射到同一位置。
> 常见处理方法包括链地址法与开放定址法。

## 一、为什么冲突无法完全避免？

假设散列表只有 `7` 个位置，但需要存储大量不同关键字。

散列函数：

$$
H(key)=key \bmod 7
$$

插入：

```text
10 → 3
24 → 3
```

不同关键字得到相同位置。

这就是冲突。

当关键字数量多于可用位置时，根据抽屉原理，冲突更不可能完全避免。

---

## 二、链地址法

### 1. 思想

散列表的每个位置保存一条链表。

发生冲突的元素挂在同一条链上。

```text
下标 0 → NULL
下标 1 → 8 → 15 → 22 → NULL
下标 2 → 9 → NULL
```

### 2. 查找过程

1. 计算散列地址。
2. 进入对应链表。
3. 沿链表顺序查找。

### 3. 节点定义

```c
typedef struct HashNode {
    int key;
    struct HashNode *next;
} HashNode;
```

### 4. 查找代码

```c
HashNode *searchChain(HashNode *table[], int tableSize, int key) {
    int index = key % tableSize;
    HashNode *p = table[index];

    while (p != NULL) {
        if (p->key == key) {
            return p;
        }

        p = p->next;
    }

    return NULL;
}
```

### 5. 优点

- 实现直观。
- 删除节点较自然。
- 表较满时仍然可以继续插入。
- 冲突元素集中在局部链表中。

### 6. 缺点

- 需要额外指针空间。
- 链表过长时，查找效率下降。
- 节点分散存储，缓存局部性通常较差。

## 三、开放定址法

### 1. 思想

所有元素都直接存储在数组中。

发生冲突时，按照探测规则寻找下一个可用位置。

```text
初始位置冲突
    ↓
继续探测其他位置
    ↓
找到空槽
```

### 2. 线性探测

公式：

$$
H_i(key)=(H(key)+i)\bmod m
$$

其中：

```text
i = 0, 1, 2, ...
```

例如，表长为 `7`：

```text
10 % 7 = 3
24 % 7 = 3  → 冲突 → 尝试 4
31 % 7 = 3  → 冲突 → 尝试 4 → 冲突 → 尝试 5
```

### 3. 线性探测代码骨架

```c
#define EMPTY 0
#define OCCUPIED 1
#define DELETED 2

typedef struct {
    int key;
    int state;
} Entry;

int searchLinear(const Entry table[], int tableSize, int key) {
    int start = key % tableSize;

    for (int i = 0; i < tableSize; i++) {
        int index = (start + i) % tableSize;

        if (table[index].state == EMPTY) {
            return -1;
        }

        if (table[index].state == OCCUPIED &&
            table[index].key == key) {
            return index;
        }
    }

    return -1;
}
```

## 四、为什么删除时需要墓碑标记？

开放定址法中，不能把删除后的槽位简单改成：

```text
从未使用过的 EMPTY
```

否则查找可能过早停止。

通常使用：

```text
DELETED
```

表示墓碑：

> 这里原来有元素，后续探测链可能仍然有效。

---

## 五、二次探测

线性探测可能出现连续聚集。

二次探测尝试使用：

$$
H_i(key)=(H(key)+i^2)\bmod m
$$

目的：

> 减少大量元素挤在连续位置上的现象。

---

## 六、双重散列

使用第二个散列函数确定步长：

$$
H_i(key)=(H_1(key)+i\times H_2(key))\bmod m
$$

相比固定步长，通常可以获得更分散的探测序列。

---

## 七、链地址法与开放定址法对比

| 对比项 | 链地址法 | 开放定址法 |
|---|---|---|
| 存储位置 | 数组槽位指向链表 | 所有元素直接存入数组 |
| 冲突处理 | 挂到链表 | 继续探测其他槽位 |
| 删除 | 较自然 | 通常需要墓碑标记 |
| 额外空间 | 需要指针 | 不需要链表节点指针 |
| 表较满时 | 仍可插入链表 | 性能明显下降 |
| 缓存局部性 | 通常较弱 | 通常较好 |

---

## 八、常见错误

1. 认为不同关键字映射到同一位置就是程序出错。
2. 开放定址删除时直接改成 `EMPTY`。
3. 查找时忘记限制探测次数，导致死循环。
4. 线性探测只检查初始位置。
5. 链地址法忘记处理空链表。
6. 把“散列函数”和“探测函数”混为一谈。

## 十、核心结论

> [!important]
> 冲突不是异常情况，而是散列表设计中必须预先处理的正常情况。

必须掌握：

1. 链地址法：同一槽位挂链表。
2. 开放定址法：冲突后继续探测数组槽位。
3. 线性探测容易产生聚集。
4. 删除开放定址元素时通常需要墓碑。
5. 冲突越严重，查找成本通常越高。
