## 条目概览

散列函数负责将关键字映射为散列表位置。 好的散列函数应当计算快速、分布较均匀，并尽量减少冲突。

> [!summary]
> 散列函数负责将关键字映射为散列表位置。
> 好的散列函数应当计算快速、分布较均匀，并尽量减少冲突。

## 一、设计目标

一个实用的散列函数通常应满足：

1. 计算速度快。
2. 映射结果落在合法数组范围内。
3. 尽量让关键字分布均匀。
4. 尽量减少大量关键字集中到同一位置。
5. 充分利用关键字本身的信息。

---

## 二、直接定址法

形式：

$$
H(key)=key
$$

或者：

$$
H(key)=a\times key+b
$$

适合关键字范围较小、较连续的情况。

例如：

```text
学号范围只有 0 到 999
```

可以直接使用学号作为下标。

### 优点

- 简单。
- 冲突少，甚至没有冲突。

### 缺点

如果关键字范围极大而实际数据很少，会浪费大量空间。

---

## 三、除留余数法

最常用的形式之一：

$$
H(key)=key \bmod p
$$

其中：

```text
p 通常不大于表长
```

例如：

$$
H(key)=key \bmod 7
$$

| 关键字 | 结果 |
|---:|---:|
| 10 | 3 |
| 18 | 4 |
| 24 | 3 |

`10` 与 `24` 发生冲突。

### 选择模数时的考虑

模数选择不合理，可能导致分布不均匀。

教材中常建议选择合适的质数，以减少某些规律性数据带来的聚集。

---

## 四、数字分析法

观察关键字各位数字的分布，选取分布较均匀的部分作为散列地址。

例如，若大量手机号前几位相同，则不应只使用前几位。

适合：

```text
提前知道关键字分布
```

缺点：

```text
依赖数据特征，通用性较差
```

---

## 五、平方取中法

步骤：

1. 将关键字平方。
2. 取平方结果中间若干位。
3. 作为散列地址。

思想：

> 中间位通常受到关键字各位数字共同影响。

---

## 六、折叠法

将较长关键字拆成若干段，再进行相加或其他组合。

例如：

```text
123456789
→ 123 + 456 + 789
```

适合：

```text
关键字位数较长
```

---

## 七、字符串散列

字符串不能直接作为数组下标，因此通常逐字符累积计算。

### 简单示例

```c
unsigned int hashString(const char *s, unsigned int tableSize) {
    unsigned int hash = 5381;

    while (*s != '\0') {
        hash = hash * 33 + (unsigned char)(*s);
        s++;
    }

    return hash % tableSize;
}
```

关键点：

- 字符串中的每个字符都会影响结果。
- 最终使用 `% tableSize` 将结果限制在合法范围内。
- 实际工程中还要考虑溢出、攻击性输入和具体用途。

## 八、散列函数与冲突处理的关系

即使散列函数设计得很好，也不能假设冲突永远不会出现。

因此：

```text
好的散列函数
+
正确的冲突处理策略
=
可用的散列表
```

---

## 九、常见错误

1. 忘记对表长取模，导致数组越界。
2. 认为散列函数能够彻底消除冲突。
3. 模数选择不合理，造成严重聚集。
4. 字符串散列只使用一个字符，导致大量冲突。
5. 不了解业务数据分布，机械套用函数。
6. 混淆“散列函数计算位置”与“冲突处理继续探测”。

## 十一、核心结论

> [!important]
> 散列函数的任务不是保证绝对无冲突，而是快速、均匀地映射关键字。

必须掌握：

1. 常用方法包括直接定址法、除留余数法、数字分析法、平方取中法和折叠法。
2. 字符串通常需要逐字符累积计算。
3. 散列函数质量会影响冲突数量。
4. 冲突仍然需要单独处理。
