## 条目概览

顺序查找也叫线性查找。 它从表中的第一个元素开始，按照存储顺序逐个比较，直到找到目标元素，或者检查完全部元素后确认目标不存在。

> [!summary]
> 顺序查找也叫线性查找。
> 它从表中的第一个元素开始，按照存储顺序逐个比较，直到找到目标元素，或者检查完全部元素后确认目标不存在。

## 一、为什么需要顺序查找？

现实中的数据不一定已经排好顺序，也不一定存放在支持随机访问的结构中。

例如，下面是一个乱序数组：

```text
下标：  0   1   2   3   4
数据： 42  18  75  26  91
```

现在需要查找：

```text
target = 26
```

由于数组没有排序，我们无法根据大小关系直接排除某一部分数据。

最自然的方法就是：

```text
42 → 18 → 75 → 26
```

找到 `26` 后停止。

这就是顺序查找。

### 核心动机

> 当我们无法利用额外规律缩小查找范围时，就从头到尾逐个检查。

顺序查找虽然简单，但它非常重要，因为很多更复杂的算法都建立在“遍历”和“逐个检查”的基础之上。

---

## 二、什么是顺序查找？

顺序查找是指：

> 从查找表的一端开始，依次将每个元素的关键字与目标值进行比较。
> 如果相等，则查找成功；如果遍历完整张表后仍未找到，则查找失败。

例如：

```text
下标：  0   1   2   3   4
数据： 17  25  31  46  58
```

查找：

```text
target = 31
```

执行过程：

```text
17 == 31 ? 否
25 == 31 ? 否
31 == 31 ? 是
```

返回：

```text
下标 2
```

## 三、顺序查找的基本流程

### 1. 数组中的流程

```text
从 arr[0] 开始
    ↓
判断 arr[i] 是否等于 target
    ↓
相等：返回当前位置 i
不相等：继续检查下一个元素
    ↓
所有元素都检查完仍未找到
    ↓
返回查找失败标志
```

### 2. 伪代码

```text
for 每一个元素:
    if 当前元素 == 目标:
        返回当前位置

返回查找失败
```

## 四、数组版本代码

### 1. 基础版本：返回第一个匹配元素的下标

```c
int sequentialSearch(const int arr[], int n, int target) {
    for (int i = 0; i < n; i++) {
        if (arr[i] == target) {
            return i;
        }
    }

    return -1;
}
```

### 2. 调用示例

```c
#include <stdio.h>

int sequentialSearch(const int arr[], int n, int target) {
    for (int i = 0; i < n; i++) {
        if (arr[i] == target) {
            return i;
        }
    }

    return -1;
}

int main(void) {
    int arr[] = {42, 18, 75, 26, 91};
    int n = sizeof(arr) / sizeof(arr[0]);

    int pos = sequentialSearch(arr, n, 26);

    if (pos == -1) {
        printf("没有找到目标元素\n");
    } else {
        printf("目标元素的下标是：%d\n", pos);
    }

    return 0;
}
```

输出：

```text
目标元素的下标是：3
```

## 五、逐行理解数组版本代码

```c
int sequentialSearch(const int arr[], int n, int target)
```

### 1. 参数含义

| 参数 | 含义 |
|---|---|
| `arr[]` | 保存数据的数组 |
| `n` | 数组中的有效元素数量 |
| `target` | 需要查找的目标值 |

这里使用：

```c
const int arr[]
```

表示函数只读取数组，不修改数组内容。

---

### 2. 为什么循环条件是 `i < n`？

```c
for (int i = 0; i < n; i++)
```

如果数组长度为 `n`，合法下标是：

```text
0, 1, 2, ..., n - 1
```

例如：

```text
数组长度：5
合法下标：0  1  2  3  4
```

如果错误地写成：

```c
i <= n
```

最后一次循环会访问：

```c
arr[n]
```

但是 `arr[n]` 已经越界。

---

### 3. 为什么找到后直接 `return i`？

```c
if (arr[i] == target) {
    return i;
}
```

因为 `i` 就是目标元素在数组中的存储位置。

例如：

```text
下标：  0   1   2
数据： 18  42  75
```

查找 `75` 时：

```c
arr[2] == 75
```

所以返回：

```c
return 2;
```

执行 `return` 后，函数立即结束，不会继续遍历后续元素。

---

### 4. 为什么查找失败返回 `-1`？

```c
return -1;
```

数组下标只能是：

```text
0, 1, 2, ...
```

`-1` 不是合法下标，因此可以作为特殊标志，表示：

```text
没有找到目标元素
```

不能用 `0` 表示失败，因为 `0` 本身是合法下标。

---

## 六、查找成功与查找失败

### 1. 查找成功

数组：

```text
42  18  75  26  91
```

查找：

```text
target = 26
```

比较过程：

```text
42 → 18 → 75 → 26
```

比较 `4` 次，返回下标 `3`。

---

### 2. 查找失败

数组：

```text
42  18  75  26  91
```

查找：

```text
target = 60
```

比较过程：

```text
42 → 18 → 75 → 26 → 91
```

全部比较完后仍然没有找到，因此返回：

```text
-1
```

### 3. 为什么失败时必须检查全部元素？

因为数组可能是乱序的。

即使前面的元素都不等于目标，也不能保证目标不会出现在最后一个位置。

## 七、重复元素应该怎么处理？

数组：

```text
下标：  0   1   2   3   4
数据： 18  42  18  75  18
```

查找：

```text
target = 18
```

具体处理方式取决于业务需求。

| 需求 | 是否可以在第一次匹配后停止？ |
|---|---|
| 判断目标是否存在 | 可以 |
| 返回第一个匹配下标 | 可以 |
| 返回任意一个匹配下标 | 可以 |
| 统计出现次数 | 不可以 |
| 返回全部匹配位置 | 不可以 |
| 删除全部匹配元素 | 不可以 |

### 1. 返回第一个匹配位置

```c
int findFirst(const int arr[], int n, int target) {
    for (int i = 0; i < n; i++) {
        if (arr[i] == target) {
            return i;
        }
    }

    return -1;
}
```

---

### 2. 统计目标出现次数

```c
int countOccurrences(const int arr[], int n, int target) {
    int count = 0;

    for (int i = 0; i < n; i++) {
        if (arr[i] == target) {
            count++;
        }
    }

    return count;
}
```

示例：

```c
int arr[] = {18, 42, 18, 75, 18};
printf("%d\n", countOccurrences(arr, 5, 18));
```

输出：

```text
3
```

---

### 3. 返回最后一个匹配位置

```c
int findLast(const int arr[], int n, int target) {
    int result = -1;

    for (int i = 0; i < n; i++) {
        if (arr[i] == target) {
            result = i;
        }
    }

    return result;
}
```

这里不能在第一次找到后立即返回，因为还需要继续检查后面是否存在新的匹配项。

---

### 4. 输出所有匹配位置

```c
#include <stdio.h>

void printAllPositions(const int arr[], int n, int target) {
    int found = 0;

    for (int i = 0; i < n; i++) {
        if (arr[i] == target) {
            printf("%d ", i);
            found = 1;
        }
    }

    if (!found) {
        printf("没有找到");
    }

    printf("\n");
}
```

## 八、按结构体中的关键字查找

顺序查找不只能查找整数，也可以查找结构体记录。

### 1. 学生信息结构体

```c
#include <stdio.h>

typedef struct {
    int id;
    char name[20];
    int score;
} Student;
```

### 2. 按学号查找学生

```c
Student *findStudentById(Student students[], int n, int targetId) {
    for (int i = 0; i < n; i++) {
        if (students[i].id == targetId) {
            return &students[i];
        }
    }

    return NULL;
}
```

### 3. 调用示例

```c
#include <stdio.h>

typedef struct {
    int id;
    char name[20];
    int score;
} Student;

Student *findStudentById(Student students[], int n, int targetId) {
    for (int i = 0; i < n; i++) {
        if (students[i].id == targetId) {
            return &students[i];
        }
    }

    return NULL;
}

int main(void) {
    Student students[] = {
        {1001, "小王", 86},
        {1002, "小李", 72},
        {1003, "小张", 93}
    };

    int n = sizeof(students) / sizeof(students[0]);

    Student *result = findStudentById(students, n, 1002);

    if (result == NULL) {
        printf("没有找到该学生\n");
    } else {
        printf("学号：%d\n", result->id);
        printf("姓名：%s\n", result->name);
        printf("成绩：%d\n", result->score);
    }

    return 0;
}
```

### 4. 为什么这里返回指针？

找到学生后，我们希望获得完整记录，而不是只获得数组下标。

```c
return &students[i];
```

返回的是该学生记录的地址。

找不到时返回：

```c
return NULL;
```

## 九、链表中的顺序查找

链表中的节点通常不是连续存储的，因此不能像数组一样使用：

```c
arr[i]
```

程序必须沿着指针逐个访问节点。

### 1. 链表节点

```c
typedef struct Node {
    int data;
    struct Node *next;
} Node;
```

### 2. 链表版本代码

```c
Node *sequentialSearch(Node *head, int target) {
    Node *p = head;

    while (p != NULL) {
        if (p->data == target) {
            return p;
        }

        p = p->next;
    }

    return NULL;
}
```

### 3. 执行过程

链表：

```text
head
 ↓
[42 | next] → [18 | next] → [75 | next] → NULL
```

查找：

```text
target = 75
```

执行过程：

```text
检查 42
    ↓
p = p->next
    ↓
检查 18
    ↓
p = p->next
    ↓
检查 75
    ↓
找到，返回当前节点地址
```

---

## 十、数组与链表中的顺序查找对比

| 对比项 | 数组 | 链表 |
|---|---|---|
| 存储方式 | 通常连续存储 | 节点通常分散存储 |
| 遍历方式 | 使用下标 `i++` | 使用 `p = p->next` |
| 成功返回值 | 常见为整数下标 | 常见为节点指针 |
| 失败返回值 | 常见为 `-1` | 常见为 `NULL` |
| 最坏时间复杂度 | $O(n)$ | $O(n)$ |

虽然数组和链表的访问方式不同，但顺序查找的核心逻辑相同：

```text
检查当前元素
    ↓
没有找到
    ↓
移动到下一个元素
    ↓
继续检查
```

## 十一、边缘情况

### 1. 空数组

当：

```c
n = 0;
```

循环条件：

```c
i < n
```

第一次判断就是：

```text
0 < 0 → 不成立
```

循环体执行 `0` 次，直接返回：

```c
-1
```

---

### 2. 空链表

当：

```c
Node *head = NULL;
```

执行：

```c
Node *p = head;
```

此时：

```text
p == NULL
```

循环条件：

```c
while (p != NULL)
```

一开始就不成立，因此循环体执行 `0` 次，直接返回：

```c
NULL
```

---

### 3. 为什么空链表不会发生空指针访问错误？

因为程序会先判断：

```c
p != NULL
```

只有条件成立时，才会执行：

```c
p->data
p->next
```

如果错误地直接访问：

```c
p->data
```

而此时：

```c
p == NULL
```

程序可能崩溃。

---

### 4. 目标在第一个位置

```text
数组：18  42  75
目标：18
```

只需比较一次：

```text
arr[0] == target
```

时间复杂度为：

```text
O(1)
```

---

### 5. 目标在最后一个位置

```text
数组：18  42  75
目标：75
```

必须比较全部元素：

```text
18 → 42 → 75
```

比较次数为：

```text
n
```

---

### 6. 数组中有重复元素

需要先明确业务需求：

```text
找第一个？
找最后一个？
统计数量？
找出全部位置？
```

不同需求决定是否可以提前停止。

## 十二、时间复杂度与空间复杂度

假设表中有 $n$ 个元素。

### 1. 最好情况

目标位于第一个位置：

```text
比较次数 = 1
```

时间复杂度：

$$
O(1)
$$

---

### 2. 最坏情况

目标位于最后一个位置，或者目标不存在：

```text
比较次数 = n
```

时间复杂度：

$$
O(n)
$$

---

### 3. 平均情况

如果每个位置被查找的概率相同，平均比较次数为：

$$
ASL_{成功}
=
\frac{1+2+\cdots+n}{n}
=
\frac{n+1}{2}
$$

忽略常数以后，仍然属于：

$$
O(n)
$$

---

### 4. 空间复杂度

基础顺序查找只使用少量额外变量：

```c
int i;
```

空间复杂度为：

$$
O(1)
$$

## 十三、平均查找长度 ASL

ASL 是 **Average Search Length** 的缩写，即：

> 平均查找长度。

它通常表示完成一次查找平均需要比较多少次。

---

### 1. 成功查找的 ASL

数组：

```text
1001  1002  1003  1004
```

如果每个元素被查找的概率相同：

| 目标元素 | 比较次数 |
|---|---:|
| `1001` | 1 |
| `1002` | 2 |
| `1003` | 3 |
| `1004` | 4 |

因此：

$$
ASL_{成功}
=
\frac{1+2+3+4}{4}
=
2.5
$$

对于长度为 $n$ 的顺序表：

$$
ASL_{成功}
=
\frac{n+1}{2}
$$

---

### 2. 失败查找的 ASL

顺序查找失败时，必须检查全部元素。

因此：

$$
ASL_{失败}=n
$$

---

### 3. 查找概率不同时怎么办？

如果每个元素被查找的概率不同，需要使用加权平均数。

例如：

| 联系人 | 比较次数 | 查询概率 |
|---|---:|---:|
| 小王 | 1 | 50% |
| 小李 | 2 | 30% |
| 小张 | 3 | 20% |

那么：

$$
ASL
=
1\times0.5
+
2\times0.3
+
3\times0.2
=
1.7
$$

一般公式为：

$$
ASL=\sum_{i=1}^{n}P_iC_i
$$

其中：

- $P_i$：查找第 $i$ 个元素的概率；
- $C_i$：找到第 $i$ 个元素需要比较的次数。

### 4. 一个重要结论

如果能够预估查询频率：

> 应当尽量将经常被查询的元素放在前面。

这样可以降低平均查找长度。

## 十四、顺序查找的优点与缺点

### 1. 优点

- 实现简单。
- 容易理解。
- 不要求数据提前排序。
- 乱序数组也能使用。
- 数组和链表都能使用。
- 数据量较小时非常实用。
- 可以灵活处理重复元素。
- 适合作为复杂算法的基础遍历框架。

### 2. 缺点

- 数据量较大时效率较低。
- 最坏情况下必须逐个检查全部元素。
- 不适合频繁查询大型静态数据集。
- 无法像折半查找一样，每次排除一半数据。

---

## 十五、什么时候适合使用顺序查找？

适合：

- 数据规模较小。
- 数据没有排序。
- 只需要查找一次或少量几次。
- 数据存储在链表中。
- 业务逻辑需要检查每个元素。
- 需要统计、筛选、删除全部匹配项。
- 尚未建立索引、散列表等更高效结构。

不适合：

- 数据规模非常大。
- 查询次数非常多。
- 数据已经有序，并且适合使用折半查找。
- 可以预先建立散列表、树或索引结构。

---

## 十六、顺序查找、折半查找与散列表的直观对比

| 方法 | 是否要求有序 | 基本思想 | 典型时间复杂度 |
|---|---|---|---:|
| 顺序查找 | 不要求 | 从头到尾逐个比较 | $O(n)$ |
| 折半查找 | 要求有序数组 | 每次排除一半范围 | $O(\log n)$ |
| 散列表查找 | 不要求有序 | 通过散列函数定位位置 | 平均 $O(1)$ |

顺序查找最简单，但并不意味着它没有价值。

> 在数据量较小、数据无序、结构简单或必须完整遍历时，顺序查找往往就是最合适的选择。

## 十七、常见错误

### 错误 1：循环条件写成 `i <= n`

错误：

```c
for (int i = 0; i <= n; i++)
```

问题：

```text
最后会访问 arr[n]，发生数组越界。
```

正确：

```c
for (int i = 0; i < n; i++)
```

---

### 错误 2：查找失败返回 `0`

错误：

```c
return 0;
```

问题：

```text
下标 0 本身是合法位置，无法区分“目标在第一个位置”和“没有找到”。
```

正确：

```c
return -1;
```

---

### 错误 3：统计全部匹配项时提前返回

错误：

```c
for (int i = 0; i < n; i++) {
    if (arr[i] == target) {
        return 1;
    }
}
```

这段代码只能判断目标是否存在，不能统计出现次数。

正确做法：

```c
int count = 0;

for (int i = 0; i < n; i++) {
    if (arr[i] == target) {
        count++;
    }
}

return count;
```

---

### 错误 4：空链表中直接访问 `p->data`

错误：

```c
Node *p = NULL;
printf("%d\n", p->data);
```

问题：

```text
通过空指针访问成员，程序可能崩溃。
```

正确做法：

```c
if (p != NULL) {
    printf("%d\n", p->data);
}
```

---

### 错误 5：误以为关键字就是数组下标

错误理解：

```text
学号 1002 就是数组下标 1002
```

正确理解：

## 十八、LeetCode 练习题

> [!note]
> 下面的题目不一定都只是“查找某一个元素”。
> 它们共同训练的是顺序查找最重要的基础能力：从头到尾遍历、逐个判断、维护状态、处理边缘情况。

### 第一组：基础遍历

#### 1. [LeetCode 485：最大连续 1 的个数](https://leetcode.cn/problems/max-consecutive-ones/)

难度：简单

题目核心：

```text
遍历数组，记录当前连续 1 的数量，并维护最大值。
```

训练目标：

- 学会逐个扫描数组。
- 学会在遍历过程中维护状态。
- 学会处理数组结尾仍然连续为 `1` 的情况。

推荐先自己写。

参考思路：

```c
int findMaxConsecutiveOnes(int *nums, int numsSize) {
    int current = 0;
    int maxCount = 0;

    for (int i = 0; i < numsSize; i++) {
        if (nums[i] == 1) {
            current++;

            if (current > maxCount) {
                maxCount = current;
            }
        } else {
            current = 0;
        }
    }

    return maxCount;
}
```

#### 2. [LeetCode 27：移除元素](https://leetcode.cn/problems/remove-element/)

难度：简单

题目核心：

```text
顺序扫描数组，将不等于 val 的元素保留下来。
```

训练目标：

- 理解为什么“删除全部匹配元素”必须完整遍历。
- 学会使用双指针。
- 理解原地修改数组。

参考思路：

```c
int removeElement(int *nums, int numsSize, int val) {
    int slow = 0;

    for (int fast = 0; fast < numsSize; fast++) {
        if (nums[fast] != val) {
            nums[slow] = nums[fast];
            slow++;
        }
    }

    return slow;
}
```

理解：

| 指针 | 作用 |
|---|---|
| `fast` | 依次检查每个元素 |
| `slow` | 指向下一个应当保留元素的位置 |

### 第二组：在顺序遍历上增加规则

#### 3. [LeetCode 26：删除有序数组中的重复项](https://leetcode.cn/problems/remove-duplicates-from-sorted-array/)

难度：简单

题目核心：

```text
数组已经有序，相同元素一定相邻。
顺序扫描数组，并把不同的元素保留下来。
```

训练目标：

- 学会利用“有序”这一附加条件。
- 理解同样是遍历，规则不同，代码也会不同。
- 练习快慢指针。

参考思路：

```c
int removeDuplicates(int *nums, int numsSize) {
    if (numsSize == 0) {
        return 0;
    }

    int slow = 1;

    for (int fast = 1; fast < numsSize; fast++) {
        if (nums[fast] != nums[fast - 1]) {
            nums[slow] = nums[fast];
            slow++;
        }
    }

    return slow;
}
```

#### 4. [LeetCode 1：两数之和](https://leetcode.cn/problems/two-sum/)

难度：简单

题目核心：

```text
在数组中找出两个元素，使它们的和等于 target。
```

最基础的暴力思路：

```text
固定第一个元素
    ↓
顺序查找后续元素中是否存在需要的值
```

暴力版本：

```c
#include <stdlib.h>

int *twoSum(int *nums, int numsSize, int target, int *returnSize) {
    int *result = malloc(2 * sizeof(int));

    if (result == NULL) {
        *returnSize = 0;
        return NULL;
    }

    for (int i = 0; i < numsSize; i++) {
        for (int j = i + 1; j < numsSize; j++) {
            if (nums[i] + nums[j] == target) {
                result[0] = i;
                result[1] = j;
                *returnSize = 2;
                return result;
            }
        }
    }

    free(result);
    *returnSize = 0;
    return NULL;
}
```

复杂度：

$$
O(n^2)
$$

训练目标：

- 理解嵌套顺序查找。
- 理解为什么嵌套循环会使复杂度上升。
- 为后续学习散列表优化做准备。

学完散列表后，可以把复杂度优化为平均：

$$
O(n)
$$

### 第三组：链表遍历

#### 5. [LeetCode 203：移除链表元素](https://leetcode.cn/problems/remove-linked-list-elements/)

难度：简单

题目核心：

```text
顺序遍历链表，删除所有值等于 val 的节点。
```

训练目标：

- 练习 `p = p->next`。
- 理解链表节点不是通过数组下标访问。
- 理解删除节点时为什么要维护前驱关系。
- 理解虚拟头节点的作用。

参考思路：

```c
#include <stdlib.h>

struct ListNode {
    int val;
    struct ListNode *next;
};

struct ListNode *removeElements(struct ListNode *head, int val) {
    struct ListNode dummy;
    dummy.next = head;

    struct ListNode *current = &dummy;

    while (current->next != NULL) {
        if (current->next->val == val) {
            struct ListNode *toDelete = current->next;
            current->next = toDelete->next;
            free(toDelete);
        } else {
            current = current->next;
        }
    }

    return dummy.next;
}
```

> [!warning]
> 删除节点后，不要立刻执行 `current = current->next`。
> 因为新的 `current->next` 也可能需要删除。

### 第四组：字符串中的顺序匹配

#### 6. [LeetCode 28：找出字符串中第一个匹配项的下标](https://leetcode.cn/problems/find-the-index-of-the-first-occurrence-in-a-string/)

难度：简单

题目核心：

```text
在字符串 haystack 中查找字符串 needle 第一次出现的位置。
```

最直观的朴素方法：

```text
从 haystack 的每一个可能起点出发
    ↓
逐个比较 needle 中的字符
```

朴素版本：

```c
int strStr(char *haystack, char *needle) {
    int n = 0;
    int m = 0;

    while (haystack[n] != '\0') {
        n++;
    }

    while (needle[m] != '\0') {
        m++;
    }

    if (m == 0) {
        return 0;
    }

    for (int i = 0; i + m <= n; i++) {
        int j = 0;

        while (j < m && haystack[i + j] == needle[j]) {
            j++;
        }

        if (j == m) {
            return i;
        }
    }

    return -1;
}
```

训练目标：

- 理解字符串本质上也是字符数组。
- 理解嵌套顺序匹配。
- 为后续学习 KMP 算法做准备。

## 二十一、核心结论

> [!important]
> 顺序查找的本质是：按照存储顺序逐个检查元素。

必须掌握：

1. 数组中使用 `i++` 逐个检查元素。
2. 链表中使用 `p = p->next` 逐个访问节点。
3. 找到目标后是否立即停止，取决于业务需求。
4. 查找失败时，通常必须遍历全部元素。
5. 顺序查找不要求数据有序。
6. 最坏时间复杂度为 $O(n)$。
7. 额外空间复杂度通常为 $O(1)$。
8. 数组查找失败通常返回 `-1`。
9. 链表查找失败通常返回 `NULL`。
10. 顺序查找虽然简单，但它是遍历、筛选、统计、双指针、字符串匹配和链表操作的基础。
