## 条目概览

分块查找也叫索引顺序查找。 它先通过索引表确定目标可能位于哪个块，再在块内进行顺序查找。

> [!summary]
> 分块查找也叫索引顺序查找。
> 它先通过索引表确定目标可能位于哪个块，再在块内进行顺序查找。

## 一、为什么需要分块查找？

顺序查找不要求数据有序，但速度较慢。

折半查找速度快，但通常要求整个数组严格有序。

现实中有时可以做到：

> 块与块之间有序，但每个块内部不必完全有序。

这时可以采用分块查找，在灵活性与效率之间取得折中。

---

## 二、基本思想

例如，将数据划分为三个块：

```text
块 1：  8  3  12
块 2： 18  25  16
块 3： 34  29  40
```

要求：

```text
块 1 中所有元素 < 块 2 中所有元素 < 块 3 中所有元素
```

块内可以无序。

建立索引表：

| 块号 | 块内最大关键字 | 起始位置 |
|---:|---:|---:|
| 1 | 12 | 0 |
| 2 | 25 | 3 |
| 3 | 40 | 6 |

查找 `18`：

1. 在索引表中判断 `18` 应位于块 2。
2. 在块 2 内顺序查找。
3. 找到 `18`。

---

## 三、为什么分块查找有效？

如果整张表有 $n$ 个元素，划分成多个块：

- 第一步只查索引表。
- 第二步只扫描目标块。
- 不需要扫描全部元素。

它本质上是：

```text
缩小范围
    ↓
局部顺序查找
```

---

## 四、数据需要满足什么条件？

### 1. 块间有序

前一个块中的最大关键字必须小于后一个块中的最小关键字。

### 2. 块内允许无序

块内部不需要逐项排序。

### 3. 需要额外维护索引表

索引表记录每个块的边界信息和位置。

## 五、代码示例

```c
typedef struct {
    int maxKey;
    int start;
    int length;
} BlockIndex;

int blockSearch(
    const int arr[],
    int n,
    const BlockIndex index[],
    int blockCount,
    int target
) {
    int block = -1;

    for (int i = 0; i < blockCount; i++) {
        if (target <= index[i].maxKey) {
            block = i;
            break;
        }
    }

    if (block == -1) {
        return -1;
    }

    int start = index[block].start;
    int end = start + index[block].length;

    for (int i = start; i < end && i < n; i++) {
        if (arr[i] == target) {
            return i;
        }
    }

    return -1;
}
```

## 六、ASL 的直观理解

分块查找的平均成本通常由两部分组成：

$$
ASL = ASL_{索引表} + ASL_{块内}
$$

如果索引表有 $b$ 个块，每块约有 $s$ 个元素，且使用顺序查找：

$$
ASL \approx \frac{b+1}{2} + \frac{s+1}{2}
$$

因为：

$$
n = bs
$$

直观上，当块数和块大小都接近 $\sqrt{n}$ 时，通常能取得较好的折中。

---

## 七、分块查找与其他方法对比

| 方法 | 数据要求 | 查询方式 |
|---|---|---|
| 顺序查找 | 无序也可以 | 扫描全部表 |
| 折半查找 | 整体有序数组 | 每次排除一半 |
| 分块查找 | 块间有序、块内可无序 | 先找块，再扫描块内 |

---

## 八、优点与缺点

### 优点

- 比纯顺序查找更快。
- 块内不要求完全有序。
- 插入新元素时，只要不破坏块间关系，调整压力小于严格有序数组。

### 缺点

- 需要额外维护索引表。
- 块划分不合理时，效率会下降。
- 一般不如理想条件下的折半查找快。

---

## 九、常见错误

1. 误以为块内也必须完全有序。
2. 忽略块间有序这一核心条件。
3. 查到索引块后忘记进行块内顺序查找。
4. 插入新元素后没有更新索引表中的最大关键字。
5. 块大小差距过大，导致某些块扫描成本过高。

## 十一、核心结论

> [!important]
> 分块查找的核心是：用索引先缩小范围，再在局部范围内顺序查找。

必须掌握：

1. 块间有序。
2. 块内可以无序。
3. 需要索引表。
4. 总成本由“查索引”和“查块内”两部分组成。
