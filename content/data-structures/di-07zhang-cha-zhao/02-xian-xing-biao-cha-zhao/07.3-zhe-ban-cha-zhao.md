## 条目概览

折半查找也叫二分查找。 它利用有序数组的大小关系，每次排除大约一半不可能包含目标的范围。

> [!summary]
> 折半查找也叫二分查找。
> 它利用有序数组的大小关系，每次排除大约一半不可能包含目标的范围。

## 一、为什么需要折半查找？

顺序查找最坏需要检查全部元素。

假设有一百万个有序数据，目标位于数组末尾：

```text
顺序查找：最多比较 1,000,000 次
```

如果每次比较后都能排除一半数据，比较次数会显著下降：

```text
1,000,000
→ 500,000
→ 250,000
→ 125,000
→ ...
```

这就是折半查找的核心价值。

---

## 二、前提条件

折半查找通常要求：

1. 数据按照关键字有序排列。
2. 数据存储在支持随机访问的结构中，最典型的是数组。

链表虽然也能找到“中间节点”，但无法像数组一样通过下标快速访问中间位置，因此通常不适合直接使用折半查找。

---

## 三、为什么数组必须有序？

有序数组：

```text
下标：  0   1   2   3   4   5   6
数据： 11  18  25  32  46  57  70
```

查找：

```text
target = 18
```

第一次比较：

```text
arr[3] = 32
18 < 32
```

因为数组有序，所以可以排除：

```text
32  46  57  70
```

只需要继续查找：

```text
11  18  25
```

如果数组乱序：

```text
11  57  25  32  18  46  70
```

即使：

```text
18 < 32
```

目标仍然可能出现在 `32` 右侧。

## 四、`low`、`high` 和 `mid`

| 变量 | 含义 |
|---|---|
| `low` | 当前查找范围最左侧下标 |
| `high` | 当前查找范围最右侧下标 |
| `mid` | 当前范围中间位置下标 |

推荐写法：

```c
mid = low + (high - low) / 2;
```

相比：

```c
mid = (low + high) / 2;
```

推荐写法可以降低 `low + high` 整数溢出的风险。

## 五、完整执行过程

数组：

```text
下标：  0   1   2   3   4   5   6
数据： 11  18  25  32  46  57  70
```

查找：

```text
target = 57
```

### 第一次

```text
low = 0
high = 6
mid = 3
arr[mid] = 32
```

因为：

```text
57 > 32
```

更新：

```text
low = mid + 1 = 4
```

### 第二次

```text
low = 4
high = 6
mid = 5
arr[mid] = 57
```

找到目标，返回下标：

```text
5
```

## 六、数组版本代码

```c
int binarySearch(const int arr[], int n, int target) {
    int low = 0;
    int high = n - 1;

    while (low <= high) {
        int mid = low + (high - low) / 2;

        if (arr[mid] == target) {
            return mid;
        } else if (arr[mid] < target) {
            low = mid + 1;
        } else {
            high = mid - 1;
        }
    }

    return -1;
}
```

---

## 七、为什么是 `mid + 1` 和 `mid - 1`？

当前已经比较过：

```c
arr[mid]
```

如果它不等于目标，就不应再次进入候选范围。

因此：

```c
low = mid + 1;
```

或：

```c
high = mid - 1;
```

不能错误地写成：

```c
low = mid;
high = mid;
```

否则某些情况下会反复检查同一个位置，导致死循环。

---

## 八、循环条件为什么是 `low <= high`？

当前实现采用闭区间：

```text
[low, high]
```

当：

```text
low == high
```

范围中仍然剩下一个元素，需要继续检查。

只有当：

```text
low > high
```

候选范围才真正为空。

---

## 九、重复元素：如何找左右边界？

基础版本遇到目标后会立即返回任意一个匹配位置。

如果数组中有重复元素：

```text
1  2  2  2  5
```

需要查找第一个 `2`，则找到 `2` 后仍要向左继续尝试。

```c
int findFirst(const int arr[], int n, int target) {
    int low = 0;
    int high = n - 1;
    int result = -1;

    while (low <= high) {
        int mid = low + (high - low) / 2;

        if (arr[mid] == target) {
            result = mid;
            high = mid - 1;
        } else if (arr[mid] < target) {
            low = mid + 1;
        } else {
            high = mid - 1;
        }
    }

    return result;
}
```

查找最后一个匹配项时，找到目标后继续向右：

```c
low = mid + 1;
```

## 十、复杂度

| 情况 | 时间复杂度 |
|---|---:|
| 最好情况：第一次命中中间元素 | $O(1)$ |
| 最坏情况 | $O(\log n)$ |
| 平均情况 | $O(\log n)$ |
| 额外空间复杂度：迭代版本 | $O(1)$ |

为什么是 $O(\log n)$？

因为每次都将候选范围缩小为原来的一半。

---

## 十一、边缘情况

### 1. 空数组

```c
n = 0;
high = -1;
```

此时：

```c
low <= high
```

一开始就不成立，直接返回 `-1`。

### 2. 只有一个元素

必须允许：

```text
low == high
```

否则会漏掉唯一元素。

### 3. 目标比最小值还小

查找范围不断向左缩小，最终：

```text
high < low
```

### 4. 目标比最大值还大

查找范围不断向右缩小，最终：

```text
low > high
```

---

## 十二、常见错误

1. 数组没有排序却使用折半查找。
2. 把 `mid` 当成元素值，而不是下标。
3. 使用 `low < high`，漏掉最后一个候选元素。
4. 更新边界时忘记 `+1` 或 `-1`。
5. 用链表硬套数组式折半查找。
6. 没有考虑重复元素的左右边界问题。

## 十五、核心结论

> [!important]
> 折半查找的本质不是“背模板”，而是利用有序性不断排除不可能范围。

必须掌握：

1. 前提：有序数组。
2. `low` 和 `high` 表示当前候选范围。
3. `mid` 是下标，`arr[mid]` 才是元素。
4. 每次比较后排除约一半候选项。
5. 迭代版本时间复杂度为 $O(\log n)$，空间复杂度为 $O(1)$。
6. 重复元素需要进一步处理左右边界。
