## 条目概览

AVL 树是一种自平衡二叉排序树。 它通过旋转操作限制树高，避免普通 BST 退化成链表。

> [!summary]
> AVL 树是一种自平衡二叉排序树。
> 它通过旋转操作限制树高，避免普通 BST 退化成链表。

## 一、为什么需要 AVL 树？

普通 BST 的性能取决于树高。

如果按顺序插入：

```text
1  2  3  4  5
```

普通 BST 可能退化为：

```text
1
 \
  2
   \
    3
     \
      4
       \
        5
```

查找复杂度从理想的：

$$
O(\log n)
$$

退化为：

$$
O(n)
$$

AVL 树通过主动调整结构，尽量避免这种退化。

---

## 二、平衡因子

节点的平衡因子定义为：

$$
BF = 左子树高度 - 右子树高度
$$

AVL 树要求任意节点的平衡因子只能是：

```text
-1、0、1
```

如果绝对值超过 `1`，说明该节点失衡，需要旋转。

---

## 三、四种失衡情况

### 1. LL 型：右旋

新节点插入到失衡节点的左孩子的左侧。

```text
      30
     /
   20
   /
 10
```

调整后：

```text
    20
   /  \
 10   30
```

### 2. RR 型：左旋

新节点插入到失衡节点的右孩子的右侧。

```text
10
  \
   20
     \
      30
```

调整后：

```text
    20
   /  \
 10   30
```

### 3. LR 型：先左旋，再右旋

新节点插入到左孩子的右侧。

```text
    30
   /
 10
   \
    20
```

### 4. RL 型：先右旋，再左旋

新节点插入到右孩子的左侧。

```text
10
  \
   30
   /
 20
```

---

## 四、旋转到底在做什么？

旋转的目标不是改变关键字顺序，而是：

> 在保持 BST 中序遍历结果不变的前提下，降低树高。

旋转前后，中序遍历仍然是：

```text
10  20  30
```

因此 BST 性质不会被破坏。

## 五、右旋代码骨架

```c
typedef struct AVLNode {
    int key;
    int height;
    struct AVLNode *left;
    struct AVLNode *right;
} AVLNode;

int max(int a, int b) {
    return a > b ? a : b;
}

int height(AVLNode *node) {
    return node == NULL ? 0 : node->height;
}

AVLNode *rightRotate(AVLNode *y) {
    AVLNode *x = y->left;
    AVLNode *t2 = x->right;

    x->right = y;
    y->left = t2;

    y->height = max(height(y->left), height(y->right)) + 1;
    x->height = max(height(x->left), height(x->right)) + 1;

    return x;
}
```

学习初期重点不是背完整 AVL 插入代码，而是先判断：

```text
为什么失衡？
属于哪种类型？
应该怎样旋转？
```

## 六、复杂度

AVL 树通过维持平衡，使树高保持在：

$$
O(\log n)
$$

因此：

| 操作 | 时间复杂度 |
|---|---:|
| 查找 | $O(\log n)$ |
| 插入 | $O(\log n)$ |
| 删除 | $O(\log n)$ |

插入和删除可能额外触发旋转，但总体仍是对数级。

---

## 七、AVL 与普通 BST 对比

| 对比项 | 普通 BST | AVL |
|---|---|---|
| 是否主动维持平衡 | 否 | 是 |
| 最坏查找复杂度 | $O(n)$ | $O(\log n)$ |
| 插入实现难度 | 较低 | 较高 |
| 删除实现难度 | 较低 | 较高 |
| 是否需要维护高度 | 不一定 | 需要 |
| 是否需要旋转 | 不需要 | 需要 |

## 八、常见错误

1. 误以为任意二叉排序树天然平衡。
2. 平衡因子左右方向写反。
3. 只记旋转名称，不会根据插入路径判断类型。
4. 旋转后忘记更新节点高度。
5. 旋转时破坏 BST 的中序顺序。
6. 认为 AVL 的插入一定比普通 BST 简单。

2. 平衡因子如何计算？
3. 哪些平衡因子是合法的？
4. LL 型失衡应该怎样旋转？
5. LR 型为什么需要两次旋转？
6. 旋转前后中序遍历结果会改变吗？

---

## 十、核心结论

> [!important]
> AVL 树通过旋转控制树高，以更复杂的维护成本换取稳定的 $O(\log n)$ 查找性能。

必须掌握：

1. AVL 首先是一棵 BST。
2. 任意节点平衡因子只能为 `-1、0、1`。
3. 失衡类型包括 LL、RR、LR、RL。
4. 旋转不会破坏中序有序性。
5. 查找、插入和删除均可保持 $O(\log n)$。
