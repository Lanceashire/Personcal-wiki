## 条目概览

二叉排序树也叫二叉搜索树，简称 BST。 对任意节点，左子树关键字更小，右子树关键字更大。查找过程类似在树上进行折半决策。

> [!summary]
> 二叉排序树也叫二叉搜索树，简称 BST。
> 对任意节点，左子树关键字更小，右子树关键字更大。查找过程类似在树上进行折半决策。

## 一、为什么需要二叉排序树？

折半查找速度快，但有一个限制：

> 它通常依赖有序数组。

如果频繁插入新元素，为了保持数组有序，可能需要移动大量元素。

二叉排序树希望同时兼顾：

- 查找
- 插入
- 删除

它通过树结构维持大小关系，不要求元素连续存放。

---

## 二、定义

对于任意节点：

1. 左子树中的所有关键字小于该节点关键字。
2. 右子树中的所有关键字大于该节点关键字。
3. 左右子树本身也分别是二叉排序树。

示例：

```text
        50
       /  \
     30    70
    / \    / \
   20 40  60 80
```

---

## 三、查找过程

查找 `60`：

```text
60 > 50
→ 去右子树

60 < 70
→ 去左子树

60 == 60
→ 找到
```

核心逻辑：

```text
target == 当前节点：成功
target < 当前节点：进入左子树
target > 当前节点：进入右子树
```

## 四、节点定义

```c
typedef struct BSTNode {
    int key;
    struct BSTNode *left;
    struct BSTNode *right;
} BSTNode;
```

---

## 五、查找代码

### 1. 递归版本

```c
BSTNode *searchBST(BSTNode *root, int target) {
    if (root == NULL || root->key == target) {
        return root;
    }

    if (target < root->key) {
        return searchBST(root->left, target);
    }

    return searchBST(root->right, target);
}
```

### 2. 迭代版本

```c
BSTNode *searchBST(BSTNode *root, int target) {
    BSTNode *p = root;

    while (p != NULL) {
        if (target == p->key) {
            return p;
        }

        if (target < p->key) {
            p = p->left;
        } else {
            p = p->right;
        }
    }

    return NULL;
}
```

## 六、插入操作

插入新关键字时，从根节点开始比较，直到找到空位置。

```c
#include <stdlib.h>

BSTNode *insertBST(BSTNode *root, int key) {
    if (root == NULL) {
        BSTNode *node = malloc(sizeof(BSTNode));

        if (node == NULL) {
            return NULL;
        }

        node->key = key;
        node->left = NULL;
        node->right = NULL;

        return node;
    }

    if (key < root->key) {
        root->left = insertBST(root->left, key);
    } else if (key > root->key) {
        root->right = insertBST(root->right, key);
    }

    return root;
}
```

这里对重复关键字采取：

```text
不重复插入
```

实际业务中也可以采用计数器等方式记录重复值。

## 七、删除操作的三种情况

### 1. 删除叶子节点

```text
没有子树
```

直接删除即可。

### 2. 删除只有一个孩子的节点

```text
让父节点直接连接到该节点唯一的孩子
```

### 3. 删除有两个孩子的节点

常见方法：

- 用中序前驱替代当前节点；
- 或用中序后继替代当前节点。

---

## 八、中序遍历为什么重要？

对二叉排序树进行中序遍历：

```text
左子树 → 根节点 → 右子树
```

会得到递增序列。

示例：

```text
20  30  40  50  60  70  80
```

因此，中序遍历也可以用来验证一棵树是否满足 BST 性质。

## 九、复杂度为什么取决于树高？

每一步只会进入一棵子树。

因此，查找成本取决于树高 $h$：

$$
O(h)
$$

### 较平衡的情况

```text
        4
       / \
      2   6
```

树高约为：

$$
O(\log n)
$$

### 极端退化情况

按顺序插入：

```text
1  2  3  4  5
```

可能形成：

```text
1
 \
  2
   \
    3
     \
      4
       \
        5
```

此时树退化成链表，查找复杂度变为：

$$
O(n)
$$

## 十、常见错误

1. 只比较当前节点，不根据大小进入正确子树。
2. 误以为 BST 一定是平衡树。
3. 忘记处理空树。
4. 删除有两个孩子的节点时，直接丢失子树。
5. 对重复关键字没有明确策略。
6. 把“二叉树”与“二叉排序树”混为一谈。

## 十三、核心结论

> [!important]
> BST 通过树结构维护大小关系，支持动态查找、插入和删除。

必须掌握：

1. 左子树关键字更小，右子树关键字更大。
2. 查找、插入和删除都依赖大小关系。
3. 中序遍历得到有序序列。
4. 性能取决于树高。
5. 普通 BST 最坏情况下可能退化成链表。
