## 条目概览

合并两个有序单链表时，可以直接摘取原链表结点并重新连接，不必复制每个数据元素。每个结点只处理一次，时间复杂度为 $O(m+n)$。

## 1. 问题定义

输入：

```text
La: headA → 1 → 3 → 7 → NULL
Lb: headB → 2 → 3 → 6 → 8 → NULL
```

输出：

```text
Lc: headC → 1 → 2 → 3 → 3 → 6 → 7 → 8 → NULL
```

默认：

- 两表均按非递减顺序排列；
- 保留重复值；
- 可以复用原结点；
- 输入表带头结点。

## 2. 三个工作指针

```c
Node *pa = headA->next;
Node *pb = headB->next;
Node *tail = headA;
```

含义：

| 指针 | 作用 |
|---|---|
| `pa` | 指向 `La` 当前未处理结点 |
| `pb` | 指向 `Lb` 当前未处理结点 |
| `tail` | 指向结果链表当前尾结点 |

复用 `headA` 作为结果头结点，可以减少一次申请。

## 3. 核心过程

若：

```text
pa->data <= pb->data
```

则把 `pa` 接到结果尾部：

```c
tail->next = pa;
pa = pa->next;
tail = tail->next;
```

否则接入 `pb`。

每次必须先推进被选中的输入指针，再或通过已接入结点继续维护尾部，避免丢失剩余链条。

## 4. 完整原地合并代码

```c
Node *mergeSortedLists(
    Node *headA,
    Node *headB
) {
    if (headA == NULL) {
        return headB;
    }

    if (headB == NULL) {
        return headA;
    }

    Node *pa = headA->next;
    Node *pb = headB->next;
    Node *tail = headA;

    while (pa != NULL && pb != NULL) {
        if (pa->data <= pb->data) {
            tail->next = pa;
            pa = pa->next;
        } else {
            tail->next = pb;
            pb = pb->next;
        }

        tail = tail->next;
    }

    tail->next = (pa != NULL) ? pa : pb;

    free(headB);
    return headA;
}
```

合并完成后：

- `headA` 成为结果链表头结点；
- `headB` 的辅助头结点被释放；
- 原 `La`、`Lb` 的数据结点已经归入结果链表；
- 调用者不应再把原 `headB` 当作有效链表使用。

## 5. 图解一轮连接

当前：

```text
结果：headA → 1
              ↑
             tail

pa → 3 → 7
pb → 2 → 3 → 6 → 8
```

因为 `2 < 3`：

```c
tail->next = pb;
pb = pb->next;
tail = tail->next;
```

结果：

```text
headA → 1 → 2
              ↑
             tail

pa → 3 → 7
pb → 3 → 6 → 8
```

## 6. 为什么剩余链可以整体接上

当 `pa == NULL` 时，`pb` 指向的剩余部分本身已经有序，并且其中所有值都不小于结果尾部，因此：

```c
tail->next = pb;
```

即可，无需逐结点继续处理。

## 7. 复杂度

设两表长度分别为 $m$ 和 $n$。

最坏情况下，每个结点都参与一次比较或连接：

$$
T(m,n)=O(m+n)
$$

只使用固定数量指针，且复用原结点：

$$
S(m,n)=O(1)
$$

这里的 $O(1)$ 指辅助空间，不包含原链表结点。

## 8. 新建结点版本

若题目要求保留两个输入链表不变，就不能摘取原结点，需要为结果复制数据。

这种版本：

- 时间仍为 $O(m+n)$；
- 新结点空间为 $O(m+n)$；
- 必须处理任意一次 `malloc` 失败并清理已建结果。

因此做题前要先看清：

```text
是否允许破坏或复用原链表
```

## 9. 去重合并

若要求相等值只保留一个，相等时可以选一个结点接入结果，并释放或跳过另一个结点。

但必须进一步说明：

- 输入链表内部是否也可能有重复；
- 被跳过的原结点是否需要释放；
- 是否允许修改原链表。

去重算法的内存所有权比保留重复版本更容易出错。

## 10. 常见错误

1. 没有保存或推进输入指针，导致后续链条丢失；
2. `tail` 接入结点后忘记后移；
3. 忘记连接未处理的剩余链；
4. 复用原结点后仍把原链表分别销毁，造成重复释放；
5. 忘记释放不再使用的第二个头结点；
6. 输入链表无序却直接套用算法；
7. 未明确相等关键字是否保留两个。

## 11. 与顺序表合并比较

| 比较项 | 有序顺序表 | 有序单链表 |
|---|---|---|
| 时间 | $O(m+n)$ | $O(m+n)$ |
| 结果存储 | 通常需要新数组区域 | 可直接重连原结点 |
| 辅助空间 | 新结果通常 $O(m+n)$ | 原地重连为 $O(1)$ |
| 核心动作 | 复制元素 | 修改指针 |
| 随机访问 | 支持 | 不支持 |
