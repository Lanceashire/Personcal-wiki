## 条目概览

单链表操作的难点不是语句多，而是必须始终保住尚未连接或释放的链条。插入遵循“新结点先接后面”，删除遵循“先保存、再绕过、后释放”。

> [!summary]
> 单链表操作的难点不是语句多，而是必须始终保住尚未连接或释放的链条。插入遵循“新结点先接后面”，删除遵循“先保存、再绕过、后释放”。

本文默认使用带头结点的单链表，并用 `index` 表示从 `0` 开始的数据元素下标。

## 1. 按位置查找数据结点

```text
数据下标：             0       1       2
head → [头结点] → [10] → [20] → [30] → NULL
```

要取得下标 `index` 的数据结点，从首元结点开始移动 `index` 次。

```c
Node *getNode(Node *head, int index) {
    if (head == NULL || index < 0) {
        return NULL;
    }

    Node *current = head->next;

    for (int i = 0; i < index; i++) {
        if (current == NULL) {
            return NULL;
        }

        current = current->next;
    }

    return current;
}
```

若 `index` 超出范围，最终返回 `NULL`。

时间复杂度：

$$
O(n)
$$

## 2. 按值查找

```c
Node *locateNode(
    Node *head,
    ElemType target
) {
    if (head == NULL) {
        return NULL;
    }

    Node *current = head->next;

    while (current != NULL) {
        if (current->data == target) {
            return current;
        }

        current = current->next;
    }

    return NULL;
}
```

最好情况为 $O(1)$，最坏和平均情况为 $O(n)$。

## 3. 已知前驱后的插入

原链表：

```text
p → q → r
```

插入 `s`：

```text
p → s → q → r
```

必须先让 `s` 接住原后继 `q`：

```c
s->next = p->next;
p->next = s;
```

执行过程：

```text
第一步：
p → q → r
    ↑
    s
    └────→ q

第二步：
p → s → q → r
```

## 4. 为什么两条语句不能反写

错误写法：

```c
p->next = s;
s->next = p->next;
```

第一句后：

```text
p → s
```

原来 `p->next` 中保存的 `q` 地址已经被覆盖。

第二句读取的 `p->next` 已经是 `s`：

```text
s->next = s;
```

结果：

```text
p → s ↻

q → r 链条失去入口
```

> [!warning] 对话中暴露的易错点
> 指针赋值不是同时发生的。每条语句都会立即改变后续语句读取到的值，因此必须先保存旧链接，再修改前驱。

## 5. 在指定下标插入

插入到数据下标 `index`，需要找到它的前驱：

| 插入位置 | 前驱 |
|---|---|
| `index = 0` | 头结点 |
| `index = 1` | 原下标 0 的数据结点 |
| `index = n` | 原尾结点 |

辅助函数：

```c
Node *getPrevious(Node *head, int index) {
    if (head == NULL || index < 0) {
        return NULL;
    }

    Node *previous = head;

    for (int i = 0; i < index; i++) {
        if (previous->next == NULL) {
            return NULL;
        }

        previous = previous->next;
    }

    return previous;
}
```

完整插入：

```c
int insertNode(
    Node *head,
    int index,
    ElemType value
) {
    Node *previous = getPrevious(head, index);

    if (previous == NULL) {
        return 0;
    }

    Node *newNode = malloc(sizeof(Node));

    if (newNode == NULL) {
        return 0;
    }

    newNode->data = value;
    newNode->next = previous->next;
    previous->next = newNode;

    return 1;
}
```

寻找前驱为 $O(n)$，修改指针为 $O(1)$，整体：

$$
O(n)
$$

若前驱指针已经给出，则插入本身是：

$$
O(1)
$$

## 6. 已知前驱后的删除

原链表：

```text
p → q → r
```

删除 `q`：

```c
Node *target = p->next;
p->next = target->next;
free(target);
```

过程：

```text
1. target 保存 q 的地址
2. p 绕过 q，直接指向 r
3. 释放 q
```

记忆：

```text
先抓住 → 再绕过 → 最后释放
```

## 7. 为什么必须保存待删除结点

只写：

```c
p->next = p->next->next;
```

虽然逻辑链条变成：

```text
p → r
```

但 `q` 的地址可能已经没有任何变量保存：

```text
q 仍占用堆内存，却无法再传给 free
```

这造成内存泄漏。

## 8. 删除的边界检查

安全的“删除后继”函数：

```c
int deleteAfter(
    Node *previous,
    ElemType *removed
) {
    if (
        previous == NULL ||
        previous->next == NULL
    ) {
        return 0;
    }

    Node *target = previous->next;

    if (removed != NULL) {
        *removed = target->data;
    }

    previous->next = target->next;
    free(target);
    return 1;
}
```

必须检查：

```c
previous == NULL
previous->next == NULL
```

否则可能访问空指针。

## 9. 按指定下标删除

```c
int deleteNode(
    Node *head,
    int index,
    ElemType *removed
) {
    if (head == NULL || index < 0) {
        return 0;
    }

    Node *previous = head;

    for (int i = 0; i < index; i++) {
        if (previous->next == NULL) {
            return 0;
        }

        previous = previous->next;
    }

    return deleteAfter(previous, removed);
}
```

删除下标 `0` 时，`previous` 保持为头结点，因此无需特殊修改 `head`。

## 10. 结点前插的特殊技巧

若只给出目标结点 `p`，而没有它的前驱，普通单链表不能直接向前移动。

在 `p` 不是尾结点时，可以：

1. 新建结点 `s`，复制 `p` 原数据；
2. `s` 插入到 `p` 后；
3. 把待插入值写入 `p`。

```c
int insertBeforeByCopy(
    Node *p,
    ElemType value
) {
    if (p == NULL) {
        return 0;
    }

    Node *newNode = malloc(sizeof(Node));

    if (newNode == NULL) {
        return 0;
    }

    newNode->data = p->data;
    newNode->next = p->next;
    p->data = value;
    p->next = newNode;
    return 1;
}
```

这种方法改变的是数据归属，若结点身份本身有外部含义，不能随意使用。

## 11. 删除给定结点的特殊技巧

若只给出非尾结点 `p`，可以复制后继数据并删除后继：

```c
int deleteGivenNode(Node *p) {
    if (p == NULL || p->next == NULL) {
        return 0;
    }

    Node *target = p->next;
    p->data = target->data;
    p->next = target->next;
    free(target);
    return 1;
}
```

它不适用于尾结点，因为尾结点没有后继可复制。

## 12. 复杂度总结

| 操作 | 条件 | 时间复杂度 |
|---|---|---:|
| 按位置访问 | 从头遍历 | $O(n)$ |
| 按值查找 | 从头遍历 | $O(n)$ |
| 已知前驱插入 | 直接改指针 | $O(1)$ |
| 按位置插入 | 先找前驱 | $O(n)$ |
| 已知前驱删除后继 | 直接改指针 | $O(1)$ |
| 按位置删除 | 先找前驱 | $O(n)$ |
