## 条目概览

单链表用结点保存数据，每个结点通过 next 指向直接后继。结点在内存中可以分散，逻辑次序由指针决定。

> [!summary]
> 单链表用结点保存数据，每个结点通过 `next` 指向直接后继。结点在内存中可以分散，逻辑次序由指针决定。

## 1. 结点结构

```c
typedef int ElemType;

typedef struct Node {
    ElemType data;
    struct Node *next;
} Node;
```

一个结点包含：

```text
[ data | next ]
```

| 字段 | 含义 |
|---|---|
| `data` | 数据域，保存当前元素 |
| `next` | 指针域，保存直接后继结点的地址 |

> [!warning] 对话中暴露的易错点
> `next` 是当前结点的后继指针域，不是“尾指针”。尾指针通常是额外变量 `tail`，用于指向整张链表的最后一个结点。

## 2. 物理地址与逻辑次序

三个结点可能存放在：

```text
地址 1000：[10 | 5200]
地址 5200：[20 | 2700]
地址 2700：[30 | NULL]
```

尽管地址不连续，沿 `next` 仍得到：

```text
10 → 20 → 30
```

链表的逻辑次序取决于指针内容，而不是结点地址大小。

## 3. 为什么不能用 `p++` 遍历

数组指针加一表示移动到相邻数组元素，因为数组元素连续。

```c
p++;
```

链表结点不保证连续，`p + 1` 只是按结点大小计算相邻地址，那里未必是逻辑后继，甚至可能不是合法对象。

链表必须读取当前结点中的后继地址：

```c
p = p->next;
```

## 4. 不带头结点的单链表

```text
head
 ↓
[10] → [20] → [30] → NULL
```

`head` 直接指向首元结点。

空表：

```c
head == NULL
```

优点是少一个辅助结点；缺点是表头插入和删除通常需要单独修改 `head`。

## 5. 带头结点的单链表

```text
head
 ↓
[头结点] → [10] → [20] → [30] → NULL
```

头结点不代表有效数据元素，主要用于统一边界操作。

带头结点空表：

```text
head → [头结点] → NULL
```

判断：

```c
head->next == NULL
```

## 6. 四个容易混淆的概念

| 名称 | 含义 |
|---|---|
| 头指针 | 指向链表起始位置的指针变量，通常写作 `head` |
| 头结点 | 为操作方便额外设置的辅助结点 |
| 首元结点 | 第一个真正保存有效数据的结点 |
| 尾结点 | 最后一个数据结点，其 `next` 通常为 `NULL` |

带头结点链表：

```text
head → [头结点] → [首元结点] → ... → [尾结点] → NULL
```

对应表达式：

```c
head                 /* 头指针 */
head 指向的结点       /* 头结点 */
head->next           /* 指向首元结点 */
tail                 /* 若维护，指向尾结点 */
tail->next == NULL
```

## 7. 头结点为什么有用

### 7.1 统一表头与中间插入

在已知前驱 `p` 后插入：

```c
newNode->next = p->next;
p->next = newNode;
```

如果在首元结点前插入，带头结点时前驱就是 `head` 指向的头结点，无需特殊修改头指针。

### 7.2 统一空表和非空表

无论表是否为空，头指针都指向有效的头结点。

```text
空表：   head → [头结点] → NULL
非空表： head → [头结点] → [数据] → ...
```

## 8. 创建带头结点链表

```c
#include <stdlib.h>

Node *createList(void) {
    Node *head = malloc(sizeof(Node));

    if (head == NULL) {
        return NULL;
    }

    head->next = NULL;
    return head;
}
```

头结点的 `data` 通常不使用，不必依赖其中的值。

## 9. 尾指针与表长

为了提高表尾插入效率，可以额外维护：

```c
Node *tail;
int length;
```

但这些辅助信息会引入维护责任：

- 插入第一个结点时更新 `tail`；
- 删除尾结点时重新定位或更新 `tail`；
- 每次成功插入或删除都更新 `length`；
- 空表时要明确 `tail` 指向头结点还是 `NULL`。

多存字段不一定自动更好，只有操作模式需要时才值得维护。

## 10. 单链表的优缺点

### 优点

1. 结点不要求连续存放；
2. 可按需动态申请；
3. 已知前驱时插入删除只需修改少量指针；
4. 不需要预留大块固定容量。

### 缺点

1. 每个结点需要额外 `next` 指针；
2. 不支持按位置随机访问；
3. 单向移动，无法直接从当前结点找到前驱；
4. 动态内存管理容易出现泄漏、悬空指针和重复释放。
