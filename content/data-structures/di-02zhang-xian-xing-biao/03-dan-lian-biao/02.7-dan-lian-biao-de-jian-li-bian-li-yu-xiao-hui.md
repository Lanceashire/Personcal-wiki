## 条目概览

头插法每次在表头插入，结果与输入顺序相反；尾插法维护尾指针后可以保持输入顺序。销毁链表时必须在释放当前结点前保存后继地址。

## 1. 结点创建辅助函数

```c
Node *createNode(ElemType value) {
    Node *node = malloc(sizeof(Node));

    if (node == NULL) {
        return NULL;
    }

    node->data = value;
    node->next = NULL;
    return node;
}
```

集中创建结点可以减少重复代码，并统一处理内存申请失败。

## 2. 头插法建表

输入：

```text
4, 7, 9
```

过程：

```text
初始：
head → [头结点] → NULL

插入 4：
head → [头结点] → [4] → NULL

插入 7：
head → [头结点] → [7] → [4] → NULL

插入 9：
head → [头结点] → [9] → [7] → [4] → NULL
```

最终顺序：

```text
9 → 7 → 4
```

与输入顺序相反。

### 2.1 完整代码

```c
Node *createByHeadInsertion(
    const ElemType values[],
    int n
) {
    if (values == NULL || n < 0) {
        return NULL;
    }

    Node *head = createNode(0);

    if (head == NULL) {
        return NULL;
    }

    for (int i = 0; i < n; i++) {
        Node *newNode = createNode(values[i]);

        if (newNode == NULL) {
            destroyList(head);
            return NULL;
        }

        newNode->next = head->next;
        head->next = newNode;
    }

    return head;
}
```

每次头插为 $O(1)$，建表总复杂度：

$$
O(n)
$$

## 3. 尾插法建表

输入：

```text
4, 7, 9
```

过程：

```text
head → [头结点] → NULL
tail ────────────┘

插入 4：
head → [头结点] → [4] → NULL
                       ↑
                      tail

插入 7：
head → [头结点] → [4] → [7] → NULL
                             ↑
                            tail
```

核心语句：

```c
tail->next = newNode;
tail = newNode;
```

### 3.1 完整代码

```c
Node *createByTailInsertion(
    const ElemType values[],
    int n
) {
    if (values == NULL || n < 0) {
        return NULL;
    }

    Node *head = createNode(0);

    if (head == NULL) {
        return NULL;
    }

    Node *tail = head;

    for (int i = 0; i < n; i++) {
        Node *newNode = createNode(values[i]);

        if (newNode == NULL) {
            destroyList(head);
            return NULL;
        }

        tail->next = newNode;
        tail = newNode;
    }

    return head;
}
```

最终顺序与输入顺序一致。

## 4. 为什么维护尾指针很重要

若每插入一个结点都从头寻找尾结点：

```text
第 1 次找 0 步
第 2 次找 1 步
...
第 n 次找 n-1 步
```

总成本：

$$
0+1+\cdots+(n-1)=O(n^2)
$$

维护 `tail` 后，每次尾插为 $O(1)$：

$$
n\times O(1)=O(n)
$$

> [!warning] 对话中暴露的易错点
> `tail = newNode` 不是多余语句。若不移动尾指针，下一次插入仍会从旧尾结点连接，链表状态将不符合预期。

## 5. 遍历

```c
void printList(const Node *head) {
    if (head == NULL) {
        return;
    }

    const Node *current = head->next;

    while (current != NULL) {
        printf("%d ", current->data);
        current = current->next;
    }

    printf("\n");
}
```

注意从：

```c
head->next
```

开始，因为头结点不属于有效数据。

遍历复杂度：

$$
O(n)
$$

## 6. 求表长

```c
int listLength(const Node *head) {
    if (head == NULL) {
        return 0;
    }

    int length = 0;
    const Node *current = head->next;

    while (current != NULL) {
        length++;
        current = current->next;
    }

    return length;
}
```

若链表结构中没有额外维护 `length`，求表长需要完整遍历：

$$
O(n)
$$

## 7. 清空链表

清空带头结点链表时保留头结点，只释放数据结点：

```c
void clearList(Node *head) {
    if (head == NULL) {
        return;
    }

    Node *current = head->next;
    head->next = NULL;

    while (current != NULL) {
        Node *nextNode = current->next;
        free(current);
        current = nextNode;
    }
}
```

先设置：

```c
head->next = NULL;
```

可以立即让表的逻辑状态变为空，再逐个释放旧结点。

## 8. 销毁链表

销毁会连头结点一起释放：

```c
void destroyList(Node *head) {
    Node *current = head;

    while (current != NULL) {
        Node *nextNode = current->next;
        free(current);
        current = nextNode;
    }
}
```

关键顺序：

```text
保存 next → free 当前结点 → 移到 next
```

错误写法：

```c
free(current);
current = current->next;
```

`current` 释放后再访问 `current->next` 属于使用已释放内存，行为未定义。

## 9. 清空与销毁的区别

| 操作 | 数据结点 | 头结点 | 操作后能否继续使用原表 |
|---|---|---|---|
| 清空 | 释放 | 保留 | 可以 |
| 销毁 | 释放 | 释放 | 不可以，原指针应置空 |

调用处建议：

```c
destroyList(head);
head = NULL;
```

这样可以减少误用悬空指针的风险。

## 10. 失败回滚

建表过程中若 `malloc` 失败，不能只返回 `NULL` 而遗留已经创建的结点。

正确策略：

```text
创建失败
→ 销毁已建立的部分链表
→ 返回失败
```

这也是前面建表代码调用 `destroyList(head)` 的原因。

## 11. 头插法与尾插法比较

| 比较项 | 头插法 | 维护尾指针的尾插法 |
|---|---|---|
| 单次插入 | $O(1)$ | $O(1)$ |
| 建表总时间 | $O(n)$ | $O(n)$ |
| 最终顺序 | 与输入相反 | 与输入相同 |
| 辅助指针 | 不需要尾指针 | 需要 `tail` |
| 常见用途 | 逆序建立、快速前插 | 保持输入顺序 |
