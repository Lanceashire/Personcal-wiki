## 条目概览

双向链表的每个结点同时保存前驱和后继地址，可以双向移动。它以额外空间和更复杂的链接维护，换取对前驱的直接访问。

> [!summary]
> 双向链表的每个结点同时保存前驱和后继地址，可以双向移动。它以额外空间和更复杂的链接维护，换取对前驱的直接访问。

## 1. 结点结构

```c
typedef struct DNode {
    ElemType data;
    struct DNode *prior;
    struct DNode *next;
} DNode;
```

结构：

```text
NULL ← [10] ⇄ [20] ⇄ [30] → NULL
```

若 `p` 指向 `[20]`：

```c
p->prior  /* 指向 [10] */
p->next   /* 指向 [30] */
```

## 2. 双向链表的不变量

对任意相邻结点 `p` 和 `q`：

```c
p->next == q
q->prior == p
```

也就是：

```c
p->next->prior == p
q->prior->next == q
```

修改链表后必须同时恢复两个方向的一致性。

## 3. 带头结点双向链表

```text
head → [头结点] ⇄ [10] ⇄ [20] → NULL
```

初始化：

```c
DNode *createDoublyList(void) {
    DNode *head = malloc(sizeof(DNode));

    if (head == NULL) {
        return NULL;
    }

    head->prior = NULL;
    head->next = NULL;
    return head;
}
```

空表：

```c
head->next == NULL
```

## 4. 在已知结点后插入

原结构：

```text
p ⇄ q
```

插入 `s`：

```text
p ⇄ s ⇄ q
```

当 `q` 存在时，四条关系是：

```c
s->next = p->next;
s->prior = p;
p->next->prior = s;
p->next = s;
```

逻辑含义：

```text
s 的后继是 q
s 的前驱是 p
q 的前驱改为 s
p 的后继改为 s
```

## 5. 通用后插函数

尾后插时：

```c
p->next == NULL
```

不能直接访问 `p->next->prior`。

```c
int insertAfter(
    DNode *p,
    ElemType value
) {
    if (p == NULL) {
        return 0;
    }

    DNode *newNode = malloc(sizeof(DNode));

    if (newNode == NULL) {
        return 0;
    }

    newNode->data = value;
    newNode->prior = p;
    newNode->next = p->next;

    if (p->next != NULL) {
        p->next->prior = newNode;
    }

    p->next = newNode;
    return 1;
}
```

## 6. 删除已知结点

原结构：

```text
p ⇄ q ⇄ r
```

删除 `q`：

```c
q->prior->next = q->next;

if (q->next != NULL) {
    q->next->prior = q->prior;
}

free(q);
```

删除后：

```text
p ⇄ r
```

## 7. 通用删除函数

假设不允许删除头结点：

```c
int deleteDNode(
    DNode *head,
    DNode *target,
    ElemType *removed
) {
    if (
        head == NULL ||
        target == NULL ||
        target == head ||
        target->prior == NULL
    ) {
        return 0;
    }

    if (removed != NULL) {
        *removed = target->data;
    }

    target->prior->next = target->next;

    if (target->next != NULL) {
        target->next->prior = target->prior;
    }

    free(target);
    return 1;
}
```

## 8. 为什么删除已知结点是 $O(1)$

单链表删除已知目标结点时，通常还需要寻找前驱。

双向链表可直接读取：

```c
target->prior
```

因此若目标结点指针已知，删除只修改常数条链接：

$$
O(1)
$$

但若题目只给下标或关键字，仍要先查找目标，整体可能是：

$$
O(n)
$$

## 9. 循环双向链表

带头结点循环双向链表的空表可以设计为：

```text
       ┌─────────────┐
       ↓             │
head → [头结点] ─────┘
       ↑             │
       └─────────────┘
```

初始化：

```c
head->next = head;
head->prior = head;
```

空表判断：

```c
head->next == head
```

尾结点：

```c
head->prior
```

这种结构可以减少首尾边界分支，但必须始终维持循环不变量。

## 10. 优势与代价

| 项目 | 单链表 | 双向链表 |
|---|---|---|
| 指针域 | 1 个 | 2 个 |
| 向后移动 | 支持 | 支持 |
| 向前移动 | 不直接支持 | 支持 |
| 已知目标删除 | 通常还需前驱 | 可直接 $O(1)$ |
| 代码复杂度 | 较低 | 较高 |
| 链接维护错误风险 | 较低 | 较高 |

每个结点多一个指针，$n$ 个结点额外增加：

$$
O(n)
$$

指针空间。

## 11. 常见错误

1. 只修改 `next`，忘记修改 `prior`；
2. 插入尾部时访问 `NULL->prior`；
3. 删除尾结点时访问 `target->next->prior`；
4. 删除头结点导致链表入口丢失；
5. 将“已知目标结点删除为 $O(1)$”误解为“按位置删除也是 $O(1)$”；
6. 循环双向链表仍使用 `NULL` 判断首尾。

## 12. 指针一致性检查

调试时可以遍历检查：

```c
int validateDoublyList(const DNode *head) {
    if (head == NULL) {
        return 0;
    }

    const DNode *current = head;

    while (current->next != NULL) {
        if (current->next->prior != current) {
            return 0;
        }

        current = current->next;
    }

    return 1;
}
```
