## 条目概览

循环链表把尾结点重新连接到链表起点，因此表中不再以 NULL 表示结束。遍历时必须使用“是否回到起点”作为终止条件。

> [!summary]
> 循环链表把尾结点重新连接到链表起点，因此表中不再以 `NULL` 表示结束。遍历时必须使用“是否回到起点”作为终止条件。

## 1. 循环单链表结构

普通带头结点单链表：

```text
head → [头结点] → [10] → [20] → [30] → NULL
```

带头结点循环单链表：

```text
head → [头结点] → [10] → [20] → [30]
         ↑                         ↓
         └─────────────────────────┘
```

尾结点满足：

```c
tail->next == head
```

## 2. 初始化与空表

```c
Node *createCircularList(void) {
    Node *head = malloc(sizeof(Node));

    if (head == NULL) {
        return NULL;
    }

    head->next = head;
    return head;
}
```

空表：

```text
head
 ↓
[头结点] ─┐
   ↑      │
   └──────┘
```

判断：

```c
head->next == head
```

这与普通带头结点链表的：

```c
head->next == NULL
```

不同。

## 3. 遍历

```c
void printCircularList(const Node *head) {
    if (head == NULL) {
        return;
    }

    const Node *current = head->next;

    while (current != head) {
        printf("%d ", current->data);
        current = current->next;
    }

    printf("\n");
}
```

> [!warning] 对话中暴露的易错点
> 循环链表中通常不会遇到 `NULL`。若仍用 `while (current != NULL)` 遍历，会不断绕圈，形成死循环。

## 4. 尾插

若维护 `tail`，空表时约定：

```c
tail = head;
```

插入新尾结点：

```c
int appendCircular(
    Node *head,
    Node **tail,
    ElemType value
) {
    if (
        head == NULL ||
        tail == NULL ||
        *tail == NULL
    ) {
        return 0;
    }

    Node *newNode = malloc(sizeof(Node));

    if (newNode == NULL) {
        return 0;
    }

    newNode->data = value;
    newNode->next = head;
    (*tail)->next = newNode;
    *tail = newNode;
    return 1;
}
```

指针变化：

```text
旧尾 → head

变为：

旧尾 → 新尾 → head
```

## 5. 为什么尾指针有时比头指针更方便

若循环单链表只维护尾指针 `tail`：

```text
首元结点：tail->next
尾结点：  tail
```

对于不带头结点的循环链表，尾指针可以同时快速访问表尾和表头，因此在队列等结构中很常见。

具体表达式取决于是否带头结点，不能机械套用。

## 6. 合并两个循环链表

若两个不带头结点的循环单链表分别维护尾指针 `tailA`、`tailB`：

```text
headA = tailA->next
headB = tailB->next
```

合并时可以交换两条尾部链接：

```c
Node *headA = tailA->next;
Node *headB = tailB->next;

tailA->next = headB;
tailB->next = headA;
```

合并后若以 `tailB` 为新尾指针，即可得到完整循环链表。

这个操作只修改常数条指针：

$$
O(1)
$$

## 7. 约瑟夫环

问题：

> $n$ 个人围成一圈，从第一个人开始报数，每报到第 $m$ 个人就将其移出，之后从下一人重新从 1 报数，直到只剩一人。

例如：

```text
n = 5, m = 3
```

初始：

```text
1 → 2 → 3 → 4 → 5 → 回到 1
```

淘汰：

```text
3, 1, 5, 2
```

最后剩余：

```text
4
```

## 8. 链表模拟约瑟夫环

使用不带头结点的循环链表，并令 `previous` 指向 `current` 的前驱：

```c
int josephus(Node **head, int m) {
    if (
        head == NULL ||
        *head == NULL ||
        m <= 0
    ) {
        return -1;
    }

    Node *current = *head;
    Node *previous = current;

    while (previous->next != current) {
        previous = previous->next;
    }

    while (current->next != current) {
        for (int count = 1; count < m; count++) {
            previous = current;
            current = current->next;
        }

        previous->next = current->next;

        if (current == *head) {
            *head = current->next;
        }

        Node *removed = current;
        current = current->next;
        free(removed);
    }

    *head = current;
    return current->data;
}
```

每淘汰一人需要移动约 $m$ 步，总体常写为：

$$
O(nm)
$$

当 $m$ 被视为常数时，可看作 $O(n)$；若 $m$ 随输入变化，则必须保留 $m$。

## 9. 数学递推解

如果只求最后幸存者编号，不要求输出淘汰顺序，可以使用递推。

使用从 `0` 开始的编号：

$$
f(1)=0
$$

$$
f(n)=(f(n-1)+m)\bmod n
$$

最终若题目编号从 `1` 开始：

$$
answer=f(n)+1
$$

迭代代码：

```c
int josephusWinner(int n, int m) {
    if (n <= 0 || m <= 0) {
        return -1;
    }

    int result = 0;

    for (int size = 2; size <= n; size++) {
        result = (result + m) % size;
    }

    return result + 1;
}
```

时间复杂度：

$$
O(n)
$$

空间复杂度：

$$
O(1)
$$

## 10. 常见错误

1. 使用 `NULL` 作为循环链表遍历终点；
2. 空表仍按普通链表判断；
3. 尾插后忘记让新尾指向 `head`；
4. 删除唯一结点时没有单独处理；
5. 约瑟夫环报数起点和淘汰后的下一起点处理错误；
6. 混淆从 `0` 开始与从 `1` 开始的编号。
