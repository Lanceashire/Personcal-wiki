## 条目概览

线性表是由零个或多个相同类型数据元素组成的有限序列。它描述一对一的逻辑关系，不限定元素必须使用数组还是链表存储。

> [!summary]
> 线性表是由零个或多个相同类型数据元素组成的有限序列。它描述一对一的逻辑关系，不限定元素必须使用数组还是链表存储。

## 1. 规范定义

线性表可写成：

$$
L=(a_1,a_2,\ldots,a_n)
$$

其中：

| 符号 | 含义 |
|---|---|
| $L$ | 线性表 |
| $a_i$ | 第 $i$ 个数据元素 |
| $n$ | 表长，即数据元素个数 |

当：

$$
n=0
$$

时，称为**空表**。

## 2. 结构特点

对于非空线性表：

1. 存在唯一的第一个元素 $a_1$；
2. 存在唯一的最后一个元素 $a_n$；
3. 除 $a_1$ 外，每个元素有且仅有一个直接前驱；
4. 除 $a_n$ 外，每个元素有且仅有一个直接后继。

```text
a1 → a2 → a3 → ... → an
```

例如：

```text
春 → 夏 → 秋 → 冬
```

`夏` 的直接前驱是 `春`，直接后继是 `秋`。

## 3. “有限序列”中的两个关键词

### 3.1 有限

线性表中的元素数量必须有限。即使程序可以不断追加元素，在任意确定时刻，表长仍是一个有限整数。

### 3.2 序列

元素之间存在位置次序。即使两个元素的值相等，它们仍可以处于不同位置。

```text
L = (5, 8, 5)
```

第一个 `5` 和第三个 `5` 的值相同，但位置不同，是两个数据元素。

## 4. 位置编号与数组下标

教材描述线性表时通常使用从 `1` 开始的位置编号：

```text
第 1 个元素、第 2 个元素、...、第 n 个元素
```

C 数组下标通常从 `0` 开始：

```text
data[0], data[1], ..., data[n - 1]
```

两者换算：

$$
数组下标=逻辑位序-1
$$

$$
逻辑位序=数组下标+1
$$

> [!warning] 易错点
> 写代码前必须先确定接口使用“逻辑位序”还是“数组下标”。二者混用是顺序表和链表题中最常见的越界来源之一。

本章代码模板默认：

```text
index 表示从 0 开始的数组式位置
```

若题目使用从 `1` 开始的位序，需要主动换算。

## 5. 线性表的抽象数据类型

线性表的常见操作包括：

| 操作 | 作用 |
|---|---|
| `InitList` | 初始化空表 |
| `DestroyList` | 销毁表并释放资源 |
| `ClearList` | 清空表中元素 |
| `ListEmpty` | 判断是否为空 |
| `ListLength` | 获取表长 |
| `GetElem` | 按位置取得元素 |
| `LocateElem` | 按值查找元素 |
| `PriorElem` | 求直接前驱 |
| `NextElem` | 求直接后继 |
| `ListInsert` | 在指定位置插入元素 |
| `ListDelete` | 删除指定位置的元素 |
| `ListTraverse` | 按逻辑顺序访问所有元素 |

抽象数据类型只规定操作语义，不规定底层实现。

```text
线性表 ADT
├── 顺序表：用连续空间实现
└── 链表：用指针链接实现
```

## 6. 基本操作的前置条件

算法接口必须说明合法范围。

若当前表长为 $n$，使用从 `0` 开始的位置：

| 操作 | 合法范围 |
|---|---|
| 读取或删除 | $0\le index<n$ |
| 插入 | $0\le index\le n$ |

插入允许：

```text
index == n
```

因为这表示在表尾追加。

删除不允许：

```text
index == n
```

因为最后一个有效元素的下标是 `n - 1`。

## 7. 线性表不是数组，也不是链表

线性表是逻辑结构；数组和链表是实现方式。

| 层次 | 示例 |
|---|---|
| 逻辑结构 | 线性表 |
| 顺序存储实现 | 顺序表 |
| 链式存储实现 | 单链表、循环链表、双向链表 |

> [!warning] 对话中暴露的易错点
> “线性表就是数组”或“线性表就是链表”都不准确。顺序表和链表具有相同的线性逻辑关系，但物理存储和操作代价不同。

## 8. 为什么要学习两种实现

不同任务关注点不同：

- 频繁按位置访问：顺序表更合适；
- 频繁在已知结点附近插入删除：链表更合适；
- 数据规模较稳定：静态顺序表简单；
- 数据规模变化明显：动态顺序表或链表更灵活。

学习重点不是判断谁“绝对更好”，而是根据操作模式选择。

## 9. 例题

### 例 1：空表有没有第一个元素？

没有。空表的表长为 `0`，不存在首元素和尾元素。

### 例 2：表中能否出现重复值？

可以。线性表强调位置次序，不要求元素值互不相同。

### 例 3：在表长为 5 的表中插入，下标范围是什么？

合法插入下标：

$$
0\le index\le5
$$

其中 `index == 5` 表示表尾追加。
