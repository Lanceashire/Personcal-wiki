## 条目概览

“普通线性表合并”和“两个有序表合并”是不同问题。前者通常要查重，最坏可能为 $O(mn)$；后者利用有序性和双指针，只需 $O(m+n)$。

> [!summary]
> “普通线性表合并”和“两个有序表合并”是不同问题。前者通常要查重，最坏可能为 $O(mn)$；后者利用有序性和双指针，只需 $O(m+n)$。

## 1. 普通线性表合并

目标：

> 将 `Lb` 中存在、但 `La` 中不存在的元素追加到 `La`。

例如：

```text
La = [1, 3, 5]
Lb = [3, 4, 6]
```

合并后：

```text
La = [1, 3, 5, 4, 6]
```

注意结果不一定有序，并且只避免把 `Lb` 中已经存在于 `La` 的值再次加入。

## 2. 普通合并算法

```c
int mergeUnique(
    SeqList *la,
    const SeqList *lb
) {
    if (la == NULL || lb == NULL) {
        return 0;
    }

    for (int i = 0; i < lb->length; i++) {
        ElemType value = lb->data[i];

        if (locateElem(la, value) == -1) {
            if (
                !insertElem(
                    la,
                    la->length,
                    value
                )
            ) {
                return 0;
            }
        }
    }

    return 1;
}
```

过程：

1. 遍历 `Lb`；
2. 对每个元素在 `La` 中按值查找；
3. 若不存在，则追加到 `La` 表尾；
4. 若容量不足，合并失败。

## 3. 普通合并复杂度

设：

$$
|La|=m,\quad |Lb|=n
$$

对 `Lb` 的每个元素，都可能在 `La` 中执行一次线性查找。

最坏总成本近似：

$$
n\times O(m)=O(mn)
$$

随着新元素追加，`La` 还会增长，但渐近上仍常记为：

$$
O(mn+n^2)
$$

若把 $m$ 和 $n$ 看作同一数量级，则为 $O(n^2)$。

> [!note] 对话补充
> 对话中将普通合并概括为“近似集合并集”。这个说法可帮助理解，但线性表允许重复元素，是否去重必须由题目语义明确规定。

## 4. 有序顺序表合并

输入：

```text
La = [1, 3, 7]
Lb = [2, 3, 6, 8]
```

若保留重复值，输出：

```text
Lc = [1, 2, 3, 3, 6, 7, 8]
```

前提：

1. `La` 已按非递减顺序排列；
2. `Lb` 已按非递减顺序排列；
3. `Lc` 容量至少为 `La.length + Lb.length`。

## 5. 双指针思想

设置：

```text
i 指向 La 当前未处理元素
j 指向 Lb 当前未处理元素
k 指向 Lc 下一个写入位置
```

每次比较：

```text
La[i] 与 Lb[j]
```

较小者一定是全部未处理元素中的最小值，可以安全写入 `Lc`。

```text
La: 1  3  7
    ↑
    i

Lb: 2  3  6  8
    ↑
    j

比较 1 和 2，将 1 写入 Lc，然后 i 后移。
```

## 6. 完整代码

```c
int mergeSortedLists(
    const SeqList *la,
    const SeqList *lb,
    SeqList *lc
) {
    if (
        la == NULL ||
        lb == NULL ||
        lc == NULL ||
        la->length + lb->length > MAX_SIZE
    ) {
        return 0;
    }

    int i = 0;
    int j = 0;
    int k = 0;

    while (
        i < la->length &&
        j < lb->length
    ) {
        if (la->data[i] <= lb->data[j]) {
            lc->data[k++] = la->data[i++];
        } else {
            lc->data[k++] = lb->data[j++];
        }
    }

    while (i < la->length) {
        lc->data[k++] = la->data[i++];
    }

    while (j < lb->length) {
        lc->data[k++] = lb->data[j++];
    }

    lc->length = k;
    return 1;
}
```

## 7. 为什么要处理剩余元素

主循环在任意一张表处理完时结束。

例如：

```text
La = [1, 2]
Lb = [3, 4, 5]
```

当 `La` 耗尽后，`Lb` 中的 `3, 4, 5` 已经有序，并且都不小于此前输出的元素，可以直接依次复制。

## 8. 正确性直觉

在每一步中：

1. `La[i]` 是 `La` 未处理部分的最小值；
2. `Lb[j]` 是 `Lb` 未处理部分的最小值；
3. 二者较小者就是所有未处理元素中的最小值；
4. 将它写入结果，不会破坏有序性。

这个不变量一直保持到某个输入表耗尽。

## 9. 复杂度

每个元素只会被比较或写入有限次。

时间复杂度：

$$
O(m+n)
$$

新建结果表的额外空间：

$$
O(m+n)
$$

若结果写入已有足够空间的目标表，除目标存储外，算法辅助变量只占：

$$
O(1)
$$

## 10. 是否去重

上面的标准合并保留重复值：

```text
[1, 3] + [3, 5]
→ [1, 3, 3, 5]
```

若题目要求集合式去重，相等时只写入一次：

```c
if (la->data[i] < lb->data[j]) {
    lc->data[k++] = la->data[i++];
} else if (la->data[i] > lb->data[j]) {
    lc->data[k++] = lb->data[j++];
} else {
    lc->data[k++] = la->data[i];
    i++;
    j++;
}
```

还要考虑同一输入表内部本身是否包含连续重复值。

> [!warning] 易错点
> “合并有序表”不自动等于“去重合并”。是否保留相同关键字必须看题目要求。

## 11. 与归并排序的关系

归并排序的核心操作正是：

> 合并两个已经有序的子序列。

区别在于：

- 本节输入是两张独立有序表；
- 归并排序通常把同一数组划分为两个有序区间，再合并到辅助空间。

## 12. 常见错误

1. 输入表并未有序却使用双指针合并；
2. 主循环结束后忘记复制剩余元素；
3. 结果表容量不足；
4. 相等时两个指针都不移动，导致死循环；
5. 混淆“保留重复”与“集合去重”；
6. 写入结果后忘记设置 `lc->length`。
