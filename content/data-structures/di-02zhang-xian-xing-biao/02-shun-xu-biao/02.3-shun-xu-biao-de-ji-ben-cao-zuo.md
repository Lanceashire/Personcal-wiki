## 条目概览

顺序表操作的核心是维护两个不变量：有效元素始终位于 data[0] 到 data[length - 1]，并且 length 始终等于有效元素个数。

> [!summary]
> 顺序表操作的核心是维护两个不变量：有效元素始终位于 `data[0]` 到 `data[length - 1]`，并且 `length` 始终等于有效元素个数。

本文默认：

```c
#define MAX_SIZE 100

typedef int ElemType;

typedef struct {
    ElemType data[MAX_SIZE];
    int length;
} SeqList;
```

接口中的 `index` 从 `0` 开始。

## 1. 按位置读取

前置条件：

$$
0\le index<length
$$

代码：

```c
int getElem(
    const SeqList *list,
    int index,
    ElemType *result
) {
    if (
        list == NULL ||
        result == NULL ||
        index < 0 ||
        index >= list->length
    ) {
        return 0;
    }

    *result = list->data[index];
    return 1;
}
```

地址可直接计算，因此：

$$
T(n)=O(1)
$$

## 2. 按值查找

```c
int locateElem(
    const SeqList *list,
    ElemType target
) {
    if (list == NULL) {
        return -1;
    }

    for (int i = 0; i < list->length; i++) {
        if (list->data[i] == target) {
            return i;
        }
    }

    return -1;
}
```

| 情况 | 比较次数 | 复杂度 |
|---|---:|---:|
| 最好 | 1 | $O(1)$ |
| 最坏 | $n$ | $O(n)$ |
| 平均 | 约与 $n$ 成正比 | $O(n)$ |

> [!warning] 对话中暴露的易错点
> 顺序表的“随机访问快”指按位置访问。无序顺序表按值查找仍要逐个比较。

## 3. 插入操作

在下标 `index` 插入 `value`。

原表：

```text
下标： 0    1    2    3
数据：[10] [20] [30] [40]
```

在 `index = 2` 插入 `25`：

```text
[10] [20] [25] [30] [40]
```

### 3.1 前置条件

表未满：

$$
length<MAX\_SIZE
$$

位置合法：

$$
0\le index\le length
$$

### 3.2 为什么从后向前移动

移动顺序：

```text
40 → 右移
30 → 右移
空出 index = 2
```

代码：

```c
for (int j = list->length; j > index; j--) {
    list->data[j] = list->data[j - 1];
}
```

若从前向后右移：

```c
data[index + 1] = data[index];
```

会先覆盖后面的旧值，导致尚未搬走的数据丢失。

### 3.3 完整代码

```c
int insertElem(
    SeqList *list,
    int index,
    ElemType value
) {
    if (
        list == NULL ||
        index < 0 ||
        index > list->length ||
        list->length >= MAX_SIZE
    ) {
        return 0;
    }

    for (int j = list->length; j > index; j--) {
        list->data[j] = list->data[j - 1];
    }

    list->data[index] = value;
    list->length++;
    return 1;
}
```

### 3.4 复杂度

若当前表长为 $n$，在下标 $i$ 插入，需要移动：

$$
n-i
$$

个元素。

| 插入位置 | 移动次数 | 复杂度 |
|---|---:|---:|
| 表尾 `i=n` | 0 | $O(1)$ |
| 表头 `i=0` | $n$ | $O(n)$ |
| 平均 | 约 $n/2$ | $O(n)$ |

## 4. 删除操作

原表：

```text
[10] [20] [30] [40]
```

删除 `index = 1`：

```text
[10] [30] [40]
```

### 4.1 前置条件

位置合法：

$$
0\le index<length
$$

### 4.2 为什么从前向后左移

删除位置形成空缺后，应依次用后一个元素覆盖前一个位置：

```text
30 → data[1]
40 → data[2]
```

代码：

```c
for (
    int j = index;
    j < list->length - 1;
    j++
) {
    list->data[j] = list->data[j + 1];
}
```

### 4.3 完整代码

```c
int deleteElem(
    SeqList *list,
    int index,
    ElemType *removed
) {
    if (
        list == NULL ||
        index < 0 ||
        index >= list->length
    ) {
        return 0;
    }

    if (removed != NULL) {
        *removed = list->data[index];
    }

    for (
        int j = index;
        j < list->length - 1;
        j++
    ) {
        list->data[j] = list->data[j + 1];
    }

    list->length--;
    return 1;
}
```

### 4.4 复杂度

在下标 $i$ 删除，需要移动：

$$
n-i-1
$$

个元素。

| 删除位置 | 移动次数 | 复杂度 |
|---|---:|---:|
| 表尾 `i=n-1` | 0 | $O(1)$ |
| 表头 `i=0` | $n-1$ | $O(n)$ |
| 平均 | 约 $(n-1)/2$ | $O(n)$ |

## 5. 遍历

```c
void printList(const SeqList *list) {
    if (list == NULL) {
        return;
    }

    for (int i = 0; i < list->length; i++) {
        printf("%d ", list->data[i]);
    }

    printf("\n");
}
```

每个元素访问一次：

$$
O(n)
$$

## 6. 清空与销毁

静态顺序表清空：

```c
void clearList(SeqList *list) {
    if (list != NULL) {
        list->length = 0;
    }
}
```

把 `length` 设为 `0` 后，原数组内容可能仍留在内存中，但它们不再属于逻辑表的有效元素。

静态数组由结构体自身管理，一般不对 `data` 调用 `free`。

## 7. 操作不变量

每次操作结束后都应满足：

```text
0 <= length <= MAX_SIZE
有效元素位于 data[0 .. length - 1]
插入成功后 length 增加 1
删除成功后 length 减少 1
操作失败时原表不应被破坏
```

## 8. 常见错误

### 错误 1：满表检查写成 `length > MAX_SIZE`

正确条件是：

```c
length >= MAX_SIZE
```

当二者相等时已经没有空位。

### 错误 2：插入范围写成 `index < length`

这样会错误拒绝表尾插入。插入应允许：

```c
index == length
```

### 错误 3：移动完成后忘记更新 `length`

数组内容可能看似正确，但表的逻辑状态已经不一致。

### 错误 4：插入时从前往后右移

会覆盖尚未保存的元素。

### 错误 5：删除循环访问 `data[length]`

最后一次合法读取应为：

```c
data[length - 1]
```
