## 条目概览

顺序表使用一段地址连续的存储空间，按照线性表的逻辑次序依次保存元素。它的核心优势是随机访问，核心限制是中间插入删除需要移动元素。

> [!summary]
> 顺序表使用一段地址连续的存储空间，按照线性表的逻辑次序依次保存元素。它的核心优势是随机访问，核心限制是中间插入删除需要移动元素。

## 1. 顺序存储原理

若每个元素占用 $w$ 个字节，数组首地址为 $base$，则下标为 $i$ 的元素地址为：

$$
LOC(data[i])=base+i\times w
$$

因为地址可以直接计算，所以按下标访问只需要常数次操作：

$$
O(1)
$$

## 2. 静态顺序表

```c
#define MAX_SIZE 100

typedef int ElemType;

typedef struct {
    ElemType data[MAX_SIZE];
    int length;
} SeqList;
```

字段含义：

| 字段 | 含义 |
|---|---|
| `data` | 存放数据元素的连续数组 |
| `length` | 当前有效元素个数 |
| `MAX_SIZE` | 数组最多可容纳的元素个数 |

初始化：

```c
void initList(SeqList *list) {
    if (list == NULL) {
        return;
    }

    list->length = 0;
}
```

## 3. `length`、容量与下标

假设：

```text
MAX_SIZE = 8
length = 3
```

内存示意：

```text
下标：  0    1    2    3    4    5    6    7
数据： [10] [20] [30] [空] [空] [空] [空] [空]
```

有效下标范围：

$$
0\le i<length
$$

当前尾元素下标：

$$
length-1
$$

下一个可插入的表尾位置：

$$
length
$$

> [!warning] 对话中暴露的易错点
> `length` 不是最后一个元素的下标，而是有效元素个数。非空表的尾元素下标是 `length - 1`。

## 4. 空表与满表

空表：

```c
list->length == 0
```

满表：

```c
list->length == MAX_SIZE
```

静态顺序表满后无法继续插入，除非修改最大容量并重新编译，或者改用动态顺序表。

## 5. 动态顺序表

动态顺序表把数据区放在堆内存中：

```c
#include <stdlib.h>

typedef struct {
    ElemType *data;
    int length;
    int capacity;
} DynamicList;
```

初始化：

```c
int initDynamicList(DynamicList *list, int initialCapacity) {
    if (list == NULL || initialCapacity <= 0) {
        return 0;
    }

    list->data = malloc(
        (size_t)initialCapacity * sizeof(ElemType)
    );

    if (list->data == NULL) {
        return 0;
    }

    list->length = 0;
    list->capacity = initialCapacity;
    return 1;
}
```

销毁：

```c
void destroyDynamicList(DynamicList *list) {
    if (list == NULL) {
        return;
    }

    free(list->data);
    list->data = NULL;
    list->length = 0;
    list->capacity = 0;
}
```

## 6. 动态扩容

容量不足时，可以申请更大的连续空间：

```c
int expandList(DynamicList *list) {
    if (list == NULL || list->capacity <= 0) {
        return 0;
    }

    int newCapacity = list->capacity * 2;
    ElemType *newData = realloc(
        list->data,
        (size_t)newCapacity * sizeof(ElemType)
    );

    if (newData == NULL) {
        return 0;
    }

    list->data = newData;
    list->capacity = newCapacity;
    return 1;
}
```

使用临时指针接收 `realloc` 结果，是为了避免扩容失败时覆盖原指针。

> [!warning] 易错点
> 不建议直接写 `list->data = realloc(...)`。若分配失败，返回值为 `NULL`，原内存地址可能因此丢失，造成内存泄漏。

## 7. 静态与动态顺序表比较

| 比较项 | 静态顺序表 | 动态顺序表 |
|---|---|---|
| 数据区 | 结构体内固定数组 | 堆内存 |
| 容量 | 编译时基本确定 | 运行时可扩容 |
| 实现难度 | 较低 | 需要内存管理 |
| 扩容 | 不支持 | 支持，但可能搬迁数据 |
| 销毁 | 通常无需单独释放数组 | 必须 `free` |

## 8. 顺序表的优缺点

### 优点

1. 按下标访问为 $O(1)$；
2. 存储密度高，没有每个元素的指针域；
3. 连续内存具有较好的缓存局部性；
4. 实现简单，遍历效率高。

### 缺点

1. 中间插入删除需要移动大量元素；
2. 静态容量可能不足或浪费；
3. 动态扩容需要重新分配并复制数据；
4. 需要一块足够大的连续内存。
