## 条目概览

串操作必须同时维护字符内容、长度、容量和结束符。求子串、连接和插入的边界计算与顺序表非常相似。

> [!summary]
> 串操作必须同时维护字符内容、长度、容量和结束符。求子串、连接和插入的边界计算与顺序表非常相似。

本文使用：

```c
#define MAX_STRING_SIZE 256

typedef struct {
    char data[MAX_STRING_SIZE];
    int length;
} StaticString;
```

## 1. 赋值

```c
int assignString(StaticString *s, const char *source) {
    if (s == NULL || source == NULL) {
        return 0;
    }

    int length = 0;

    while (source[length] != '\0') {
        if (length >= MAX_STRING_SIZE) {
            return 0;
        }
        length++;
    }

    for (int i = 0; i < length; i++) {
        s->data[i] = source[i];
    }

    s->length = length;
    return 1;
}
```

若结构不依赖 `'\0'`，不必在 `data[length]` 写结束符；若要传给 C 字符串函数，则数组应额外预留位置。

## 2. 串比较

```c
int compareString(
    const StaticString *a,
    const StaticString *b
) {
    int minLength =
        a->length < b->length ?
        a->length : b->length;

    for (int i = 0; i < minLength; i++) {
        if (a->data[i] < b->data[i]) return -1;
        if (a->data[i] > b->data[i]) return 1;
    }

    if (a->length < b->length) return -1;
    if (a->length > b->length) return 1;
    return 0;
}
```

最坏比较：

$$
O(\min(m,n))
$$

## 3. 连接

目标：

```text
"data" + "structure" = "datastructure"
```

容量条件：

$$
|a|+|b|\le MAX\_STRING\_SIZE
$$

```c
int concatString(
    const StaticString *a,
    const StaticString *b,
    StaticString *result
) {
    if (
        a == NULL ||
        b == NULL ||
        result == NULL ||
        a->length + b->length > MAX_STRING_SIZE
    ) {
        return 0;
    }

    int k = 0;

    for (int i = 0; i < a->length; i++) {
        result->data[k++] = a->data[i];
    }

    for (int j = 0; j < b->length; j++) {
        result->data[k++] = b->data[j];
    }

    result->length = k;
    return 1;
}
```

时间：

$$
O(m+n)
$$

## 4. 求子串

从下标 `position` 开始取 `length` 个字符。

合法条件：

$$
0\le position\le |S|
$$

$$
0\le length
$$

$$
position+length\le |S|
$$

```c
int substring(
    const StaticString *source,
    int position,
    int length,
    StaticString *result
) {
    if (
        source == NULL ||
        result == NULL ||
        position < 0 ||
        length < 0 ||
        position + length > source->length
    ) {
        return 0;
    }

    for (int i = 0; i < length; i++) {
        result->data[i] =
            source->data[position + i];
    }

    result->length = length;
    return 1;
}
```

时间：

$$
O(length)
$$

## 5. 插入子串

在位置 `position` 插入长度为 $m$ 的串，原串长度为 $n$。

先将后缀向右移动 $m$ 位：

```text
必须从后向前移动
```

移动次数：

$$
n-position
$$

总时间复杂度：

$$
O(n+m)
$$

它与顺序表插入完全同构。

## 6. 删除子串

删除区间：

```text
[position, position + length)
```

后续字符向左移动：

```text
从前向后覆盖
```

移动次数：

$$
n-(position+length)
$$

## 7. 替换

替换可理解为：

```text
查找模式
→ 删除匹配段
→ 插入新串
```

若替换串与模式串长度不同，顺序存储中可能需要移动大量字符。

## 8. C 标准库对应

| 需求 | 常见函数 | 主要风险 |
|---|---|---|
| 求长 | `strlen` | 必须有 `'\0'` |
| 复制 | `strcpy` | 目标容量不足 |
| 连接 | `strcat` | 目标容量不足 |
| 比较 | `strcmp` | 返回值只看符号 |
| 查找字符 | `strchr` | 返回指针或 `NULL` |
| 查找子串 | `strstr` | 朴素或库内部实现依平台而异 |

课程代码应明确容量，不应无条件使用不检查边界的复制和连接。
