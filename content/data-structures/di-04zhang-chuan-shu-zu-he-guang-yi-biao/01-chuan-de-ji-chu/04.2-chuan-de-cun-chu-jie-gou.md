## 条目概览

串可以定长顺序存储、动态堆分配或块链存储。C 风格字符串依赖 '\0'，教材结构也可能显式保存长度，两种设计的边界和代价不同。

> [!summary]
> 串可以定长顺序存储、动态堆分配或块链存储。C 风格字符串依赖 `'\0'`，教材结构也可能显式保存长度，两种设计的边界和代价不同。

## 1. 定长顺序存储

```c
#define MAX_STRING_SIZE 256

typedef struct {
    char data[MAX_STRING_SIZE];
    int length;
} StaticString;
```

约定有效字符位于：

```text
data[0] ... data[length - 1]
```

优点：

- 随机访问 $O(1)$；
- 结构简单；
- 长度显式，无需每次扫描 `'\0'`。

缺点：

- 容量固定；
- 连接或插入可能溢出；
- 未使用区域可能浪费。

## 2. C 风格字符串

```c
char text[256] = "hello";
```

使用 `'\0'` 标记结束：

```text
h e l l o \0
```

`strlen(text)` 需要从头扫描到 `'\0'`：

$$
O(n)
$$

若未正确写入结束符，`strlen`、`printf("%s")` 等函数可能越界读取，产生未定义行为。

## 3. 两种长度表示比较

| 设计 | 获取长度 | 可否包含 `'\0'` 数据 | 维护责任 |
|---|---:|---|---|
| 结束符 | $O(n)$ | 通常不可以 | 始终保证末尾结束符 |
| 显式 `length` | $O(1)$ | 可以按普通字符处理 | 修改后同步更新长度 |

实际工程结构也可同时保存长度并额外保留 `'\0'`，兼顾效率与 C API 兼容。

## 4. 堆分配存储

```c
typedef struct {
    char *data;
    int length;
    int capacity;
} HeapString;
```

初始化：

```c
int initHeapString(HeapString *s, int capacity) {
    if (s == NULL || capacity < 0) {
        return 0;
    }

    s->data = malloc((size_t)capacity + 1);

    if (s->data == NULL) {
        return 0;
    }

    s->length = 0;
    s->capacity = capacity;
    s->data[0] = '\0';
    return 1;
}
```

多申请一个字节用于结束符。

销毁：

```c
void destroyHeapString(HeapString *s) {
    if (s == NULL) return;

    free(s->data);
    s->data = NULL;
    s->length = 0;
    s->capacity = 0;
}
```

## 5. 扩容计算

若当前容量为 $c$，采用倍增：

$$
newCapacity=2c
$$

连续追加 $n$ 个字符时，虽然某次扩容要复制 $O(n)$ 个字符，但总复制量是几何级数：

$$
1+2+4+\cdots+n<2n
$$

因此尾部追加的均摊时间可达到：

$$
O(1)
$$

这是动态数组与动态字符串常用的扩容策略。

## 6. 块链存储

每个结点保存多个字符：

```c
#define BLOCK_SIZE 4

typedef struct StringBlock {
    char data[BLOCK_SIZE];
    int used;
    struct StringBlock *next;
} StringBlock;
```

示意：

```text
[d a t a] → [s t r u] → [c t _ _] → NULL
```

若每结点只保存一个字符，指针空间比例很高；使用字符块可以提高存储密度。

块链存储适合频繁分段连接的大串，但随机访问和实现复杂度较高。

## 7. 存储密度

若每个块保存 $b$ 个字符，指针占 $p$ 字节，忽略其他字段，存储密度近似：

$$
\frac{b}{b+p}
$$

$b$ 越大，指针开销占比越低，但块内未用空间也可能增加。

## 8. UTF-8 边界

C 的 `char` 表示字节，不一定表示一个用户看到的字符。

UTF-8 中一个中文字符通常占多个字节。因此：

```c
strlen("数据")
```

返回的是字节数，不一定是字符数。

本课程基础模式匹配通常按字节或单字节字符讨论。处理 Unicode 字符数、截取和大小写时，需要额外编码规则。

## 9. 选择建议

| 场景 | 结构 |
|---|---|
| 短串、容量确定 | 定长顺序存储 |
| 长度动态变化、需兼容 C API | 堆分配并保留 `'\0'` |
| 超长文本、频繁分段拼接 | 块链或更高级字符串结构 |
| 高频随机访问 | 顺序存储 |
