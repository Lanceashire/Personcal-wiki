## 条目概览

若失配后回退到的模式字符与刚刚失配的字符相同，那么它与同一个主串字符比较必然再次失败。nextval 直接跳过这种无效比较。

> [!summary]
> 若失配后回退到的模式字符与刚刚失配的字符相同，那么它与同一个主串字符比较必然再次失败。`nextval` 直接跳过这种无效比较。

## 1. 普通 `next` 的重复比较

模式：

```text
P = "aaaab"
```

0 下标普通 `next`：

```text
下标：  0  1  2  3  4
字符：  a  a  a  a  b
next： -1  0  1  2  3
```

若在 `j=3` 的字符 `'a'` 处与主串字符失配，普通回退到：

```text
j = next[3] = 2
```

但：

```text
P[3] == P[2] == 'a'
```

刚才主串字符与 `P[3]` 不相等，它与相同的 `P[2]` 也必然不相等，这次比较没有价值。

## 2. 0 下标 `nextval` 定义

在构造出候选 `k` 后：

```text
若 P[j] != P[k]，nextval[j] = k
若 P[j] == P[k]，nextval[j] = nextval[k]
```

第二种情况继续跳过相同字符链。

## 3. 构造代码

```c
void buildNextVal(
    const char *pattern,
    int m,
    int nextVal[]
) {
    if (
        pattern == NULL ||
        nextVal == NULL ||
        m <= 0
    ) {
        return;
    }

    int j = 0;
    int k = -1;
    nextVal[0] = -1;

    while (j < m - 1) {
        if (
            k == -1 ||
            pattern[j] == pattern[k]
        ) {
            j++;
            k++;

            if (pattern[j] != pattern[k]) {
                nextVal[j] = k;
            } else {
                nextVal[j] = nextVal[k];
            }
        } else {
            k = nextVal[k];
        }
    }
}
```

注意循环中 `j++` 后才访问 `pattern[j]`，因此条件必须是：

```c
j < m - 1
```

## 4. 手算 `aaaab`

普通：

```text
next0 = [-1, 0, 1, 2, 3]
```

优化：

### 位置 1

```text
P[1] == P[0] == 'a'
```

所以跳过：

```text
nextval[1] = nextval[0] = -1
```

### 位置 2

```text
P[2] == P[1]
```

```text
nextval[2] = nextval[1] = -1
```

### 位置 3

```text
P[3] == P[2]
```

```text
nextval[3] = -1
```

### 位置 4

候选 `k=3`：

```text
P[4]='b' != P[3]='a'
```

```text
nextval[4] = 3
```

结果：

```text
nextval = [-1, -1, -1, -1, 3]
```

## 5. 示例：`ababaca`

普通：

```text
P:        a  b  a  b  a  c  a
next0:   -1  0  0  1  2  3  0
```

逐项优化：

```text
nextval[0] = -1
```

位置 1：

```text
P[1]='b' != P[0]='a'
nextval[1]=0
```

位置 2：

```text
候选 P[0]='a' 与 P[2]='a' 相同
nextval[2]=nextval[0]=-1
```

位置 3：

```text
P[3]='b' 与候选 P[1]='b' 相同
nextval[3]=nextval[1]=0
```

位置 4：

```text
P[4]='a' 与候选 P[2]='a' 相同
nextval[4]=nextval[2]=-1
```

位置 5：

```text
P[5]='c' != 候选 P[3]='b'
nextval[5]=3
```

位置 6：

```text
P[6]='a' 与候选 P[0]='a' 相同
nextval[6]=nextval[0]=-1
```

所以：

```text
nextval = [-1, 0, -1, 0, -1, 3, -1]
```

## 6. `nextval` 是否改变复杂度

普通 KMP 已经是：

$$
O(n+m)
$$

`nextval` 主要减少某些模式下的无效字符比较，改善常数因素，并不改变渐近复杂度。

## 7. 何时优化明显

模式中存在大量连续或周期性重复字符时：

```text
"aaaaab"
"ababababx"
```

普通 `next` 可能连续回退到相同字符，`nextval` 能直接跨过。

## 8. 与 `pi` 版本的关系

现代代码通常直接使用 `pi` 的 `while` 回退：

```c
while (j > 0 && text[i] != pattern[j]) {
    j = pi[j - 1];
}
```

这种写法清晰、通用，通常不单独构造 `nextval`。但 `nextval` 是教材和考试的重要概念，能帮助理解“避免必然重复失败”。

## 9. 常见错误

1. 没先明确 `nextval` 是 0 下标还是 1 下标；
2. 比较错成主串字符，构造阶段只涉及模式串；
3. 相同字符时写 `nextval[j]=next[j]`，没有真正跳过；
4. 把 `nextval` 当成新的正确性条件；
5. 认为 `nextval` 把复杂度从 $O(nm)$ 降到 $O(n)$；真正完成数量级改进的是 KMP 本身。
