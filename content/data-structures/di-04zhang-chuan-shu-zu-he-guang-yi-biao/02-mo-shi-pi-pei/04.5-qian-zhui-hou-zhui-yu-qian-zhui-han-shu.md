## 条目概览

KMP 的预处理本质是：对模式串每个前缀，计算它的最长相等真前缀和真后缀长度。这个长度常记作 pi[i] 或 LPS。

> [!summary]
> KMP 的预处理本质是：对模式串每个前缀，计算它的最长相等真前缀和真后缀长度。这个长度常记作 `pi[i]` 或 LPS。

## 1. 前缀与后缀

对串：

```text
S = "ababa"
```

前缀：

```text
""
"a"
"ab"
"aba"
"abab"
"ababa"
```

后缀：

```text
""
"a"
"ba"
"aba"
"baba"
"ababa"
```

## 2. 真前缀与真后缀

真前缀不能等于整个串；真后缀也不能等于整个串。

`"ababa"` 的非空真前缀：

```text
"a", "ab", "aba", "abab"
```

非空真后缀：

```text
"a", "ba", "aba", "baba"
```

相等部分：

```text
"a", "aba"
```

最长长度：

$$
3
$$

> [!warning] 易错点
> 若允许整个串本身同时作为前缀和后缀，答案永远等于串长，对失配回退没有意义，所以必须使用“真”前缀和真后缀。

## 3. 前缀函数定义

模式串：

$$
P=P[0]P[1]\cdots P[m-1]
$$

定义：

$$
\pi[i]
$$

为子串：

$$
P[0..i]
$$

的最长相等真前缀和真后缀长度。

显然：

$$
\pi[0]=0
$$

并且：

$$
0\le \pi[i]\le i
$$

## 4. 手算示例一：`ababaca`

模式：

```text
下标：0 1 2 3 4 5 6
字符：a b a b a c a
```

逐项计算：

| `i` | 前缀 `P[0..i]` | 最长相等真前后缀 | `pi[i]` |
|---:|---|---|---:|
| 0 | `a` | 无 | 0 |
| 1 | `ab` | 无 | 0 |
| 2 | `aba` | `a` | 1 |
| 3 | `abab` | `ab` | 2 |
| 4 | `ababa` | `aba` | 3 |
| 5 | `ababac` | 无 | 0 |
| 6 | `ababaca` | `a` | 1 |

因此：

```text
pi = [0, 0, 1, 2, 3, 0, 1]
```

## 5. 手算示例二：`aaaa`

```text
P = "aaaa"
```

| 前缀 | 最长相等真前后缀 | 长度 |
|---|---|---:|
| `a` | 无 | 0 |
| `aa` | `a` | 1 |
| `aaa` | `aa` | 2 |
| `aaaa` | `aaa` | 3 |

```text
pi = [0, 1, 2, 3]
```

重复字符越多，可复用的前后缀通常越长。

## 6. 递推计算

已知：

```text
pi[0 .. i-1]
```

计算 `pi[i]` 时，先尝试延长上一位置的最长边界：

```c
int k = pi[i - 1];
```

比较：

```c
pattern[i] 与 pattern[k]
```

若不同，最长候选边界不能直接延长，但候选边界本身还可能有更短边界：

```c
k = pi[k - 1];
```

不断回退，直到：

- 找到可延长的字符；
- 或 `k == 0`。

## 7. 前缀函数代码

```c
void buildPrefixFunction(
    const char *pattern,
    int m,
    int pi[]
) {
    if (
        pattern == NULL ||
        pi == NULL ||
        m <= 0
    ) {
        return;
    }

    pi[0] = 0;

    for (int i = 1; i < m; i++) {
        int k = pi[i - 1];

        while (
            k > 0 &&
            pattern[i] != pattern[k]
        ) {
            k = pi[k - 1];
        }

        if (pattern[i] == pattern[k]) {
            k++;
        }

        pi[i] = k;
    }
}
```

## 8. 逐步计算 `ababaca`

### `i = 4`

此前：

```text
pi[3] = 2
```

候选长度 `k = 2`，比较：

```text
P[4] = 'a'
P[2] = 'a'
```

相等，延长：

```text
pi[4] = 3
```

### `i = 5`

此前：

```text
k = pi[4] = 3
```

比较：

```text
P[5] = 'c'
P[3] = 'b'
```

失配，回退：

```text
k = pi[2] = 1
```

再比较：

```text
P[5] = 'c'
P[1] = 'b'
```

仍失配，再回退：

```text
k = pi[0] = 0
```

比较：

```text
P[5] = 'c'
P[0] = 'a'
```

失配：

```text
pi[5] = 0
```

## 9. 为什么可以沿 `pi` 回退

若长度为 `k` 的前缀等于当前后缀，但无法被新字符延长，那么新的候选必须是这个长度为 `k` 的前缀自身的某个边界。

最长的更短候选正是：

```text
pi[k - 1]
```

因此无需枚举所有长度。

## 10. 复杂度

看起来 `while` 嵌套在 `for` 中，但 `k` 的增长和回退总次数是线性的。每个字符不会导致无限回退。

构造时间：

$$
O(m)
$$

空间：

$$
O(m)
$$

## 11. 术语对应

| 名称 | 常见含义 |
|---|---|
| `pi` | prefix function，当前前缀的最长边界长度 |
| LPS | longest proper prefix which is also suffix |
| failure function | 失配后应保留的匹配信息 |
| border | 同时为真前缀和真后缀的串 |

不同资料可能数组定义不同，必须先读定义再比较数值。
