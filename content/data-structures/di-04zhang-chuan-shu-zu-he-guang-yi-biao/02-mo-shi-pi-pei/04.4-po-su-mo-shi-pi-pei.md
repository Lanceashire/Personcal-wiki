## 条目概览

朴素模式匹配从主串每个可能起点尝试与模式串逐字符比较。失配后，主串候选起点右移一位，模式串从头重新比较。

## 1. 问题定义

主串：

```text
T = "ababcabcacbab"
```

模式串：

```text
P = "abcac"
```

寻找 `P` 第一次完整出现在 `T` 中的起始下标；不存在返回 `-1`。

设：

$$
n=|T|,\quad m=|P|
$$

合法候选起点共有：

$$
n-m+1
$$

个。

## 2. 双下标过程

- `i` 指向主串当前比较字符；
- `j` 指向模式串当前比较字符。

字符相同：

```c
i++;
j++;
```

失配时，若本轮起点为：

$$
s=i-j
$$

下一轮起点为：

$$
s+1
$$

所以：

```c
i = i - j + 1;
j = 0;
```

## 3. C 语言实现

```c
int bruteForceSearch(
    const char *text,
    int n,
    const char *pattern,
    int m
) {
    if (
        text == NULL ||
        pattern == NULL ||
        n < 0 ||
        m < 0
    ) {
        return -1;
    }

    if (m == 0) {
        return 0;
    }

    int i = 0;
    int j = 0;

    while (i < n && j < m) {
        if (text[i] == pattern[j]) {
            i++;
            j++;
        } else {
            i = i - j + 1;
            j = 0;
        }
    }

    return j == m ? i - m : -1;
}
```

## 4. 为什么匹配成功位置是 `i - m`

匹配完成时：

```text
j == m
```

`i` 已经指向匹配片段后一个位置。

因此起点：

$$
i-m
$$

## 5. 逐轮示例

```text
T = a a a a b
P = a a a b
```

第一轮起点 `0`：

```text
a a a a b
a a a b
      ↑ 第4个字符失配
```

比较 4 次。

第二轮起点 `1`：

```text
a a a a b
  a a a b
```

比较 4 次，成功。

总比较次数：

$$
8
$$

## 6. 最坏情况计算

若每个候选起点都比较到模式末尾才失配，每轮最多比较 $m$ 次，起点最多 $n-m+1$ 个：

$$
T(n,m)=O((n-m+1)m)
$$

常简写为：

$$
O(nm)
$$

例如：

```text
T = "aaaaaaaaab"
P = "aaaab"
```

大量已匹配的 `a` 会被重复比较。

## 7. 最好情况

若每个候选起点第一个字符就失配：

$$
O(n)
$$

若很早匹配成功，实际比较次数可能更少。

## 8. 朴素算法浪费在哪里

失配前已经知道：

```text
text[s ... s+j-1] == pattern[0 ... j-1]
```

但朴素算法把这段信息全部丢弃，下一轮又从模式首字符开始比较。

KMP 的核心问题是：

> 已匹配前缀内部是否存在一个后缀，可以继续充当模式串前缀？

## 9. 空模式约定

很多字符串 API 约定空模式在任意串中的第一次出现位置为 `0`：

```text
indexOf("abc", "") = 0
```

具体考试题应服从题目或教材接口定义。
