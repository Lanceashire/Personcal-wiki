## 条目概览

KMP 失配时不回退主串指针，而是利用模式串已匹配部分的最长相等前后缀，把模式串移动到仍可能成功的位置。

> [!summary]
> KMP 失配时不回退主串指针，而是利用模式串已匹配部分的最长相等前后缀，把模式串移动到仍可能成功的位置。

## 1. 已匹配信息

假设已经匹配了模式前 `j` 个字符：

```text
text[i-j ... i-1] == pattern[0 ... j-1]
```

下一字符失配：

```text
text[i] != pattern[j]
```

朴素算法会让候选起点右移一位，并重新比较大量字符。

KMP 问：

> 已匹配段的后缀中，是否有一部分等于模式前缀？

若有，就可以保留这部分匹配。

## 2. 回退公式

已匹配长度为 `j`，最长可复用前后缀长度是：

```c
pi[j - 1]
```

失配后：

```c
j = pi[j - 1];
```

主串下标 `i` 不变，再比较：

```text
text[i] 与 pattern[j]
```

## 3. 图解

模式已匹配部分：

```text
P[0 ... j-1] = "abab"
```

它的最长相等真前后缀：

```text
"ab"
```

若下一字符失配：

```text
主串：... a b a b X
模式：    a b a b Y
```

已知末尾 `"ab"` 与模式开头 `"ab"` 相等，可以移动成：

```text
主串：... a b a b X
模式：        a b a b Y
              ↑
         前两个字符无需重比
```

新匹配长度：

$$
j=2
$$

## 4. 前缀函数版搜索

```c
int kmpSearch(
    const char *text,
    int n,
    const char *pattern,
    int m,
    const int pi[]
) {
    if (
        text == NULL ||
        pattern == NULL ||
        pi == NULL ||
        n < 0 ||
        m < 0
    ) {
        return -1;
    }

    if (m == 0) {
        return 0;
    }

    int j = 0;

    for (int i = 0; i < n; i++) {
        while (
            j > 0 &&
            text[i] != pattern[j]
        ) {
            j = pi[j - 1];
        }

        if (text[i] == pattern[j]) {
            j++;
        }

        if (j == m) {
            return i - m + 1;
        }
    }

    return -1;
}
```

## 5. 为什么主串 `i` 不回退

失配前已经确认主串的一段等于模式前缀。回退 `j` 是在寻找与这段主串后缀一致的更短模式前缀。

当前主串字符 `text[i]` 尚未成功匹配，所以保持 `i` 不变，继续尝试更短候选。

在 `for` 版本中，同一个 `i` 的多次尝试发生在内部 `while` 中；结束后外层才进入 `i+1`。

## 6. 完整匹配示例

```text
T = "abababca"
P = "ababca"
```

模式前缀函数：

```text
P : a b a b c a
pi: 0 0 1 2 0 1
```

先匹配：

```text
T[0..3] = "abab"
P[0..3] = "abab"
```

接下来：

```text
T[4] = 'a'
P[4] = 'c'
```

失配，原 `j=4`：

```c
j = pi[3] = 2;
```

保持 `T[4]`，比较：

```text
T[4] = 'a'
P[2] = 'a'
```

成功，继续比较，最终在下标 `2` 匹配：

```text
T: a b a b a b c a
P:     a b a b c a
       ↑ 起点2
```

## 7. 查找全部出现位置

若不在首次成功时返回，而是记录位置：

```c
position = i - m + 1;
```

为了寻找可能重叠的下一次匹配，成功后：

```c
j = pi[m - 1];
```

例如：

```text
T = "aaaaa"
P = "aaa"
```

出现位置：

```text
0, 1, 2
```

若成功后直接 `j=0`，可能漏掉重叠匹配。

## 8. KMP 不代表永不重复比较主串字符

某个主串字符在内部 `while` 中可能与多个模式字符比较，但主串指针不会向左回退。

总比较仍是线性的，因为 `j` 的前进和回退总量受限。

## 9. KMP 的不变量

处理 `text[i]` 前，`j` 表示：

> `text[0..i-1]` 的某个后缀与 `pattern[0..j-1]` 相等，并且 `j` 是当前保留的匹配长度。

失配回退只改变 `j`，但仍保持这一关系。

## 10. 复杂度

预处理模式：

$$
O(m)
$$

扫描主串：

$$
O(n)
$$

总时间：

$$
O(n+m)
$$

辅助数组：

$$
O(m)
$$

## 11. 常见错误

1. 失配时写 `j = pi[j]`，可能越界或不缩小；
2. `j == 0` 时访问 `pi[j-1]`；
3. 成功位置写成 `i-m` 而不是 `i-m+1`；
4. 空模式未处理；
5. 查找全部匹配时成功后直接清零，漏掉重叠；
6. 把主串和模式串的预处理对象弄反。
