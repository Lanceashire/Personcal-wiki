## 条目概览

不同资料中的 next 数值可能不同，原因通常是下标从 0 还是 1 开始，以及数组表示“已匹配长度”还是“失配后模式下标”。先读定义，再做换算。

> 不同资料中的 `next` 数值可能不同，原因通常是下标从 0 还是 1 开始，以及数组表示“已匹配长度”还是“失配后模式下标”。先读定义，再做换算。

## 1. 三套常见表示

本笔记统一列出：

### A. 前缀函数 `pi`

```text
pi[i] = P[0..i] 的最长相等真前后缀长度
```

### B. 0 下标 `next0`

```text
next0[0] = -1
next0[j] = pi[j-1]，j >= 1
```

它表示模式在下标 `j` 失配后，新的模式下标。

### C. 教材常见 1 下标 `next1`

```text
next1[1] = 0
```

位置从 `1` 开始，失配后令：

```text
j = next1[j]
```

## 2. 三者换算

对 0 下标模式位置 `j`：

$$
next0[j]=
\begin{cases}
-1,&j=0\\
\pi[j-1],&j\ge1
\end{cases}
$$

1 下标位置 $q=j+1$：

$$
next1[q]=next0[j]+1
$$

因此首项：

```text
next0[0] = -1
next1[1] = 0
```

只是哨兵表示不同。

## 3. 示例：`ababaca`

```text
0下标字符： a b a b a c a
0下标位置： 0 1 2 3 4 5 6
pi：        0 0 1 2 3 0 1
next0：    -1 0 0 1 2 3 0
```

对应 1 下标：

```text
1下标位置： 1 2 3 4 5 6 7
字符：      a b a b a c a
next1：     0 1 1 2 3 4 1
```

注意：

```text
pi[i]
```

描述以 `i` 结尾的前缀；而：

```text
next0[j]
```

描述在 `j` 位置失配前，前面 `j` 个已匹配字符可保留多少。

二者天然错开一位。

## 4. 0 下标 `next0` 构造代码

```c
void buildNext0(
    const char *pattern,
    int m,
    int next[]
) {
    if (
        pattern == NULL ||
        next == NULL ||
        m <= 0
    ) {
        return;
    }

    int j = 0;
    int k = -1;
    next[0] = -1;

    while (j < m - 1) {
        if (
            k == -1 ||
            pattern[j] == pattern[k]
        ) {
            j++;
            k++;
            next[j] = k;
        } else {
            k = next[k];
        }
    }
}
```

## 5. 代码变量含义

在构造过程中：

- `j`：正在计算 `next[j+1]` 所需的已知位置；
- `k`：当前候选相等前后缀长度，也可看作候选模式下标；
- `next[k]`：当前候选失配后，继续尝试更短候选。

当：

```c
k == -1
```

表示已经没有可继续尝试的非空候选，从模式开头重新建立。

## 6. 手算方法一：直接看前缀

计算：

```text
P = "ABABC"
```

先求 `pi`：

| `i` | 前缀 | 最长边界 | `pi[i]` |
|---:|---|---|---:|
| 0 | `A` | 无 | 0 |
| 1 | `AB` | 无 | 0 |
| 2 | `ABA` | `A` | 1 |
| 3 | `ABAB` | `AB` | 2 |
| 4 | `ABABC` | 无 | 0 |

```text
pi = [0, 0, 1, 2, 0]
```

转换：

```text
next0[0] = -1
next0[1] = pi[0] = 0
next0[2] = pi[1] = 0
next0[3] = pi[2] = 1
next0[4] = pi[3] = 2
```

所以：

```text
next0 = [-1, 0, 0, 1, 2]
```

1 下标：

```text
next1 = [0, 1, 1, 2, 3]
```

## 7. 手算方法二：逐格理解

`next0[j]` 只看：

```text
P[0 .. j-1]
```

即失配位置 `j` 之前已经匹配的部分。

例如 `P = "ABABC"`，计算 `next0[4]`：

```text
已匹配部分 P[0..3] = "ABAB"
```

最长相等真前后缀：

```text
"AB"
```

长度为 `2`：

```text
next0[4] = 2
```

不要把失配位置字符 `P[4]='C'` 也包含进去。

> [!warning] 易错点
> `pi[4]` 看 `"ABABC"`，而 `next0[4]` 看失配前的 `"ABAB"`。这就是二者错一位的根本原因。

## 8. 0 下标 `next` 搜索代码

```c
int kmpSearchWithNext(
    const char *text,
    int n,
    const char *pattern,
    int m,
    const int next[]
) {
    if (m == 0) return 0;

    int i = 0;
    int j = 0;

    while (i < n && j < m) {
        if (
            j == -1 ||
            text[i] == pattern[j]
        ) {
            i++;
            j++;
        } else {
            j = next[j];
        }
    }

    return j == m ? i - j : -1;
}
```

当 `j == -1`：

```text
模式无法继续回退
→ 主串前进一位
→ 模式回到0
```

由：

```c
i++;
j++;
```

共同完成。

## 9. 教材 1 下标构造框架

若模式位置为 `1..m`：

```c
next[1] = 0;
j = 1;
k = 0;

while (j < m) {
    if (k == 0 || pattern[j] == pattern[k]) {
        j++;
        k++;
        next[j] = k;
    } else {
        k = next[k];
    }
}
```

这里是假想数组支持从 `1` 开始。C 实现时往往需要额外偏移，不建议不加说明直接照抄。

## 10. 考试遇到 `next` 题的步骤

1. 查看模式位置从 `0` 还是 `1` 开始；
2. 查看首项是 `-1`、`0` 还是 `1`；
3. 看失配语句是 `j=next[j]` 还是 `j=pi[j-1]`；
4. 先写该资料的定义；
5. 再计算，禁止直接套另一套表。

## 11. 常见错误

1. 把 `pi` 原样当作 `next0`；
2. 计算 `next0[j]` 时把 `P[j]` 包含进去；
3. 0 下标和 1 下标混用；
4. 首项哨兵值与搜索代码不配套；
5. 构造时失配写 `k=0`，丢弃可复用的更短边界；
6. 只背某个模式的数组，不会从定义重算。
