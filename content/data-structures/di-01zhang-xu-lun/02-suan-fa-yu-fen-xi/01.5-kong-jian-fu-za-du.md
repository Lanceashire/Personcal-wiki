## 条目概览

空间复杂度描述输入规模增大时，算法额外占用空间的增长趋势。分析时通常重点关注辅助空间，而不是输入数据本身已经占用的空间。

> [!summary]
> 空间复杂度描述输入规模增大时，算法额外占用空间的增长趋势。分析时通常重点关注辅助空间，而不是输入数据本身已经占用的空间。

## 1. 空间由哪些部分构成

程序运行时可能需要：

1. 程序代码空间；
2. 输入数据空间；
3. 常量和局部变量；
4. 动态申请的辅助空间；
5. 函数调用栈空间。

复杂度分析通常重点研究随输入规模 $n$ 变化的额外空间。

## 2. 固定数量变量

```c
int sumArray(const int arr[], int n) {
    int sum = 0;

    for (int i = 0; i < n; i++) {
        sum += arr[i];
    }

    return sum;
}
```

除输入外，只使用 `sum` 和 `i` 等固定数量变量：

$$
S(n)=O(1)
$$

## 3. 辅助数组

```c
int *temp = malloc(n * sizeof(int));
```

辅助数组长度随 $n$ 线性增长：

$$
S(n)=O(n)
$$

典型例子是数组版归并排序。

## 4. 二维辅助空间

若创建 $n\times n$ 的矩阵：

```c
int matrix[n][n];
```

则额外空间为：

$$
S(n)=O(n^2)
$$

## 5. 递归调用栈

递归函数每调用一层，通常需要保存：

- 参数；
- 局部变量；
- 返回地址；
- 必要的寄存器状态。

阶乘递归：

```c
int factorial(int n) {
    if (n <= 1) {
        return 1;
    }

    return n * factorial(n - 1);
}
```

调用链：

```text
factorial(4)
→ factorial(3)
→ factorial(2)
→ factorial(1)
```

递归深度为 $O(n)$，每层栈帧占用常数空间，因此：

$$
S(n)=O(n)
$$

> [!warning] 对话中暴露的易错点
> 递归函数即使没有显式创建数组，也可能因为调用栈而使用 $O(n)$ 或 $O(\log n)$ 的额外空间。

## 6. 时间与空间的权衡

有些算法使用更多空间来换取更快速度。

例如散列表使用额外表空间，使精确查找平均可以接近：

$$
O(1)
$$

有些算法则选择原地修改以降低辅助空间，但代码可能更复杂。

| 方案 | 时间 | 辅助空间 |
|---|---:|---:|
| 数组归并排序 | $O(n\log n)$ | $O(n)$ |
| 堆排序 | $O(n\log n)$ | $O(1)$ |

不能只看一个指标判断算法绝对优劣。

## 7. 原地算法

若算法使用的额外空间不随输入规模增长，通常称为原地算法：

$$
S(n)=O(1)
$$

原地不表示完全不使用变量，而是只使用固定数量的辅助变量。

## 8. 常见判断方法

1. 列出除输入外新创建的数据结构；
2. 判断其规模是否随 $n$ 增长；
3. 若存在递归，计算最大递归深度；
4. 不要把不同时间先后使用且可复用的空间盲目相加；
5. 取运行过程中同时占用空间的峰值。
