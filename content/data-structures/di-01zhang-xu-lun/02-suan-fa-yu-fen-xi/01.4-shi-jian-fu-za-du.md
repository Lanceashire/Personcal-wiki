## 条目概览

时间复杂度描述输入规模 $n$ 增大时，算法基本操作次数的增长趋势。它不是程序运行的精确秒数。

> [!summary]
> 时间复杂度描述输入规模 $n$ 增大时，算法基本操作次数的增长趋势。它不是程序运行的精确秒数。

## 1. 基本模型

设算法对规模为 $n$ 的输入执行基本操作的次数为：

$$
T(n)
$$

若存在正常数 $c$ 和 $n_0$，使得当 $n\ge n_0$ 时：

$$
T(n)\le c\cdot f(n)
$$

则记作：

$$
T(n)=O(f(n))
$$

大 $O$ 表示渐近上界，复习阶段通常用它描述算法随数据规模增长的数量级。

## 2. 化简规则

### 2.1 忽略常数项

$$
T(n)=1000n+500
$$

化简为：

$$
O(n)
$$

### 2.2 忽略低阶项

$$
T(n)=8n^2+100n+9
$$

最高阶项为 $n^2$，因此：

$$
T(n)=O(n^2)
$$

### 2.3 忽略最高阶项的常数系数

$$
8n^2=O(n^2)
$$

> [!warning] 易错点
> 忽略常数不表示常数在真实运行中完全没有影响，而是表示渐近复杂度主要研究规模增大后的增长趋势。

## 3. 常见数量级

从增长较慢到较快：

$$
O(1)<O(\log n)<O(n)<O(n\log n)<O(n^2)<O(n^3)<O(2^n)<O(n!)
$$

当 $n$ 很大时，数量级的差异通常比常数优化更关键。

## 4. 常见循环分析

### 4.1 固定步长循环

```c
for (int i = 0; i < n; i++) {
    operation();
}
```

循环约执行 $n$ 次：

$$
T(n)=O(n)
$$

### 4.2 倍增循环

```c
for (int i = 1; i <= n; i *= 2) {
    operation();
}
```

若执行 $k$ 次后结束，则：

$$
2^k>n
$$

所以：

$$
k=O(\log_2 n)
$$

因此时间复杂度为：

$$
O(\log n)
$$

复杂度中通常不强调对数底数，因为换底只会产生常数因子。

### 4.3 独立嵌套循环

```c
for (int i = 0; i < n; i++) {
    for (int j = 0; j < n; j++) {
        operation();
    }
}
```

总次数：

$$
n\times n=n^2
$$

复杂度：

$$
O(n^2)
$$

### 4.4 等差求和型循环

```c
for (int i = 1; i <= n; i++) {
    for (int j = 0; j < i; j++) {
        operation();
    }
}
```

总次数：

$$
1+2+\cdots+n=\frac{n(n+1)}{2}
$$

复杂度：

$$
O(n^2)
$$

### 4.5 等比求和型循环

```c
for (int i = 1; i <= n; i *= 2) {
    for (int j = 0; j < i; j++) {
        operation();
    }
}
```

总次数近似：

$$
1+2+4+\cdots+n<2n
$$

所以不是 $O(n\log n)$，而是：

$$
O(n)
$$

> [!warning] 对话中暴露的易错点
> 看到两层循环不能直接断定为 $O(n^2)$。必须计算内层在每次外层循环中实际执行多少次，再求总和。

## 5. 顺序语句与分支语句

### 5.1 顺序执行

若两段代码复杂度分别为 $O(f(n))$ 和 $O(g(n))$，顺序执行后取增长较快者：

$$
O(f(n)+g(n))=O(\max(f(n),g(n)))
$$

### 5.2 条件分支

最坏时间复杂度通常取成本最大的分支，再加上判断条件本身的成本。

```c
if (condition) {
    linearWork(n);
} else {
    constantWork();
}
```

最坏情况为：

$$
O(n)
$$

## 6. 最好、最坏与平均情况

以顺序查找为例：

| 情况 | 比较次数 | 复杂度 |
|---|---:|---:|
| 最好 | 第一个元素即命中 | $O(1)$ |
| 最坏 | 最后命中或查找失败 | $O(n)$ |
| 平均 | 取决于输入分布 | $O(n)$ |

没有特别说明时，题目常关注最坏时间复杂度，但必须结合题意判断。

## 7. 递归时间复杂度

递归算法通常通过递推式分析。

例如：

```c
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}
```

递推式：

$$
T(n)=T(n-1)+O(1)
$$

展开后：

$$
T(n)=O(n)
$$

## 8. 练习

判断下列代码的复杂度：

```c
for (int i = n; i > 0; i /= 2) {
    for (int j = 0; j < n; j++) {
        operation();
    }
}
```

外层执行 $O(\log n)$ 次，每次内层执行 $O(n)$ 次，因此：

$$
O(n\log n)
$$
