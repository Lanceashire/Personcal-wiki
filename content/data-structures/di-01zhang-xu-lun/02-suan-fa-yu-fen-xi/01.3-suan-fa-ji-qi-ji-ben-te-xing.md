## 条目概览

算法是对特定问题求解步骤的有限、明确描述。算法必须有穷、确定、可行，可以有零个或多个输入，并至少产生一个输出。

## 1. 算法是什么

算法描述从输入到输出的求解过程。

例如，在数组中寻找最大值：

```c
int maxValue(const int arr[], int n) {
    int max = arr[0];

    for (int i = 1; i < n; i++) {
        if (arr[i] > max) {
            max = arr[i];
        }
    }

    return max;
}
```

输入是数组与长度，输出是最大值，中间步骤明确且能够在有限时间内完成。

## 2. 五个基本特性

| 特性 | 含义 |
|---|---|
| 有穷性 | 执行有限步骤后必须结束 |
| 确定性 | 每一步的含义明确，在相同条件下不会产生歧义 |
| 可行性 | 每一步都能通过有限次基本操作实现 |
| 输入 | 可以有零个或多个输入 |
| 输出 | 至少有一个输出 |

记忆提示：

```text
有、确、可、入、出
```

## 3. 基本特性与评价标准

算法“能够成立”和算法“质量较高”是两个问题。

| 类别 | 内容 |
|---|---|
| 基本特性 | 有穷性、确定性、可行性、输入、输出 |
| 评价标准 | 正确性、可读性、健壮性、时间效率、空间效率 |

> [!warning] 对话中暴露的易错点
> 可读性、健壮性和高效率很重要，但它们不是算法定义中的五个基本特性。

## 4. 算法与程序

算法是一种求解思想和步骤描述，程序是算法使用某种编程语言的具体实现。

同一个算法可以分别用 C、C++、Java 或 Python 实现。程序还需要处理：

- 数据类型；
- 内存申请与释放；
- 输入输出格式；
- 编译器和运行环境；
- 异常与边界条件。

## 5. 正确性层次

分析算法正确性时，至少应检查：

1. 对典型合法输入能得到正确结果；
2. 对边界输入能正确处理；
3. 对所有满足前提的输入均能终止并得到正确结果；
4. 对不合法输入有合理的防护策略。

例如数组最大值算法要求：

```text
n > 0
arr != NULL
```

如果没有这个前提，直接读取 `arr[0]` 可能产生未定义行为。
