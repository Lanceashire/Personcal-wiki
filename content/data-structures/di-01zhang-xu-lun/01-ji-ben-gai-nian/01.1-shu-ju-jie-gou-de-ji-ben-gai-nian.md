## 条目概览

数据结构研究数据元素之间的关系、这些关系在计算机中的表示，以及作用在数据上的操作。它不只是“把数据存起来”。

## 1. 概念概览

学习数据结构时，需要从“单个值”上升到“有组织的数据整体”。

例如，一张学生表不只是若干姓名和成绩，还包含：

1. 每条记录由哪些数据项构成；
2. 记录之间是否存在先后、层级或网络关系；
3. 数据在内存中如何存放；
4. 如何执行查找、插入、删除和排序。

## 2. 基本术语

| 术语 | 含义 | 示例 |
|---|---|---|
| 数据 | 能被计算机识别、存储和处理的信息载体 | 数字、字符、图像、声音 |
| 数据元素 | 数据的基本单位，通常作为整体处理 | 一名学生的一条记录 |
| 数据项 | 构成数据元素的不可再分的最小单位 | 学号、姓名、成绩 |
| 数据对象 | 性质相同的数据元素的集合 | 全部学生记录 |
| 数据结构 | 相互之间存在一种或多种特定关系的数据元素集合 | 线性表、树、图 |

一条学生记录可以写成：

```c
typedef struct {
    int id;
    char name[20];
    float score;
} Student;
```

其中：

- `Student` 变量可以看作一个数据元素；
- `id`、`name`、`score` 是数据项；
- 多个 `Student` 构成学生数据对象。

## 3. 数据结构的三个方面

数据结构通常从三个方面研究：

### 3.1 逻辑结构

描述数据元素之间的抽象关系，不考虑具体存储地址。

```text
A → B → C
```

只说明存在一对一的前后关系。

### 3.2 存储结构

也称物理结构，描述逻辑关系如何在计算机中表示。

同一个线性关系既可以使用连续数组保存，也可以使用指针连接结点。

### 3.3 数据运算

指作用在数据结构上的操作，例如：

- 创建与销毁；
- 查找；
- 插入；
- 删除；
- 遍历；
- 排序；
- 修改。

> [!warning] 对话中暴露的易错点
> “数据结构就是数据加存储结构”不够完整。更准确的理解是：数据元素之间的逻辑关系、这种关系的存储表示，以及定义在其上的基本运算。

## 4. 抽象数据类型

抽象数据类型（ADT）强调：

> 先说明“有什么数据、允许做什么”，暂时不限定“具体怎样实现”。

例如“栈”规定：

```text
只能在栈顶插入和删除
后进先出
```

但它既可以由数组实现，也可以由链表实现。

```text
栈的逻辑规则
├── 顺序栈：数组实现
└── 链栈：链表实现
```

这说明逻辑结构和具体实现可以分离。

## 5. 例题

### 例 1：文件系统属于什么结构？

文件夹中可以包含多个子文件夹或文件，形成一对多的层级关系，因此逻辑上属于树形结构。

### 例 2：社交网络属于什么结构？

一个用户可以与多个用户相连，用户之间是多对多关系，因此逻辑上属于图状结构。

### 例 3：数组是否等于线性表？

不完全等同：

- 线性表是一种逻辑结构；
- 数组是常见的顺序存储工具；
- 线性表可以用数组实现为顺序表，也可以用链表实现。

## 6. 易错点与边界情况

1. 数据元素不一定是一个基本类型，也可以是一整条结构体记录。
2. 数据项是组成数据元素的更小单位。
3. 逻辑结构不讨论内存地址是否连续。
4. 同一种逻辑结构可以有多种存储实现。
5. 同一种存储工具也可以服务于不同问题，不能只凭代码中出现数组就判断逻辑结构。
