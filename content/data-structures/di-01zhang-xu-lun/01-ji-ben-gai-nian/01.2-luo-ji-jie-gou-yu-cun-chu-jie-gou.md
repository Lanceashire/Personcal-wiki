逻辑结构回答“元素之间是什么关系”，存储结构回答“这种关系如何放入计算机”。二者不能混为一谈。

## 1. 四类逻辑结构

| 逻辑结构 | 元素关系 | 典型例子 |
|---|---|---|
| 集合结构 | 除同属一个集合外，没有明确关系 | 一个班级中的学生 |
| 线性结构 | 一对一的前后关系 | 线性表、栈、队列 |
| 树形结构 | 一对多的层级关系 | 文件系统、二叉树 |
| 图状结构 | 多对多的连接关系 | 道路网、社交网络 |

### 1.1 线性结构的边界

除第一个元素外，每个元素通常只有一个直接前驱；除最后一个元素外，每个元素通常只有一个直接后继。

```text
A → B → C → D
```

### 1.2 树形结构与图状结构

树强调层级与父子关系，图允许更一般的连接。

```text
树：                 图：
   A                A ─ B
   / \               │ ╲ │
  B   C              C ─ D
```

## 2. 常见存储结构

### 2.1 顺序存储

使用连续存储单元依次保存数据元素。

```text
[12][25][37][41]
```

特点：

- 支持按下标快速随机访问；
- 存储密度通常较高；
- 中间插入、删除往往需要移动元素；
- 固定数组可能存在容量限制。

### 2.2 链式存储

元素可以分散存放，通过指针表示逻辑关系。

```text
[12 | next] → [25 | next] → [37 | NULL]
```

特点：

- 不要求结点地址连续；
- 插入删除主要修改链接；
- 需要额外指针空间；
- 按位置访问通常要从头遍历。

### 2.3 索引存储

在数据之外建立辅助索引表。

```text
索引项 → 某一块数据的地址
```

它类似书籍目录：先查目录，再定位正文。

### 2.4 散列存储

通过散列函数将关键字映射到存储位置：

$$
address = H(key)
$$

例如：

$$
H(key)=key\bmod 10
$$

不同关键字可能映射到同一位置，这称为散列冲突。

## 3. 顺序存储与链式存储比较

| 比较项 | 顺序存储 | 链式存储 |
|---|---|---|
| 内存位置 | 通常连续 | 可以分散 |
| 按位置访问 | $O(1)$ | 通常 $O(n)$ |
| 已知前驱后的插入 | 可能移动元素 | 修改指针可为 $O(1)$ |
| 额外空间 | 通常较少 | 需要指针域 |
| 缓存局部性 | 较好 | 通常较弱 |
| 容量变化 | 固定数组扩容较麻烦 | 动态申请较灵活 |

> [!warning] 对话中暴露的易错点
> “数组查找一定比链表快”是错误的笼统说法。数组按下标访问是 $O(1)$，但在无序数组中按值查找仍可能是 $O(n)$。

## 4. 为什么数组能随机访问

假设数组首地址为 $base$，每个元素占 $w$ 个字节，则下标为 $i$ 的元素地址是：

$$
LOC(a_i)=base+i\times w
$$

计算地址只需要固定次数的算术运算，因此按下标访问为：

$$
O(1)
$$

链表结点地址没有这种统一公式，只能沿 `next` 逐个前进。

## 5. 如何选择存储结构

选择时应考虑：

1. 是否频繁按下标访问；
2. 是否频繁在中间插入和删除；
3. 数据规模是否容易变化；
4. 是否需要范围查询或精确查询；
5. 是否能接受额外索引或指针空间；
6. 是否要求元素保持有序。

不存在适合所有场景的唯一最佳结构。

## 6. 例题

### 例 1：播放列表需要频繁跳到第 500 首

若主要按序号访问，顺序存储更直接。

### 例 2：任务列表频繁在已知任务后插入新任务

若已经持有目标结点指针，链式存储可以通过修改指针完成插入。

### 例 3：根据学号频繁精确查询学生

若无需保持顺序，可考虑散列表；若需要范围查询和有序遍历，树形索引可能更合适。
