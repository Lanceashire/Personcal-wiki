## 条目概览

归并排序先拆分，再合并两个有序区间。时间复杂度稳定为 $O(n\log n)$，而且稳定，但数组版本需要额外空间。

> [!summary]
> 归并排序先拆分，再合并两个有序区间。时间复杂度稳定为 $O(n\log n)$，而且稳定，但数组版本需要额外空间。

## 一、过程

```text
5  2  4  1
→ [5  2] [4  1]
→ [5] [2] [4] [1]
→ [2  5] [1  4]
→ [1  2  4  5]
```

## 二、代码

```c
#include <stdlib.h>

void merge(int arr[], int left, int mid, int right) {
    int len = right - left + 1;
    int *temp = malloc(len * sizeof(int));
    if (temp == NULL) return;

    int i = left, j = mid + 1, k = 0;

    while (i <= mid && j <= right) {
        if (arr[i] <= arr[j]) temp[k++] = arr[i++];
        else temp[k++] = arr[j++];
    }

    while (i <= mid) temp[k++] = arr[i++];
    while (j <= right) temp[k++] = arr[j++];

    for (int p = 0; p < len; p++) arr[left + p] = temp[p];
    free(temp);
}

void mergeSort(int arr[], int left, int right) {
    if (left >= right) return;
    int mid = left + (right - left) / 2;
    mergeSort(arr, left, mid);
    mergeSort(arr, mid + 1, right);
    merge(arr, left, mid, right);
}
```

## 三、复杂度

| 项目 | 结论 |
|---|---:|
| 最好、平均、最坏时间 | $O(n\log n)$ |
| 空间复杂度 | $O(n)$ |
| 稳定性 | 稳定 |

## 四、为什么稳定？

合并时使用：

```c
arr[i] <= arr[j]
```

相等时优先取左侧元素，保留原本相对顺序。

## 五、为什么适合外部排序？

归并主要进行顺序读取和顺序写出，适合磁盘数据。

## 七、合并过程详细示例

左侧：

```text
2  5  8
```

右侧：

```text
1  4  7
```

过程：

| 比较 | 取出 | 临时数组 |
|---|---:|---|
| `2` 与 `1` | `1` | `1` |
| `2` 与 `4` | `2` | `1 2` |
| `5` 与 `4` | `4` | `1 2 4` |
| `5` 与 `7` | `5` | `1 2 4 5` |
| `8` 与 `7` | `7` | `1 2 4 5 7` |
| 左侧剩余 | `8` | `1 2 4 5 7 8` |

## 八、为什么合并时相等要优先取左边？

左侧元素原本位于右侧元素之前。

相等时优先取左侧：

```c
arr[i] <= arr[j]
```

可以保留原相对顺序。

如果写成：

```c
arr[i] < arr[j]
```

相等时右侧先进入结果，可能破坏稳定性。

## 九、边缘情况

- 空数组：无需排序。
- 单元素区间：递归终止。
- 一侧提前耗尽：复制另一侧剩余元素。
- 动态内存申请失败：需要处理。
- 链表版本：可通过指针连接，减少复制。
