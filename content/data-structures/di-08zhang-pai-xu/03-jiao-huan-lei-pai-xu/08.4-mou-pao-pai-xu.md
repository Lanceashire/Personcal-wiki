## 条目概览

冒泡排序通过相邻比较和交换，使较大的元素逐渐移动到数组末尾。

> [!summary]
> 冒泡排序通过相邻比较和交换，使较大的元素逐渐移动到数组末尾。

上级知识点：[[排序总览]]

## 一、过程

```text
原始：5  2  4  1
第一趟：
5 和 2 交换 → 2  5  4  1
5 和 4 交换 → 2  4  5  1
5 和 1 交换 → 2  4  1  5
```

一趟结束后，最大值进入最终位置。

## 二、代码

```c
void bubbleSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        int swapped = 0;

        for (int j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
                swapped = 1;
            }
        }

        if (!swapped) break;
    }
}
```

## 三、复杂度

| 情况 | 时间复杂度 |
|---|---:|
| 最好：已有序且有提前退出 | $O(n)$ |
| 平均 | $O(n^2)$ |
| 最坏：逆序 | $O(n^2)$ |
| 空间复杂度 | $O(1)$ |
| 稳定性 | 稳定 |

## 四、关键结论

- 交换次数等于逆序对数量。
- 完全逆序时最大交换次数为：

$$
\frac{n(n-1)}{2}
$$

- 使用 `>` 而不是 `>=`，相等元素不交换，因此稳定。

## 七、完整执行示例

原数组：

```text
5  2  4  1
```

第一趟：

```text
5 与 2：交换 → 2  5  4  1
5 与 4：交换 → 2  4  5  1
5 与 1：交换 → 2  4  1  5
```

第二趟：

```text
2 与 4：不交换
4 与 1：交换 → 2  1  4  5
```

第三趟：

```text
2 与 1：交换 → 1  2  4  5
```

## 八、为什么交换次数等于逆序对数量？

每次交换相邻逆序元素，会恰好消除一个逆序对。

排序结束后逆序对数量为 `0`。

因此：

```text
冒泡交换次数 = 初始逆序对数量
```

## 九、边缘情况

- 空数组：无需操作。
- 单元素：无需操作。
- 已有序：第一趟无交换，提前结束。
- 逆序：交换最多。
- 重复元素：只有严格大于时交换，才能保持稳定。
