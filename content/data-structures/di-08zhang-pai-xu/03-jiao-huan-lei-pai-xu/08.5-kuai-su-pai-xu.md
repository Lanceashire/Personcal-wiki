## 条目概览

快速排序选择基准值，将较小元素放左边、较大元素放右边，再递归处理左右区间。

> [!summary]
> 快速排序选择基准值，将较小元素放左边、较大元素放右边，再递归处理左右区间。

上级知识点：[[排序总览]]

## 一、思想

```text
5  2  7  3  6  1  4
选择 pivot = 4
划分后：2  3  1 | 4 | 6  7  5
```

## 二、代码模板

```c
void quickSort(int arr[], int left, int right) {
    if (left >= right) return;

    int i = left;
    int j = right;
    int pivot = arr[left];

    while (i < j) {
        while (i < j && arr[j] >= pivot) j--;
        while (i < j && arr[i] <= pivot) i++;

        if (i < j) {
            int temp = arr[i];
            arr[i] = arr[j];
            arr[j] = temp;
        }
    }

    arr[left] = arr[i];
    arr[i] = pivot;

    quickSort(arr, left, i - 1);
    quickSort(arr, i + 1, right);
}
```

## 三、复杂度

| 情况    |        时间复杂度 |
| ----- | -----------: |
| 最好    | $O(n\log n)$ |
| 平均    | $O(n\log n)$ |
| 最坏    |     $O(n^2)$ |
| 平均递归栈 |  $O(\log n)$ |
| 最坏递归栈 |       $O(n)$ |
| 稳定性   |          不稳定 |

## 四、为什么会退化？

数组已有序且每次选最左元素为基准时，可能不断得到：

```text
左区间为空，右区间剩 n - 1 个元素
```

总成本近似：

$$
(n-1)+(n-2)+\cdots+1=O(n^2)
$$

## 五、常见优化

- 随机基准
- 三数取中
- 小区间使用插入排序
- 重复元素很多时使用三路划分

不同教材可能使用不同划分模板。考试时不要混用模板。

## 七、一趟划分到底保证什么？

一趟划分后，只保证：

```text
基准左侧元素不大于 pivot
基准右侧元素不小于 pivot
pivot 到达最终位置
```

左右区间内部：

```text
不一定有序
```

还需要递归处理。

## 八、为什么平均很快？

理想情况下，每次接近均分：

```text
n
→ n / 2
→ n / 4
→ ...
```

递归深度约为：

$$
\log_2 n
$$

每层处理总量约为：

$$
n
$$

所以：

$$
O(n\log n)
$$

## 九、为什么可能退化？

数组已经升序：

```text
1  2  3  4  5
```

每次选择最左元素：

```text
左侧为空
右侧剩 n - 1 个
```

递归深度接近：

$$
n
$$

因此退化为：

$$
O(n^2)
$$

## 十、边缘情况

- 空区间或单元素区间：立即返回。
- 大量重复元素：普通二路划分效率可能下降，可使用三路划分。
- 已排序数组：固定端点基准容易退化。
- 指针没有收缩：可能死循环。
- 不同教材模板边界不同：不要混用。
