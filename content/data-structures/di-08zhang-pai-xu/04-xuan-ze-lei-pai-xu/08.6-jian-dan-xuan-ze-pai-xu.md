## 条目概览

每一趟从未排序区中选出最小值，并将它交换到未排序区最前面。

> [!summary]
> 每一趟从未排序区中选出最小值，并将它交换到未排序区最前面。

上级知识点：[[排序总览]]

## 一、过程

```text
原始：5  2  4  1
第一趟选最小值 1：1 | 2  4  5
第二趟选最小值 2：1  2 | 4  5
第三趟选最小值 4：1  2  4 | 5
```

## 二、代码

```c
void selectionSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        int minIndex = i;

        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIndex]) {
                minIndex = j;
            }
        }

        if (minIndex != i) {
            int temp = arr[i];
            arr[i] = arr[minIndex];
            arr[minIndex] = temp;
        }
    }
}
```

## 三、复杂度

比较次数固定：

$$
\frac{n(n-1)}{2}
$$

| 项目 | 结论 |
|---|---:|
| 最好、平均、最坏时间 | $O(n^2)$ |
| 最多交换次数 | $n-1$ |
| 空间复杂度 | $O(1)$ |
| 稳定性 | 不稳定 |

## 四、为什么不稳定？

```text
5₁  5₂  2
```

第一趟把 `2` 与 `5₁` 交换：

```text
2  5₂  5₁
```

两个 `5` 的相对顺序改变。

## 五、与插入排序区别

- 插入排序：拿当前元素往前插。
- 选择排序：扫完整个未排序区，选最小值再交换。

## 七、完整过程

原数组：

```text
5  2  4  1
```

第一趟：

```text
扫描 5、2、4、1
最小值为 1
交换到下标 0
结果：1 | 2  4  5
```

第二趟：

```text
扫描 2、4、5
最小值为 2
结果：1  2 | 4  5
```

第三趟：

```text
扫描 4、5
最小值为 4
结果：1  2  4 | 5
```

## 八、为什么已有序仍然是 $O(n^2)$？

数组：

```text
1  2  3  4  5
```

即使看起来已经有序，每一趟仍需扫描未排序区，确认最小值确实在当前位置。

总比较次数仍为：

$$
\frac{n(n-1)}{2}
$$

## 九、为什么交换次数少？

每趟只在确定最小值后进行一次交换。

最多：

$$
n-1
$$

次交换。
