## 条目概览

升序堆排序通常建立大根堆，反复把堆顶最大值交换到数组末尾。

> [!summary]
> 升序堆排序通常建立大根堆，反复把堆顶最大值交换到数组末尾。

上级知识点：[[排序总览]]

## 一、堆

大根堆：每个父节点都大于或等于孩子，堆顶是最大值。

```text
        90
       /  \
     70    80
    / \    /
   20 60  50
```

## 二、下标公式

下标从 `0` 开始：

$$
左孩子=2i+1
$$

$$
右孩子=2i+2
$$

$$
父节点=\left\lfloor\frac{i-1}{2}\right\rfloor
$$

最后一个非叶子节点：

$$
\left\lfloor\frac{n}{2}\right\rfloor-1
$$

## 三、调整代码

```c
void heapify(int arr[], int n, int root) {
    int largest = root;
    int left = 2 * root + 1;
    int right = 2 * root + 2;

    if (left < n && arr[left] > arr[largest]) largest = left;
    if (right < n && arr[right] > arr[largest]) largest = right;

    if (largest != root) {
        int temp = arr[root];
        arr[root] = arr[largest];
        arr[largest] = temp;
        heapify(arr, n, largest);
    }
}
```

## 四、完整排序

```c
void heapSort(int arr[], int n) {
    for (int i = n / 2 - 1; i >= 0; i--) {
        heapify(arr, n, i);
    }

    for (int end = n - 1; end > 0; end--) {
        int temp = arr[0];
        arr[0] = arr[end];
        arr[end] = temp;
        heapify(arr, end, 0);
    }
}
```

## 五、复杂度

| 项目 | 复杂度 |
|---|---:|
| 建堆 | $O(n)$ |
| 总排序 | $O(n\log n)$ |
| 额外空间 | $O(1)$ |
| 稳定性 | 不稳定 |

## 六、常见错误

- 升序排序误用小根堆
- 孩子下标写错
- 交换堆顶后，调整范围没有缩小
- 忘记从最后一个非叶子节点开始建堆

## 七、为什么堆排序属于选择类排序？

简单选择排序每一趟要扫描未排序区，找出最大值或最小值。

堆排序也在反复“选择”极值，但它使用堆结构维护极值。

因此：

> 堆排序是对简单选择排序的改进。

## 八、数组与树之间的对应关系

下标从 `0` 开始：

| 关系 | 公式 |
|---|---|
| 左孩子 | $2i+1$ |
| 右孩子 | $2i+2$ |
| 父节点 | $\lfloor(i-1)/2\rfloor$ |
| 最后一个非叶子节点 | $\lfloor n/2\rfloor-1$ |

例如：

```text
数组：90  70  80  20  60  50
下标： 0   1   2   3   4   5
```

对应：

```text
        90
       /  \
     70    80
    / \    /
   20 60  50
```

## 九、为什么建堆是 $O(n)$ 而不是 $O(n\log n)$？

不是每个节点都会向下移动到最底层。

越靠近底层，节点越多，但可移动高度越小。

将所有节点实际调整成本加总后，常见结论为：

$$
O(n)
$$

## 十、边缘情况

- 空数组或单元素数组：天然有序。
- 左孩子存在但右孩子不存在：必须分别判断边界。
- 交换堆顶后：有效堆长度减小。
- 升序排序：通常用大根堆。
- 降序排序：通常用小根堆。
