## 一、总表

| 算法 | 最好 | 平均 | 最坏 | 额外空间 | 稳定性 |
|---|---:|---:|---:|---:|---|
| 直接插入排序 | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 稳定 |
| 希尔排序 | 与增量有关 | 与增量有关 | 与增量有关 | $O(1)$ | 不稳定 |
| 冒泡排序 | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 稳定 |
| 快速排序 | $O(n\log n)$ | $O(n\log n)$ | $O(n^2)$ | 平均 $O(\log n)$ | 不稳定 |
| 简单选择排序 | $O(n^2)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 不稳定 |
| 堆排序 | $O(n\log n)$ | $O(n\log n)$ | $O(n\log n)$ | $O(1)$ | 不稳定 |
| 归并排序 | $O(n\log n)$ | $O(n\log n)$ | $O(n\log n)$ | $O(n)$ | 稳定 |
| 基数排序 | $O(d(n+r))$ | $O(d(n+r))$ | $O(d(n+r))$ | $O(n+r)$ | 通常稳定 |

## 二、看到描述立刻认算法

| 描述 | 算法 |
|---|---|
| 像整理扑克牌 | 直接插入排序 |
| 按间隔分组，逐步缩小间隔 | 希尔排序 |
| 相邻交换，大数逐步到末尾 | 冒泡排序 |
| 选基准、左右分区、递归处理 | 快速排序 |
| 每趟选最小值放前面 | 简单选择排序 |
| 建堆，反复取堆顶 | 堆排序 |
| 先拆分，再合并 | 归并排序 |
| 按个位、十位、百位分桶 | 基数排序 |

## 三、场景选择

| 场景 | 优先想到 |
|---|---|
| 小数组或基本有序数组 | 直接插入排序 |
| 平均性能优秀的内存数组排序 | 快速排序 |
| 最坏仍为 $O(n\log n)$ 且原地 | 堆排序 |
| 稳定且最坏仍为 $O(n\log n)$ | 归并排序 |
| 超大磁盘文件 | 外部归并排序 |
| 位数有限的整数或字符串 | 基数排序 |

## 十、考前口诀

### 稳定性

```text
插帽归基稳
希快选堆不稳
```

### 一句话辨认

```text
插入：抓住当前牌，往前插
希尔：跨步插入，最后归一
冒泡：相邻交换，大数沉底
快排：找基准，左右分区
选择：扫完剩余，选择最小
堆排：堆顶最大，反复拿走
归并：先拆到底，再合并
基数：个位十位，依次装桶
```

## 十一、容易混淆的结论

1. 快排平均很快，但最坏是 $O(n^2)$。
2. 堆排序最坏是 $O(n\log n)$，且原地，但不稳定。
3. 归并排序稳定且最坏是 $O(n\log n)$，但需要额外空间。
4. 选择排序数组已有序时仍为 $O(n^2)$。
5. 插入排序基本有序时接近 $O(n)$。
6. 希尔排序复杂度依赖增量序列。
7. 基数排序不是比较排序。
8. 外部排序重点是磁盘 I/O。
