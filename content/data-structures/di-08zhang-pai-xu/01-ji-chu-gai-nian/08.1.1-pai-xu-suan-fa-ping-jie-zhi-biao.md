## 条目概览

时间 + 空间 + 稳定性 + 初始数据状态 + 使用场景

上级知识点：[[排序总览]]

## 一、必须比较的指标

| 指标    | 关注什么？            |
| ----- | ---------------- |
| 时间复杂度 | 数据规模增大后，排序成本怎样增长 |
| 空间复杂度 | 额外使用多少内存         |
| 稳定性   | 相同关键字记录的相对顺序是否保持 |
| 比较次数  | 比较关键字大小多少次       |
| 移动次数  | 记录被复制或移动多少次      |
| 交换次数  | 两个位置互换多少次        |
| 是否原地  | 是否只使用少量额外空间      |

## 二、稳定性口诀

```text
插帽归基稳
希快选堆不稳
```

对应：

| 稳定 | 不稳定 |
|---|---|
| 直接插入排序 | 希尔排序 |
| 冒泡排序 | 快速排序 |
| 归并排序 | 简单选择排序 |
| 基数排序 | 堆排序 |

## 三、初始顺序是否影响性能？

| 算法     | 初始顺序影响      |
| ------ | ----------- |
| 直接插入排序 | 明显，基本有序时很快  |
| 冒泡排序   | 明显，优化版可提前停止 |
| 简单选择排序 | 比较次数基本固定    |
| 快速排序   | 明显，基准不佳时会退化 |
| 堆排序    | 不明显         |
| 归并排序   | 不明显         |

## 四、核心结论

评价排序算法不能只看平均速度，还要综合考虑：

```text
时间 + 空间 + 稳定性 + 初始数据状态 + 使用场景
```

## 九、评价算法时容易犯的错误

### 错误 1：只背平均复杂度

快速排序平均为：

$$
O(n\log n)
$$

但最坏可能退化为：

$$
O(n^2)
$$

### 错误 2：只看速度，不看空间

归并排序时间复杂度稳定，但数组版本通常需要：

$$
O(n)
$$

辅助空间。

### 错误 3：忽略稳定性

排序结果数值正确，不代表业务结果一定符合预期。

### 错误 4：把交换次数和比较次数混为一谈

简单选择排序比较很多次，但每一趟最多交换一次。

### 错误 5：认为复杂算法一定更好

小规模或基本有序数据中，直接插入排序可能非常实用。

## 十、场景选择题模板

| 场景                   | 常见答案   |
| -------------------- | ------ |
| 基本有序的小数组             | 直接插入排序 |
| 希望平均速度很快             | 快速排序   |
| 要求稳定且最坏 $O(n\log n)$ | 归并排序   |
| 要求原地且最坏 $O(n\log n)$ | 堆排序    |
| 外部文件太大，无法一次进内存       | 外部归并排序 |
| 固定位数整数，可按位分配         | 基数排序   |
