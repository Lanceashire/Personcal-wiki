## 条目概览

排序解决的问题是：按照某个关键字，将一组记录重新排列成递增或递减序列。 本章重点不仅是代码，还包括复杂度、稳定性、比较次数、移动次数和适用场景。

## 三、分类速记

| 类别 | 算法 | 核心动作 |
|---|---|---|
| 插入类 | 直接插入排序 | 将当前元素插入前面的有序区 |
| 插入类 | 希尔排序 | 按间隔分组，跨步插入 |
| 交换类 | 冒泡排序 | 相邻交换，大数逐步到末尾 |
| 交换类 | 快速排序 | 选基准、左右分区、递归处理 |
| 选择类 | 简单选择排序 | 每趟选择最小值放到前面 |
| 选择类 | 堆排序 | 建堆，反复取堆顶 |
| 归并类 | 归并排序 | 先拆分，再有序合并 |
| 分配类 | 基数排序 | 按位分桶，再依次收集 |

## 八、考试答题时的固定思路

看到一种排序算法时，按以下顺序回答：

1. **核心动作是什么？**
2. **一趟之后哪个位置确定？**
3. **最好、平均、最坏复杂度是什么？**
4. **是否稳定？为什么？**
5. **额外空间是多少？**
6. **是否受初始顺序影响？**
7. **适合什么场景？**
8. **常见边缘情况有哪些？**
