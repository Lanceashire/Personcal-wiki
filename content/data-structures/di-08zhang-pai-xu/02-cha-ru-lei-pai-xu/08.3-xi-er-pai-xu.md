## 条目概览

希尔排序是分组的插入排序。先让元素进行远距离移动，再逐渐缩小间隔，最后以 gap = 1 完成排序。

> [!summary]
> 希尔排序是分组的插入排序。先让元素进行远距离移动，再逐渐缩小间隔，最后以 `gap = 1` 完成排序。

上级知识点：[[排序总览]]

## 一、核心思想

常见增量：

```text
gap = n / 2
gap = gap / 2
...
gap = 1
```

当 `gap = 3` 时，按照下标间隔分组，而不是切成连续小数组。

## 二、代码

```c
void shellSort(int arr[], int n) {
    for (int gap = n / 2; gap > 0; gap /= 2) {
        for (int i = gap; i < n; i++) {
            int temp = arr[i];
            int j = i;

            while (j >= gap && arr[j - gap] > temp) {
                arr[j] = arr[j - gap];
                j -= gap;
            }

            arr[j] = temp;
        }
    }
}
```

## 三、特点

| 项目 | 结论 |
|---|---|
| 时间复杂度 | 与增量序列有关 |
| 空间复杂度 | $O(1)$ |
| 稳定性 | 不稳定 |
| 本质 | 跨步插入，最后归一 |

## 四、为什么不稳定？

相同关键字可能位于不同分组，跨步移动时相对顺序可能改变。

## 五、常见错误

- 忘记最终必须有 `gap = 1`
- 组内移动误写成 `j--`，应为 `j -= gap`
- 死背一个固定时间复杂度

## 七、逐行拆解

```c
for (int gap = n / 2; gap > 0; gap /= 2)
```

逐步缩小增量。

```c
for (int i = gap; i < n; i++)
```

对于每个分组，首元素天然有序，所以从第二个同组元素开始。

```c
while (j >= gap && arr[j - gap] > temp)
```

与同组前一个元素比较。

```c
j -= gap;
```

不是 `j--`，因为组内元素间隔为 `gap`。

## 八、为什么能加速？

直接插入排序中，元素只能一步步移动。

希尔排序允许：

```text
一次跨越 gap 个位置
```

使远距离逆序关系更早得到改善。

## 九、边缘情况

- `n = 0` 或 `n = 1`：无需排序。
- `gap = 1`：退化为直接插入排序。
- 增量序列不合理：可能影响性能。
- 分组时不是连续切片，而是按下标间隔分组。
