## 条目概览

像整理扑克牌：每次拿出一个新元素，将它插入前面已经排好序的区域。

> [!summary]
> 像整理扑克牌：每次拿出一个新元素，将它插入前面已经排好序的区域。

上级知识点：[[排序总览]]

## 一、过程

```text
原始：5 | 2  4  6  1
插入 2：2  5 | 4  6  1
插入 4：2  4  5 | 6  1
插入 6：2  4  5  6 | 1
插入 1：1  2  4  5  6
```

## 二、C 语言代码

```c
void insertionSort(int arr[], int n) {
    for (int i = 1; i < n; i++) {
        int temp = arr[i];
        int j = i - 1;

        while (j >= 0 && arr[j] > temp) {
            arr[j + 1] = arr[j];
            j--;
        }

        arr[j + 1] = temp;
    }
}
```

## 三、关键理解

1. `arr[0]` 单独一个元素天然有序，所以从 `i = 1` 开始。
2. 先保存 `temp`，再移动较大元素，避免覆盖当前值。
3. 最终插入位置是 `j + 1`。
4. 使用 `>` 而不是 `>=`，相等元素不会互相跨越，因此稳定。

## 四、复杂度

| 情况 | 时间复杂度 |
|---|---:|
| 最好：已有序 | $O(n)$ |
| 平均 | $O(n^2)$ |
| 最坏：逆序 | $O(n^2)$ |
| 额外空间 | $O(1)$ |
| 稳定性 | 稳定 |

## 五、适用场景

- 小规模数组
- 基本有序数组
- 需要稳定排序

## 六、常见错误

- 忘记保存 `temp`
- `j >= 0` 判断位置错误导致越界
- 插入位置误写成 `arr[j]`
- 使用 `>=` 导致稳定性被破坏

## 七、逐行拆解

```c
for (int i = 1; i < n; i++)
```

`arr[0]` 单独一个元素天然有序，所以从第二个元素开始插入。

```c
int temp = arr[i];
```

先保存待插入元素，否则后移操作可能覆盖它。

```c
int j = i - 1;
```

从有序区末尾开始向前寻找插入位置。

```c
while (j >= 0 && arr[j] > temp)
```

只要前面的元素更大，就向后移动。

```c
arr[j + 1] = arr[j];
```

注意这是“移动”，不是不断交换。

```c
arr[j + 1] = temp;
```

循环结束后，`j + 1` 才是正确插入位置。

## 八、边缘情况

### 空数组或单元素数组

不会进入外层循环，天然正确。

### 已经有序

```text
1  2  3  4
```

每次只比较一次，不需要移动。

### 完全逆序

```text
4  3  2  1
```

每个新元素都需要一路向前移动，成本最高。

### 重复元素

只要判断条件使用：

```c
arr[j] > temp
```

而不是：

```c
arr[j] >= temp
```

相等元素不会互相跨越，因此稳定。

## 九、为什么和简单选择排序很像？

两者都维护：

```text
已排序区 | 未排序区
```

但区别是：

| 对比项 | 直接插入排序 | 简单选择排序 |
|---|---|---|
| 关注谁 | 当前待插入元素 | 剩余区最小元素 |
| 动作 | 元素后移，再插入 | 扫描完后交换 |
| 已有序时 | 很快 | 仍需完整扫描 |
| 稳定性 | 稳定 | 不稳定 |
