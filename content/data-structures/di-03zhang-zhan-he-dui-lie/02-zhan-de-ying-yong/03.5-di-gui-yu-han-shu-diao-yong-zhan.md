## 条目概览

每次函数调用都要暂存参数、局部变量和返回位置。后调用的函数必须先返回，因此运行时自然使用栈管理函数调用。

> [!summary]

## 1. 什么是栈帧

一次函数调用对应一个活动记录，也称栈帧。概念上可能保存：

- 实参或形参；
- 局部变量；
- 返回地址；
- 调用者部分状态；
- 临时计算结果。

具体布局由编译器、平台和调用约定决定，但“每次调用需要独立保存现场”这一思想不变。

## 2. 为什么符合 LIFO

```c
main()
  → functionA()
      → functionB()
```

返回顺序必须是：

```text
functionB 返回
→ functionA 返回
→ main 继续
```

最后调用的 `functionB` 最先返回，正是后进先出。

## 3. 阶乘调用过程

```c
int factorial(int n) {
    if (n <= 1) {
        return 1;
    }

    return n * factorial(n - 1);
}
```

计算：

```text
factorial(4)
```

递归展开：

```text
factorial(4)
→ 4 * factorial(3)
→ 4 * 3 * factorial(2)
→ 4 * 3 * 2 * factorial(1)
```

压栈：

```text
栈顶
[factorial(1)]
[factorial(2)]
[factorial(3)]
[factorial(4)]
栈底
```

回归：

```text
factorial(1) 返回 1
factorial(2) 返回 2
factorial(3) 返回 6
factorial(4) 返回 24
```

## 4. 递归的两个阶段

### 递推阶段

问题规模不断缩小，新的函数调用不断压栈。

### 回归阶段

达到递归出口后，栈帧逐层弹出，计算结果逐层返回。

缺少递归出口会导致调用不断增长，最终可能发生调用栈溢出。

## 5. 递归三要素

1. **递归定义**：大问题如何转成更小的同类问题；
2. **规模缩小**：每次调用必须向终止条件靠近；
3. **递归出口**：最小问题直接求解，不再调用自身。

例如阶乘：

$$
n!=n\times(n-1)!
$$

出口：

$$
1!=1
$$

## 6. 时间与空间复杂度

阶乘递推式：

$$
T(n)=T(n-1)+O(1)
$$

所以：

$$
T(n)=O(n)
$$

最大递归深度为 $n$，每层栈帧使用常数级空间：

$$
S(n)=O(n)
$$

> [!warning] 对话中暴露的易错点
> 递归函数即使没有显式申请数组，也会因调用栈产生额外空间。空间复杂度不能只看代码里有没有 `malloc`。

## 7. 斐波那契递归为什么慢

```c
long long fib(int n) {
    if (n <= 1) {
        return n;
    }

    return fib(n - 1) + fib(n - 2);
}
```

调用树中存在大量重复计算：

```text
fib(5)
├── fib(4)
│   ├── fib(3)
│   └── fib(2)
└── fib(3)
```

朴素递归时间复杂度近似指数级：

$$
O(2^n)
$$

最大递归深度仍为：

$$
O(n)
$$

时间复杂度取决于总调用数量，空间复杂度取决于同时存在的最大调用深度，二者不能混淆。

## 8. 递归转迭代

阶乘可改写为：

```c
int factorialIterative(int n) {
    int result = 1;

    for (int i = 2; i <= n; i++) {
        result *= i;
    }

    return result;
}
```

时间仍为：

$$
O(n)
$$

辅助空间降低为：

$$
O(1)
$$

但并非所有递归都适合简单循环替代。树遍历、回溯等问题常需显式栈保存待处理状态。

## 9. 尾递归

尾递归是递归调用成为函数最后一步：

```c
int factorialTail(int n, int acc) {
    if (n <= 1) {
        return acc;
    }

    return factorialTail(n - 1, n * acc);
}
```

某些语言或编译器可能优化尾调用，但 C 标准不保证尾递归优化。分析 C 程序空间时，不能默认它一定是 $O(1)$。

## 10. 递归与显式栈

递归：

```text
由运行时自动管理调用栈
```

非递归改写：

```text
程序员显式创建栈保存待处理状态
```

例如二叉树深度优先遍历，可以使用递归，也可以使用 `Node *` 栈。

## 11. 常见错误

1. 没有递归出口；
2. 参数没有向出口靠近；
3. 把递归调用总数当作递归深度；
4. 忽略栈帧空间；
5. 默认 C 一定优化尾递归；
6. 递归层数过大导致栈溢出；
7. 用朴素递归重复计算大量子问题。
