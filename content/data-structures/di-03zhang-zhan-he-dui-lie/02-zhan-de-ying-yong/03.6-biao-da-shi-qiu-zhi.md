## 条目概览

表达式求值的核心困难是运算符有优先级和结合性。后缀表达式消除了括号与优先级歧义，可以用一个操作数栈从左到右求值。

> [!summary]
> 表达式求值的核心困难是运算符有优先级和结合性。后缀表达式消除了括号与优先级歧义，可以用一个操作数栈从左到右求值。

## 1. 三种表达式

以：

$$
(3+4)\times5
$$

为例：

| 类型 | 表达式 |
|---|---|
| 中缀 | `(3 + 4) * 5` |
| 前缀 | `* + 3 4 5` |
| 后缀 | `3 4 + 5 *` |

中缀符合人类习惯，但需要优先级和括号；后缀表达式的运算顺序由排列本身决定。

## 2. 后缀表达式求值

规则：

1. 遇到操作数，入栈；
2. 遇到二元运算符，连续弹出两个操作数；
3. 先弹出的是右操作数，后弹出的是左操作数；
4. 计算结果重新入栈；
5. 扫描结束后，栈中应只剩最终结果。

## 3. 为什么操作数顺序不能反

后缀：

```text
8 3 -
```

栈：

```text
先压 8
再压 3
```

遇到 `-`：

```c
right = pop();  /* 3 */
left = pop();   /* 8 */
result = left - right;
```

得到：

```text
8 - 3 = 5
```

若反写会变成 `3 - 8`。

## 4. 逐步示例

后缀表达式：

```text
2 1 + 3 *
```

| 记号 | 操作 | 栈 |
|---|---|---|
| `2` | 入栈 | `2` |
| `1` | 入栈 | `2, 1` |
| `+` | 计算 `2+1` | `3` |
| `3` | 入栈 | `3, 3` |
| `*` | 计算 `3*3` | `9` |

结果为 `9`。

## 5. C 语言核心实现

以下假设输入已被切分成整数与运算符记号：

```c
int applyOperator(
    int left,
    int right,
    char operator,
    int *result
) {
    if (result == NULL) return 0;

    switch (operator) {
        case '+':
            *result = left + right;
            return 1;
        case '-':
            *result = left - right;
            return 1;
        case '*':
            *result = left * right;
            return 1;
        case '/':
            if (right == 0) return 0;
            *result = left / right;
            return 1;
        default:
            return 0;
    }
}
```

完整解析还需处理：

- 多位数；
- 空格；
- 负数；
- 非法记号；
- 除零；
- 整数溢出。

## 6. 中缀转后缀

需要：

- 输出序列；
- 运算符栈。

规则概要：

1. 操作数直接输出；
2. 左括号入栈；
3. 右括号到来时，弹出至左括号；
4. 普通运算符到来时，先弹出优先级更高的栈顶运算符；
5. 扫描结束后弹出所有剩余运算符。

例如：

```text
中缀：a + b * c
后缀：a b c * +
```

乘法优先，因此 `*` 在后缀中先于 `+` 被执行。

## 7. 运算符优先级与结合性

| 运算符 | 优先级 | 常见结合性 |
|---|---:|---|
| `()` | 最高控制作用 | - |
| `* / %` | 高 | 左结合 |
| `+ -` | 低 | 左结合 |

左结合意味着：

```text
a - b - c
```

解释为：

```text
(a - b) - c
```

## 8. 双栈直接求中缀表达式

也可使用：

```text
操作数栈 + 运算符栈
```

遇到新运算符时，根据优先级决定：

- 直接压栈；
- 先执行栈顶运算；
- 遇到括号时处理到对应边界。

这种方法可直接求值，但规则更复杂。学习顺序建议：

```text
后缀求值
→ 中缀转后缀
→ 双栈中缀求值
```

## 9. 复杂度

若记号数为 $n$，每个记号最多入栈、出栈常数次：

$$
T(n)=O(n)
$$

最坏栈空间：

$$
S(n)=O(n)
$$

## 11. 常见错误

1. 减法、除法颠倒左右操作数；
2. 将多位数逐字符处理；
3. 未处理负数；
4. 中缀转后缀时忽略结合性；
5. 遇到右括号后忘记丢弃左括号；
6. 表达式非法但仍返回栈顶；
7. 除数为零。
