## 条目概览

选择栈还是队列，本质上是在选择“下一步优先处理谁”：优先处理最近产生的任务用栈，优先处理最早等待的任务用队列。

> [!summary]
> 选择栈还是队列，本质上是在选择“下一步优先处理谁”：优先处理最近产生的任务用栈，优先处理最早等待的任务用队列。

## 1. 问题识别

看到以下特征优先想到栈：

- 最近一次操作撤销；
- 嵌套结构；
- 匹配最近未完成对象；
- 深度优先；
- 回溯；
- 需要逆序。

看到以下特征优先想到队列：

- 先到先处理；
- 按层扩展；
- 最短步数的无权图搜索；
- 任务调度与缓冲；
- 数据流排队。

## 2. 两个栈实现队列

维护：

```text
inStack：负责接收入队元素
outStack：负责提供出队元素
```

入队：

```text
直接压入 inStack
```

出队：

1. 若 `outStack` 非空，直接弹出；
2. 若 `outStack` 为空，把 `inStack` 全部转移过去；
3. 再从 `outStack` 弹出。

转移会把顺序反转：

```text
inStack 底→顶：1 2 3
转移后 outStack 顶→底：1 2 3
```

最早进入的 `1` 位于 `outStack` 栈顶。

### 均摊复杂度

单次转移可能是 $O(n)$，但每个元素只会：

- 进入 `inStack` 一次；
- 转移一次；
- 从 `outStack` 弹出一次。

因此一系列操作的均摊时间为：

$$
O(1)
$$

> [!warning] 易错点
> 只有在 `outStack` 为空时才转移。每次出队都来回搬运会使效率恶化并破坏清晰的不变量。

## 3. 队列实现栈

使用一个队列：

1. 新元素入队；
2. 将前面的旧元素依次出队再入队；
3. 让新元素旋转到队头。

若入栈前有 $n$ 个元素，新元素入队后旋转 $n$ 次。

这样：

- 入栈 $O(n)$；
- 出栈 $O(1)$。

也可以选择让入栈 $O(1)$、出栈 $O(n)$，取决于将代价放在哪个操作。

## 4. BFS 为什么使用队列

广度优先搜索从起点开始按距离层次扩展：

```text
距离 0：起点
距离 1：起点的邻居
距离 2：邻居的未访问邻居
...
```

较早发现的结点应较早扩展，因此使用 FIFO 队列。

伪代码：

```text
起点入队并标记
while 队列非空:
    当前结点出队
    for 每个未访问邻居:
        标记邻居
        邻居入队
```

无权图中，第一次到达某结点时得到的路径边数最少。

## 5. 二叉树层序遍历

层序遍历同样使用队列：

```text
根结点入队
→ 弹出根，左右孩子入队
→ 按进入顺序继续处理
```

若要区分每一层，可在每轮开始记录：

```c
int levelSize = queueLength;
```

本轮只处理这 `levelSize` 个结点，新加入的孩子留到下一轮。

## 6. DFS 为什么常使用栈

深度优先搜索优先沿一条路径深入，尚未完成的分支需要暂存。

递归 DFS 使用隐式调用栈；非递归 DFS 使用显式栈。

```text
DFS：最近发现的结点优先 → 栈
BFS：最早发现的结点优先 → 队列
```

## 7. 单调栈入门

单调栈让栈中元素保持单调，用来寻找：

- 下一个更大元素；
- 下一个更小元素；
- 左右第一个更高或更低元素；
- 柱状图最大矩形。

处理“下一个更大元素”时，栈中通常保存尚未找到答案的下标。新元素更大时，不断弹出并确定答案。

每个元素最多入栈、出栈一次：

$$
O(n)
$$

## 8. 典型题目

| 问题 | 结构 | 核心原因 |
|---|---|---|
| 有效括号 | 栈 | 匹配最近左括号 |
| 逆波兰表达式 | 栈 | 暂存操作数 |
| 每日温度 | 单调栈 | 找右侧第一个更大值 |
| 二叉树层序遍历 | 队列 | 按层和发现顺序处理 |
| 岛屿数量 | 栈或队列 | DFS/BFS 均可 |
| 滑动窗口最大值 | 双端队列 | 维护窗口候选最大值 |
