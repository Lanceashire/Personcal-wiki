## 条目概览

链队列通常维护队头指针和队尾指针。带头结点后，入队在尾部连接新结点，出队删除首元结点；删除最后一个数据结点后必须让队尾重新指向头结点。

> [!summary]
> 链队列通常维护队头指针和队尾指针。带头结点后，入队在尾部连接新结点，出队删除首元结点；删除最后一个数据结点后必须让队尾重新指向头结点。

## 1. 存储结构

```c
typedef struct QueueNode {
    ElemType data;
    struct QueueNode *next;
} QueueNode;

typedef struct {
    QueueNode *front;
    QueueNode *rear;
    int length;
} LinkedQueue;
```

使用辅助头结点：

```text
front
  ↓
[头结点] → [10] → [20] → [30] → NULL
                              ↑
                             rear
```

空队列：

```text
front
  ↓
[头结点] → NULL
  ↑
 rear
```

因此：

```c
queue->front == queue->rear
```

## 2. 初始化

```c
int initLinkedQueue(LinkedQueue *queue) {
    if (queue == NULL) {
        return 0;
    }

    QueueNode *dummy = malloc(sizeof(QueueNode));

    if (dummy == NULL) {
        return 0;
    }

    dummy->next = NULL;
    queue->front = dummy;
    queue->rear = dummy;
    queue->length = 0;
    return 1;
}
```

## 3. 入队

```c
int enLinkedQueue(
    LinkedQueue *queue,
    ElemType value
) {
    if (
        queue == NULL ||
        queue->rear == NULL
    ) {
        return 0;
    }

    QueueNode *newNode = malloc(sizeof(QueueNode));

    if (newNode == NULL) {
        return 0;
    }

    newNode->data = value;
    newNode->next = NULL;

    queue->rear->next = newNode;
    queue->rear = newNode;
    queue->length++;
    return 1;
}
```

核心：

```text
旧队尾连接新结点
→ rear 移动到新队尾
```

## 4. 出队

待删除结点始终是：

```c
queue->front->next
```

```c
int deLinkedQueue(
    LinkedQueue *queue,
    ElemType *result
) {
    if (
        queue == NULL ||
        queue->front == NULL ||
        queue->front == queue->rear
    ) {
        return 0;
    }

    QueueNode *target = queue->front->next;

    if (result != NULL) {
        *result = target->data;
    }

    queue->front->next = target->next;

    if (queue->rear == target) {
        queue->rear = queue->front;
    }

    free(target);
    queue->length--;
    return 1;
}
```

## 5. 为什么最后一个结点要特殊处理

出队前：

```text
front → [头结点] → [唯一数据结点] → NULL
                         ↑
                        rear
```

删除该结点后，如果不修改 `rear`：

```text
rear 仍指向已经释放的内存
```

它会成为悬空指针。

正确状态：

```text
front → [头结点] → NULL
  ↑
 rear
```

## 6. 读取队头

```c
int getLinkedHead(
    const LinkedQueue *queue,
    ElemType *result
) {
    if (
        queue == NULL ||
        result == NULL ||
        queue->front == queue->rear
    ) {
        return 0;
    }

    *result = queue->front->next->data;
    return 1;
}
```

## 7. 销毁

```c
void destroyLinkedQueue(LinkedQueue *queue) {
    if (queue == NULL) {
        return;
    }

    QueueNode *current = queue->front;

    while (current != NULL) {
        QueueNode *nextNode = current->next;
        free(current);
        current = nextNode;
    }

    queue->front = NULL;
    queue->rear = NULL;
    queue->length = 0;
}
```

## 8. 不带头结点的链队列

也可以让 `front` 直接指向首元结点。

空队列：

```c
front == NULL && rear == NULL
```

入队第一个结点和删除最后一个结点都需要同时修改 `front`、`rear`，边界分支更多。

带头结点版本通常更适合教材学习和统一操作。

## 9. 复杂度

维护 `front` 与 `rear` 后：

| 操作 | 时间复杂度 |
|---|---:|
| 入队 | $O(1)$ |
| 出队 | $O(1)$ |
| 读取队头 | $O(1)$ |
| 判空 | $O(1)$ |

空间随元素数量增长：

$$
O(n)
$$

每个数据结点还需要一个指针域。

## 10. 链队列与循环队列比较

| 比较项 | 循环队列 | 链队列 |
|---|---|---|
| 存储 | 连续数组 | 动态结点 |
| 容量 | 固定或需扩容 | 按需申请 |
| 入队/出队 | $O(1)$ | $O(1)$ |
| 额外指针 | 无逐元素指针 | 每结点一个 |
| 缓存局部性 | 较好 | 较弱 |
| 失败原因 | 队满 | `malloc` 失败 |

## 11. 常见错误

1. 入队后忘记移动 `rear`；
2. 从头结点而不是首元结点读取数据；
3. 删除最后一个结点后未重置 `rear`；
4. 出队后忘记 `free`；
5. 销毁时漏掉头结点；
6. 空队列判断与是否带头结点的设计不一致。
