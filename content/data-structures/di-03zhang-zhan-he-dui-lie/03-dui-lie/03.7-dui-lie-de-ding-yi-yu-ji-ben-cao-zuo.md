## 条目概览

队列是只允许在一端插入、另一端删除的受限线性表。入队端称为队尾，出队端称为队头，遵循先进先出。

> [!summary]
> 队列是只允许在一端插入、另一端删除的受限线性表。入队端称为队尾，出队端称为队头，遵循先进先出。

## 1. 结构与术语

```text
出队 ← [A] [B] [C] ← 入队
       队头         队尾
```

若依次入队：

```text
A, B, C
```

出队顺序：

```text
A, B, C
```

英文：

```text
First In, First Out
```

简称：

```text
FIFO
```

## 2. 基本操作

| 操作 | 含义 |
|---|---|
| `InitQueue` | 初始化空队列 |
| `DestroyQueue` | 销毁队列 |
| `QueueEmpty` | 判断队列是否为空 |
| `EnQueue` | 从队尾插入 |
| `DeQueue` | 从队头删除并返回 |
| `GetHead` / `Front` | 读取队头但不删除 |
| `QueueLength` | 获取队列长度 |

## 3. 栈与队列对比

| 比较项 | 栈 | 队列 |
|---|---|---|
| 插入位置 | 栈顶 | 队尾 |
| 删除位置 | 栈顶 | 队头 |
| 顺序 | LIFO | FIFO |
| 典型应用 | 递归、括号、DFS | 调度、缓冲、BFS |

## 4. 队列为什么有用

队列表达“先到先处理”：

- 打印任务；
- 操作系统就绪队列；
- 网络数据缓冲；
- 消息队列；
- 广度优先搜索；
- 二叉树层序遍历；
- 生产者与消费者；
- 离散事件模拟。

## 5. 存储实现

```text
队列
├── 顺序队列
│   └── 循环队列
└── 链队列
```

无论实现方式如何，都必须明确：

- `front` 指向哪里；
- `rear` 指向哪里；
- 空队列如何表示；
- 满队列如何判断；
- 入队和出队后指针如何变化。

## 6. 上溢与下溢

### 队列下溢

空队列出队或读取队头。

### 队列上溢

固定容量顺序队列已满仍入队。

链队列则主要受动态内存是否申请成功限制。

## 7. 操作序列

初始空队列：

```text
[]
```

入队 `10, 20, 30`：

```text
[10, 20, 30]
 ↑           ↑
front       rear
```

出队一次：

```text
返回 10
[20, 30]
```

接着入队 `40`：

```text
[20, 30, 40]
```

## 8. 易错点

1. 把队头和队尾的操作方向写反；
2. 把队列理解为“两端都能随意删除”；
3. 未明确 `rear` 指向尾元素还是尾元素后一位置；
4. 空队列仍读取队头；
5. 顺序队列数组后部满时忽略前部空位；
6. 链队列最后一个结点出队后忘记同步队尾。
