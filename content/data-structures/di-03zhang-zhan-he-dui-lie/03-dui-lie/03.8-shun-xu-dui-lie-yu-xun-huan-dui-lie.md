## 条目概览

普通顺序队列会产生“数组前面有空位但队尾已到末端”的假溢出。循环队列通过模运算把数组首尾逻辑相连，重复利用已释放位置。

> [!summary]
> 普通顺序队列会产生“数组前面有空位但队尾已到末端”的假溢出。循环队列通过模运算把数组首尾逻辑相连，重复利用已释放位置。

## 1. 普通顺序队列

约定：

- `front` 指向当前队头元素；
- `rear` 指向下一个可写位置；
- 空队列时 `front == rear`。

```c
#define QUEUE_CAPACITY 100

typedef struct {
    ElemType data[QUEUE_CAPACITY];
    int front;
    int rear;
} SeqQueue;
```

初始：

```c
front = 0;
rear = 0;
```

## 2. 假溢出

假设容量为 5：

```text
初始：
[ ][ ][ ][ ][ ]
 F,R
```

入队 5 个，再出队 3 个：

```text
[空][空][空][40][50]
             F       R已到数组末端
```

数组前面有空位，但若 `rear` 只能继续向右，就无法再入队。这不是存储空间真正用完，而是**假溢出**。

每次出队后搬移所有元素可以解决，但单次出队会变成 $O(n)$。

## 3. 循环队列思想

将数组逻辑上看成环：

```text
0 → 1 → 2 → ... → capacity-1
↑                       ↓
└───────────────────────┘
```

下一个位置：

$$
next(i)=(i+1)\bmod capacity
$$

当指针到达数组末端后，下一步回到下标 `0`。

## 4. 牺牲一个存储单元方案

本节采用最常见约定：

- `front` 指向队头元素；
- `rear` 指向下一个可写位置；
- 保留一个位置不用，以区分空和满。

空队列：

$$
front=rear
$$

满队列：

$$
(rear+1)\bmod capacity=front
$$

最大有效元素个数：

$$
capacity-1
$$

## 5. 初始化

```c
typedef struct {
    ElemType data[QUEUE_CAPACITY];
    int front;
    int rear;
} CircularQueue;

void initQueue(CircularQueue *queue) {
    if (queue == NULL) {
        return;
    }

    queue->front = 0;
    queue->rear = 0;
}
```

## 6. 入队

```c
int enQueue(
    CircularQueue *queue,
    ElemType value
) {
    if (queue == NULL) {
        return 0;
    }

    if (
        (queue->rear + 1) %
            QUEUE_CAPACITY ==
        queue->front
    ) {
        return 0;
    }

    queue->data[queue->rear] = value;
    queue->rear =
        (queue->rear + 1) % QUEUE_CAPACITY;
    return 1;
}
```

顺序：

```text
先在 rear 写入
→ rear 前进到下一空位
```

## 7. 出队

```c
int deQueue(
    CircularQueue *queue,
    ElemType *result
) {
    if (
        queue == NULL ||
        queue->front == queue->rear
    ) {
        return 0;
    }

    if (result != NULL) {
        *result = queue->data[queue->front];
    }

    queue->front =
        (queue->front + 1) % QUEUE_CAPACITY;
    return 1;
}
```

顺序：

```text
读取 front
→ front 前进
```

## 8. 读取队头

```c
int getQueueHead(
    const CircularQueue *queue,
    ElemType *result
) {
    if (
        queue == NULL ||
        result == NULL ||
        queue->front == queue->rear
    ) {
        return 0;
    }

    *result = queue->data[queue->front];
    return 1;
}
```

## 9. 队列长度公式

由于 `rear` 可能绕到 `front` 前面，不能直接使用：

```text
rear - front
```

统一公式：

$$
length=
(rear-front+capacity)\bmod capacity
$$

例如：

```text
capacity = 8
front = 6
rear = 2
```

则：

$$
length=(2-6+8)\bmod8=4
$$

## 10. 为什么空和满会冲突

如果不保留空位，并仍只使用 `front`、`rear`：

```text
空队列：front == rear
装满并绕一圈后：front == rear
```

同一状态无法区分空和满。

解决方案有三类：

1. 牺牲一个存储单元；
2. 增加 `size` 记录元素个数；
3. 增加标志位，记录最后一次操作是入队还是出队。

## 11. 使用 `size` 的方案

```c
typedef struct {
    ElemType data[QUEUE_CAPACITY];
    int front;
    int rear;
    int size;
} SizedQueue;
```

判空：

```c
size == 0
```

判满：

```c
size == QUEUE_CAPACITY
```

这种方案可使用全部数组位置，但每次入队和出队都必须正确更新 `size`。

## 12. 复杂度

| 操作 | 时间复杂度 |
|---|---:|
| 判空、判满 | $O(1)$ |
| 入队 | $O(1)$ |
| 出队 | $O(1)$ |
| 读取队头 | $O(1)$ |
| 求长度 | $O(1)$ |

## 14. 常见错误

1. 忘记模运算；
2. 用 `rear == capacity` 直接判满；
3. 未说明是否牺牲一个位置；
4. 长度直接写 `rear - front`；
5. 入队时先移动 `rear`，与当前约定不匹配；
6. 空队列读取队头；
7. 容量和最大有效元素数混淆。
