## 条目概览

顺序栈使用数组保存元素，并使用 top 记录栈顶位置。本笔记统一约定：top 指向当前栈顶元素下标，空栈时 top = -1。

> [!summary]
> 顺序栈使用数组保存元素，并使用 `top` 记录栈顶位置。本笔记统一约定：`top` 指向当前栈顶元素下标，空栈时 `top = -1`。

## 1. 存储结构

```c
#define STACK_CAPACITY 100

typedef int ElemType;

typedef struct {
    ElemType data[STACK_CAPACITY];
    int top;
} SeqStack;
```

初始化：

```c
void initStack(SeqStack *stack) {
    if (stack != NULL) {
        stack->top = -1;
    }
}
```

## 2. `top` 的含义

```text
下标：  0    1    2    3
数据： [3]  [6]  [9]  [空]
                   ↑
                  top
```

此时：

```c
stack.top == 2
stack.data[stack.top] == 9
```

> [!warning] 对话中暴露的易错点
> `top` 是整数下标，`data[top]` 才是栈顶元素。两者不是同一个概念。

## 3. 判空与判满

空栈：

```c
stack->top == -1
```

满栈：

```c
stack->top == STACK_CAPACITY - 1
```

当前元素个数：

$$
length=top+1
$$

## 4. 入栈

逻辑步骤：

```text
检查满栈
→ top 上移
→ 在新栈顶写入元素
```

代码：

```c
int push(SeqStack *stack, ElemType value) {
    if (
        stack == NULL ||
        stack->top >= STACK_CAPACITY - 1
    ) {
        return 0;
    }

    stack->data[++stack->top] = value;
    return 1;
}
```

为什么使用：

```c
++stack->top
```

因为当前 `top` 指向旧栈顶，需要先上移到新位置，再写入。

等价写法：

```c
stack->top++;
stack->data[stack->top] = value;
```

## 5. 出栈

逻辑步骤：

```text
检查空栈
→ 读取当前栈顶
→ top 下移
```

```c
int pop(SeqStack *stack, ElemType *result) {
    if (
        stack == NULL ||
        stack->top < 0
    ) {
        return 0;
    }

    if (result != NULL) {
        *result = stack->data[stack->top];
    }

    stack->top--;
    return 1;
}
```

也可写为：

```c
*result = stack->data[stack->top--];
```

但初学阶段拆开写更容易检查执行顺序。

## 6. 读取栈顶

```c
int getTop(
    const SeqStack *stack,
    ElemType *result
) {
    if (
        stack == NULL ||
        result == NULL ||
        stack->top < 0
    ) {
        return 0;
    }

    *result = stack->data[stack->top];
    return 1;
}
```

不修改 `top`，因此元素仍留在栈中。

## 7. 操作过程

初始：

```text
top = -1
```

依次入栈 `3, 6, 9`：

```text
push(3): top = 0, data[0] = 3
push(6): top = 1, data[1] = 6
push(9): top = 2, data[2] = 9
```

出栈一次：

```text
返回 9
top 从 2 变为 1
```

数组中的旧值 `9` 可能仍存在，但已不属于有效栈。

## 8. 另一种 `top` 约定

有些实现让 `top` 指向下一个可写位置：

```text
空栈 top = 0
入栈 data[top++] = value
出栈 value = data[--top]
```

两种约定都可行，但判空、判满和操作公式不同。

| 约定 | 空栈 | 入栈 |
|---|---|---|
| 指向当前栈顶 | `top == -1` | `data[++top] = x` |
| 指向下一空位 | `top == 0` | `data[top++] = x` |

> [!warning] 易错点
> 不要把两套约定的公式混合使用。写题前先用一句话标明 `top` 的语义。

## 9. 共享栈

两个栈可以共享同一数组，从两端向中间增长：

```text
栈1 → → → [空闲区域] ← ← ← 栈2
```

设：

```c
top1 = -1;
top2 = capacity;
```

满栈条件：

```c
top1 + 1 == top2
```

共享栈适合两个栈空间需求此消彼长的情况，可以减少固定分区造成的浪费。

## 10. 复杂度

| 操作 | 时间复杂度 |
|---|---:|
| 判空、判满 | $O(1)$ |
| 入栈 | $O(1)$ |
| 出栈 | $O(1)$ |
| 读取栈顶 | $O(1)$ |

静态顺序栈占用固定容量空间：

$$
O(capacity)
$$

## 11. 常见错误

1. 入栈前不判满；
2. 出栈前不判空；
3. `top` 初值与操作公式不匹配；
4. `data[top++] = value` 在当前约定下覆盖旧栈顶；
5. `pop` 后清零数组才认为完成，混淆物理残留与逻辑有效性；
6. 把 `STACK_CAPACITY` 当作最后合法下标。

4. 出栈后为什么不必清零旧数组位置？
5. 两种 `top` 约定能否混用？
