## 条目概览

栈是只允许在线性表同一端插入和删除的受限线性表。允许操作的一端称为栈顶，另一端称为栈底。

> [!summary]
> 栈是只允许在线性表同一端插入和删除的受限线性表。允许操作的一端称为栈顶，另一端称为栈底。

## 1. 规范定义

栈可写为：

$$
S=(a_1,a_2,\ldots,a_n)
$$

其中：

- $a_1$ 位于栈底；
- $a_n$ 位于栈顶；
- 插入和删除都在栈顶进行。

```text
栈顶 top
  ↓
[an]
[...]
[a2]
[a1]
  ↑
栈底 bottom
```

## 2. 后进先出

最后压入的元素最先弹出：

```text
压入顺序：A → B → C
弹出顺序：C → B → A
```

英文：

```text
Last In, First Out
```

简称：

```text
LIFO
```

## 3. 基本操作

| 操作 | 含义 |
|---|---|
| `InitStack` | 初始化空栈 |
| `DestroyStack` | 销毁栈 |
| `StackEmpty` | 判断栈是否为空 |
| `Push` | 将元素压入栈顶 |
| `Pop` | 删除并返回栈顶元素 |
| `GetTop` / `Peek` | 读取栈顶元素但不删除 |
| `StackLength` | 获取当前元素个数 |

`Pop` 与 `GetTop` 的区别：

```text
Pop：读取 + 删除
GetTop：只读取
```

## 4. 操作序列是否合法

若按顺序将 `1, 2, 3` 入栈，可能的完整出栈序列包括：

```text
3, 2, 1
2, 3, 1
2, 1, 3
1, 3, 2
1, 2, 3
```

但：

```text
3, 1, 2
```

不可能。

原因：

1. 为先弹出 `3`，必须先压入 `1,2,3`；
2. 弹出 `3` 后，栈顶为 `2`；
3. 不可能越过 `2` 先弹出 `1`。

判断栈序列时，核心是不允许越过当前栈顶。

## 5. 栈为什么有用

很多问题都需要“暂存尚未完成的工作，并优先处理最近产生的工作”：

- 函数调用与返回；
- 递归；
- 括号匹配；
- 表达式求值；
- 浏览器返回；
- 编辑器撤销；
- 深度优先搜索；
- 二叉树非递归遍历；
- 回溯算法。

## 6. 栈的逻辑结构与存储实现

栈的 LIFO 规则属于逻辑层。

实现可以是：

```text
栈
├── 顺序栈：数组
└── 链栈：链表
```

无论实现方式如何，外部操作语义都不变。

## 7. 上溢与下溢

### 上溢

固定容量顺序栈已满仍执行入栈：

```text
stack overflow
```

### 下溢

空栈执行出栈或读取栈顶：

```text
stack underflow
```

链栈通常没有固定容量上溢，但 `malloc` 失败仍会导致入栈失败。

## 8. 易错点

> [!warning] 对话中暴露的易错点
> “只允许在同一端操作”指插入和删除都在栈顶。栈底不是另一个可自由操作的端点。

1. 把“栈顶位置”与“栈顶元素”混为一谈；
2. 把读取栈顶误写成出栈；
3. 空栈时仍访问栈顶；
4. 认为链栈永远不会入栈失败；
5. 判断出栈序列时只看元素是否出现，忽略 LIFO 约束。
