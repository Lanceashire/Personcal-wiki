## 条目概览

链栈以单链表头部作为栈顶。这样入栈是头插，出栈是删除首结点，两者都不需要遍历。

> [!summary]
> 链栈以单链表头部作为栈顶。这样入栈是头插，出栈是删除首结点，两者都不需要遍历。

## 1. 存储结构

```c
typedef struct StackNode {
    ElemType data;
    struct StackNode *next;
} StackNode;

typedef struct {
    StackNode *top;
    int length;
} LinkedStack;
```

结构：

```text
top
 ↓
[9] → [6] → [3] → NULL
```

空栈：

```c
stack->top == NULL
```

## 2. 为什么栈顶放在链表头部

单链表头部插入：

```text
不需要寻找前驱
```

单链表头部删除：

```text
直接修改头指针
```

均为：

$$
O(1)
$$

若将栈顶放在普通单链表尾部，出栈时为了找到新尾结点通常需要寻找前驱：

$$
O(n)
$$

## 3. 初始化

```c
void initLinkedStack(LinkedStack *stack) {
    if (stack == NULL) {
        return;
    }

    stack->top = NULL;
    stack->length = 0;
}
```

链栈通常不需要辅助头结点，因为 `top` 本身就可以直接指向首个数据结点。

## 4. 入栈

原栈：

```text
top
 ↓
[8] → [5] → NULL
```

压入 `11`：

```text
top
 ↓
[11] → [8] → [5] → NULL
```

代码：

```c
int pushLinked(
    LinkedStack *stack,
    ElemType value
) {
    if (stack == NULL) {
        return 0;
    }

    StackNode *newNode = malloc(sizeof(StackNode));

    if (newNode == NULL) {
        return 0;
    }

    newNode->data = value;
    newNode->next = stack->top;
    stack->top = newNode;
    stack->length++;
    return 1;
}
```

顺序不能反：

```c
newNode->next = stack->top;
stack->top = newNode;
```

先接住旧栈顶，再移动 `top`。

## 5. 出栈

原栈：

```text
top
 ↓
[10] → [7] → [4] → NULL
```

弹出后：

```text
top
 ↓
[7] → [4] → NULL
```

代码：

```c
int popLinked(
    LinkedStack *stack,
    ElemType *result
) {
    if (
        stack == NULL ||
        stack->top == NULL
    ) {
        return 0;
    }

    StackNode *target = stack->top;

    if (result != NULL) {
        *result = target->data;
    }

    stack->top = target->next;
    free(target);
    stack->length--;
    return 1;
}
```

> [!note] 对话补充
> 链栈出栈与单链表删除完全遵循同一内存安全顺序：先抓住目标，再移动入口，最后释放。

## 6. 读取栈顶

```c
int getLinkedTop(
    const LinkedStack *stack,
    ElemType *result
) {
    if (
        stack == NULL ||
        stack->top == NULL ||
        result == NULL
    ) {
        return 0;
    }

    *result = stack->top->data;
    return 1;
}
```

## 7. 销毁

```c
void destroyLinkedStack(LinkedStack *stack) {
    if (stack == NULL) {
        return;
    }

    StackNode *current = stack->top;

    while (current != NULL) {
        StackNode *nextNode = current->next;
        free(current);
        current = nextNode;
    }

    stack->top = NULL;
    stack->length = 0;
}
```

## 8. 链栈会不会“满”

链栈没有预设固定容量，不会像静态顺序栈那样因为数组位置耗尽而满。

但仍可能因为：

- 系统可用内存不足；
- `malloc` 失败；
- 人为设置最大长度；

而无法继续入栈。

所以“链栈永不满”并不严谨。

## 9. 顺序栈与链栈比较

| 比较项 | 顺序栈 | 链栈 |
|---|---|---|
| 存储 | 连续数组 | 分散结点 |
| 入栈/出栈 | $O(1)$ | $O(1)$ |
| 固定容量 | 静态版本有 | 通常无 |
| 每元素额外指针 | 无 | 有 |
| 缓存局部性 | 较好 | 较弱 |
| 内存申请 | 初始化或扩容时 | 每次入栈 |
| 实现风险 | 越界 | 泄漏、悬空指针 |

## 10. 常见错误

1. 把栈顶放在单链表尾部，却仍声称出栈为 $O(1)$；
2. 入栈时先覆盖 `top`，丢失旧链；
3. 出栈后不释放目标结点；
4. 释放目标后再读取其 `next`；
5. `length` 与实际结点数不同步；
6. 销毁后未把 `top` 置为 `NULL`。
